# 0018 SPEC Command history completion

Type: spec

Replaces:
- none

Related:
- 0013
- 0016

## Goal

Improve command-line history so executed commands are stored once, repeated commands
move to the top of history, and command entry can use Far-like history completion.

## Context

CSharpFar already has a command line and command execution flow. The current request
changes how command history is stored and how the command line behaves depending on
panel visibility:

1. history must not accumulate duplicate commands;
2. repeating an existing command must make it the most recent history item;
3. when panels are hidden through `Ctrl+O`, Up and Down must browse command history in
   the command line;
4. when panels are visible, typing a command prefix must show a non-blocking
   suggestion window with commands that start with the typed text.

## Scope

Implement command history as a unique, recency-ordered list.

Rules:

1. Do not store empty or whitespace-only commands.
2. If an executed command already exists in history, remove the old entry and add the
   command as the most recent item.
3. Keep history ordering predictable for `Alt+F8`, Up/Down browsing, and completion.
4. Preserve existing command execution behavior.

Implement hidden-panel command-line history browsing:

1. When panels are hidden by `Ctrl+O`, Up and Down browse command history in the command
   line instead of panel selection.
2. Browsing should replace the current command-line text with the selected history
   entry.
3. Browsing is not circular. When the user reaches the oldest or newest history item,
   additional Up or Down presses stay on that item.
4. Editing or typing after browsing should continue from the text currently displayed.

Implement visible-panel command completion:

1. When panels are visible and the user starts typing command text, show a suggestion
   window above the command line.
2. Suggestions contain history commands that start with the current command-line text.
3. Suggestions are unique and ordered by history recency.
4. Up and Down move the selection inside the suggestion window while it is visible.
5. Enter accepts the selected suggestion and places that command text into the command
   line.
6. Escape temporarily hides the suggestion window.
7. After Escape hides the window, the next command-line typing/editing key may show it
   again if matches exist.
8. The suggestion window must not block normal command-line typing or editing, except
   that Up and Down select suggestions while the window is visible.
9. The window is displayed above the command line with a single-line border and no
   shadow.
10. The window content uses up to 8 command rows, plus the border rows.
11. If the console is too short, reduce the suggestion window height or hide it.

## Non-goals

Do not implement in this spec:

1. persistent command history across application restarts unless it already exists;
2. fuzzy, substring, or case-sensitive mode switches for completion;
3. a one-panel UI mode;
4. configurable history size or key bindings;
5. changing `Alt+F8` semantics beyond duplicate-free recency ordering;
6. completion for file names, paths, environment variables, or aliases;
7. a new UI framework for the suggestion window.

## Plan

1. Locate current command history storage, `Alt+F8`, command-line rendering, and panel
   visibility handling.
2. Change history insertion to unique recency ordering.
3. Add command-line history browsing state for the hidden-panels mode.
4. Add suggestion state and rendering above the command line for visible-panels mode.
5. Route Up, Down, Enter, Escape, and text-editing keys through the new command-line
   behavior without changing unrelated panel commands.
6. Add focused tests for history uniqueness, hidden-panel browsing, visible-panel
   suggestions, and small-height rendering.

## Pause points

- Stop before changing command execution if history insertion cannot be isolated from
  process launch.
- Stop before adding a persistent history store if the current code only has in-memory
  history.
- Stop before changing low-level console input parsing; this spec should use existing
  key events unless current events are insufficient.

## Acceptance criteria

- Executing the same command several times leaves one history entry.
- Repeating an old command moves it to the most recent history position.
- `Alt+F8` shows duplicate-free command history in recency order.
- With panels hidden by `Ctrl+O`, Up and Down browse command history in the command
  line.
- Hidden-panel history browsing is not circular; it stops at the oldest and newest
  commands.
- With panels visible, typing a command prefix shows a suggestion window above the
  command line when matching history commands exist.
- The suggestion window lists only commands that start with the current command-line
  text.
- The suggestion window has a single-line border and no shadow.
- Up and Down select suggestions while the suggestion window is visible.
- Enter accepts the selected suggestion into the command line without executing it.
- Escape hides the suggestion window until the next command-line typing or editing key.
- Command-line typing and editing continue while suggestions are shown.
- The suggestion window uses at most 8 content rows, plus its border, and hides or
  shrinks cleanly on short consoles.
- Existing panel navigation, `Ctrl+O`, `Alt+F8`, and command execution behavior still
  work outside the new command-line cases.

## Outcome

Implementation summary:
- Changed in-memory and JSON command history insertion to ignore whitespace-only
  commands, remove an existing exact command before adding the new item, and keep the
  resulting list ordered oldest-to-newest for existing consumers.
- Normalized loaded JSON command history in memory so old duplicate command entries do
  not appear in `Alt+F8` or completion.
- Added visible-panel command history completion state to `Application`: typed command
  text refreshes prefix matches, Up/Down select matches while the list is visible,
  Enter accepts the selected command into the command line, and Escape temporarily hides
  the list until the next command-line edit.
- Added hidden-panel command history browsing for `Ctrl+O` mode: Up starts from the
  most recent command, additional Up moves older, Down moves newer, and both directions
  stop at history edges.
- Added `CommandHistoryCompletionRenderer` for a single-bordered suggestion window above
  the command line, capped at 8 content rows and hidden when the console has no room for
  the border plus at least one command row.
- Added focused tests for duplicate-free command history, JSON normalization, hidden
  command-line history browsing, non-circular history edges, visible completion
  acceptance, bordered rendering, Escape hiding, and short console behavior.
- Follow-up fix: command history is now updated before shell execution starts, so a
  long-running or stuck command is already present in in-memory history and persisted
  by the JSON history store before the process blocks the UI.
