# 0052 SPEC Viewer Far Hotkeys

Type: spec

Replaces:
- none

Related:
- 0012

## Goal

Make the F3 file viewer accept the main Far Manager viewer keys so users can navigate,
switch modes, search, change encoding, edit, and move to neighbouring files without
learning a CSharpFar-only layout.

## Context

`0032` and `0033` made the viewer a single streaming text/hex implementation with manual
encoding selection. The current keyboard layout still differs from Far Manager:

- text/hex toggle is `H` instead of `F4`;
- `F8` opens the full encoding picker instead of cycling common code pages;
- `F3` does not close the viewer;
- viewer-local search is missing;
- `+` and `-` cannot move to neighbouring files from the panel list;
- several Far keys are not recognized and fail silently.

The built-in editor from `0046` is now available, so `F6` can be wired from viewer to
editor without introducing a separate editing path.

## Scope

- Add Far-like key handling to `LargeFileViewer`:
  - `F1` shows viewer help;
  - `F2` toggles line wrapping and `Shift+F2` toggles word/character wrap mode;
  - `F3`, `Esc`, and `F10` close the viewer;
  - `F4` toggles text/hex mode, with `H` kept as an alternate key;
  - `F5`, `Alt+F5`, `F9`, `Ctrl+F10`, `F11`, `Alt+F11`, bookmark keys, and unsupported
    UI toggles show a clear "not supported" message;
  - `F6` opens the current file in the built-in editor when an editor launcher is
    available;
  - `F7` opens viewer search, `Shift+F7` and `Space` repeat forward search, and
    `Alt+F7` searches backwards;
  - `Alt+F8` opens the existing go-to dialog;
  - plain `F8` cycles common encodings and `Shift+F8` opens the full encoding picker;
  - `+` and `-` move to the next or previous regular file when the viewer was opened
    from a local panel file list;
  - `Ctrl+Left` and `Ctrl+Right` scroll horizontally by a wider Far-like step;
  - `Ctrl+Shift+Left` and `Ctrl+Shift+Right` jump to the start/end of the current
    displayed line;
  - `Alt+PageUp` and `Alt+PageDown` move several pages at once.
- Preserve the current streaming viewer design. Do not load the entire file for ordinary
  rendering or navigation.
- Add viewer-local text search with case-sensitive, whole-word, and regex options.
- Add viewer-local hex byte sequence search in hex mode.
- Highlight the current found match and clear it with `Ctrl+U`.
- Copy the current found text to the clipboard with `Ctrl+C` or `Ctrl+Insert` when a
  clipboard service is available.
- Keep the existing `G` go-to key while adding `Alt+F8`.
- Update README viewer keys and focused tests.

## Non-goals

- Printing files with `Alt+F5`.
- Far plugin menu/history support for `F11` and `Alt+F11`.
- Persistent viewer bookmarks or viewer history.
- Persistent viewer settings.
- Full settings dialog for viewer colors and scrollbars.
- Mouse text selection and arbitrary text copy from the viewer.
- Exact Far acceleration timing for `Alt+PageUp`/`Alt+PageDown`; this spec only requires a
  deterministic larger page step.
- Changing Quick View behavior.

## Acceptance criteria

- `F3`, `Esc`, and `F10` close the viewer.
- `F4` and `H` both toggle text/hex mode.
- `F8` cycles common encodings; `Shift+F8` opens the full encoding picker; `Alt+F8` opens
  go-to.
- `F7`, `Shift+F7`, `Alt+F7`, and `Space` perform viewer-local search/repeat search.
- Search supports text mode and hex mode, highlights the current match, and `Ctrl+U`
  clears the highlight.
- `Ctrl+C` and `Ctrl+Insert` copy the current found text if a clipboard is available.
- `F6` invokes the built-in editor through the existing editor path when available.
- `+` and `-` move among regular local panel files and keep the panel cursor on the last
  viewed file after exit.
- Unsupported Far viewer keys produce an explicit message instead of silently doing
  nothing.
- The README viewer section documents the new keys.

## Outcome

Implementation summary:
- Added `LargeFileViewerOptions` so the viewer can receive local panel sibling file paths,
  a current-file callback, an editor launcher, and a clipboard service without depending
  directly on panel or editor state.
- Reworked `LargeFileViewer` key handling around modifiers and added Far-like commands:
  `F1`, `F2`/`Shift+F2`, `F3`, `F4`, `F6`, `F7` variants, `F8` variants, `+`/`-`,
  `Ctrl+Left`/`Ctrl+Right`, `Ctrl+Shift+Left`/`Ctrl+Shift+Right`, `Alt+PageUp`/
  `Alt+PageDown`, `Ctrl+U`, `Ctrl+C`, and `Ctrl+Insert`.
- Kept `H`, `F`, and `G` as existing alternate viewer keys.
- Added explicit unsupported messages for reserved Far viewer commands such as raw mode,
  print, settings, grep filter, plugin/history, panel positioning, UI toggles, undo
  movement, and numbered bookmarks.
- Added viewer-local find UI and search helpers for literal/regex text search, whole-word
  matching, case sensitivity, and hex byte sequence search in hex mode.
- Added current-match highlighting and clipboard copy for the active match.
- Added simple text wrapping and word/character wrap mode state.
- Wired local panel F3 opens to pass sibling-file context and an editor launcher. `+` and
  `-` now open the next/previous regular local panel file and update the panel cursor to
  the last viewed file.
- Added `BlockCache.Clear()` so the viewer can refresh cached bytes after returning from
  the editor.
- Updated built-in help, README viewer text, and focused viewer tests.

Deviations:
- `Alt+PageUp` and `Alt+PageDown` use a deterministic five-page jump instead of Far's
  smooth acceleration timing.
- Viewer search is streaming over lines/chunks, but it still runs synchronously inside the
  viewer loop. A cancellable progress UI remains follow-up work for very large searches.
- `F1` opens the existing full built-in help with an updated viewer section rather than a
  separate viewer-only help popup.
- `F6` opens the built-in editor through the existing editor entry point and refreshes the
  viewer after return. Passing the exact viewer byte/line position into the editor was not
  added because the editor currently has no public "open at position" contract.
- Unsupported Far commands show messages; printing, plugins, persistent bookmarks/history,
  and viewer settings were not implemented.

Follow-up:
- Add a cancellable viewer search progress path if large-file search latency becomes
  visible in real terminal use.
- Add an editor "open at position" contract, then pass the viewer position on `F6`.
- Add persistent viewer bookmarks/history and a real viewer settings dialog in separate
  specs if needed.
- Run manual terminal QA for the full Far key table on representative text, binary, and
  growing log files.
