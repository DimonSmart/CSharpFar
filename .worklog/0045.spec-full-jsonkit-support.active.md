## Goal

Make `FarNet.JsonKit` 0.5.1 function as a real CSharpFar/FarNet module for
the features documented by the JsonKit README: opening JSON, browsing array and
object panels, opening nested values, editing values, deleting or nulling
items, saving JSON back to its source file, and using the module menu commands.

## Context

Spec `0044` proved the first real slice: the official JsonKit module can be
loaded, `jk:open file=<sample.json>` dispatches through the shim, and the root
JSON object is shown as a read-only CSharpFar panel.

Full JsonKit support requires more than command loading. JsonKit relies on
FarNet panel/explorer lifecycle methods, child panels, panel command/key
callbacks, content editing, content replacement, delete semantics, and some UI
services. CSharpFar should expose those through the native FarNet host model
instead of restoring the removed proxy plugin architecture.

## Scope

- Keep targeting `FarNet.JsonKit` 0.5.1 copied under
  `FarNet\Modules\JsonKit`.
- Implement FarNet panel navigation needed by `Enter` on JSON arrays and
  objects.
- Implement FarNet explorer content read/edit paths used by JsonKit `F4`.
- Implement JsonKit delete and set-null behavior for selected items:
  `Del`/`F8` and `ShiftDel`/`ShiftF8`.
- Implement panel command/key dispatch needed by JsonKit `CtrlS` save and its
  panel-local menu actions.
- Implement enough FarNet editor or editor-equivalent support for JsonKit value
  editing to update the backing JSON node.
- Implement enough clipboard/help/menu behavior for documented JsonKit menu
  items: Edit array of strings, Open from clipboard, Help.
- Add smoke tests or harness checks around real JsonKit operations, not only
  synthetic fixtures.
- Keep unsupported FarNet APIs explicit and diagnostic-driven.

## Non-goals

- Supporting other large FarNet modules in this spec unless their required API
  directly overlaps JsonKit.
- Implementing Far Manager macro compatibility beyond what JsonKit needs.
- Adding a security sandbox for FarNet modules.
- Replacing CSharpFar's native module domain with the old plugin proxy model.
- Supporting JsonKit versions newer than 0.5.1 without a separate compatibility
  check.

## Plan

1. Inventory JsonKit runtime blockers with smoke harness commands for:
   opening a root object, entering nested arrays/objects, editing a scalar,
   deleting, setting null, saving, and menu actions.
2. Extend the FarNet shim with the exact official method/type signatures needed
   by JsonKit panel operations: `OpenFileEventArgs`, `GetContentEventArgs`,
   `SetFileEventArgs`, `SetTextEventArgs`, `PanelEventArgs`,
   `KeyEventArgs`/`KeyInfo`, and panel command/key callbacks.
3. Extend `FarNetPanelAdapter` with a FarNet-aware interface for native panel
   operations that do not fit plain `IFilePanelSource`: open child panel, edit
   current file, delete selected files, set selected files to null, save panel,
   and invoke panel menu actions.
4. Route CSharpFar `Enter`, `F4`, `F8`/`Del`, `ShiftF8`/`ShiftDel`, and `CtrlS`
   to the FarNet adapter when the active source is a FarNet panel; keep normal
   behavior for local/SFTP/FTP panels.
5. Implement a minimal editor bridge. Prefer CSharpFar's existing editor when
   possible, but add a callback path so saving editor content calls JsonKit's
   FarNet set-content API instead of only editing a temporary file.
6. Add real JsonKit smoke checks:
   - `jk:open file=<sample.json>` opens root object.
   - Enter on a nested array/object opens a child panel.
   - F4 edits a scalar and updates the panel item.
   - delete removes an item.
   - shift-delete sets an item to null.
   - CtrlS writes the changed JSON file.
   - menu Open from clipboard opens JSON text or a JSON file path when clipboard
     support is available in the test host.
7. Run `dotnet build CSharpFar.slnx`, `dotnet test CSharpFar.slnx`, and real
   JsonKit smoke harness checks.

## Pause points

- Stop if JsonKit editing requires a broad non-modal FarNet editor lifecycle
  that cannot be mapped safely to CSharpFar's current editor in a small patch.
- Stop if a required API would change CSharpFar's public module domain in a way
  that affects FTP/SFTP panels.
- Stop if full menu/clipboard support depends on platform APIs unavailable in
  tests; document the manual check instead of faking behavior in production.

## Acceptance criteria

- `FarNet.JsonKit` 0.5.1 is discovered and loaded through the CSharpFar-owned
  `FarNet.dll` shim.
- `jk:open file=<sample.json>` opens an object or array panel with no
  diagnostics.
- Enter on nested JSON arrays and objects opens the corresponding child panel.
- F4 editing updates JSON values in the panel.
- Del/F8 removes selected JSON items.
- ShiftDel/ShiftF8 sets selected JSON items to `null`.
- CtrlS saves modified JSON back to the source file.
- JsonKit menu actions either work or fail with explicit documented unsupported
  diagnostics; the final outcome records any remaining gap.
- `dotnet build CSharpFar.slnx` and `dotnet test CSharpFar.slnx` pass.
