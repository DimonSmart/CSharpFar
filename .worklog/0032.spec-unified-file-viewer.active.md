# 0032 SPEC Unified File Viewer

Type: spec

Replaces:
- 0031

Related:
- 0014
- 0016
- 0020

## Goal

Use one F3 viewer implementation for every file and let the user switch between text and
hex display modes.

## Context

`0031` introduced a streaming viewer for large files, but kept the old in-memory
`FileViewer` path for small text files. The current requirement removes that split:
viewing should always use the streaming byte reader, block cache, lazy scanner, and bounded
rendering path.

The viewer also needs an explicit display mode. Binary files should open as hex by
default, but every file must be viewable as text or as a 16-byte-per-row hex dump. Text
rendering must sanitize control characters before writing to the console so file content
cannot act as terminal control input/output.

## Scope

- Route every F3 open through the streaming viewer.
- Remove the old in-memory viewer loop from `FileViewer`.
- Keep `TextFileReader` only for other existing users such as editor and Quick View.
- Add a viewer display mode state with text and hex modes.
- Default binary-looking files to hex and text-looking files to text.
- Add a keyboard command to toggle the current file between text and hex mode.
- Keep navigation, horizontal scrolling, jump, and follow behavior working in both modes.
- Support vertical text scrolling with the mouse wheel.
- Render text mode with sanitized control characters.
- Render hex mode as a 16-byte-per-row dump with printable ASCII preview only.
- Update tests and README.

## Non-goals

- Editing files in either mode.
- Search in the viewer.
- Persistent per-file mode settings.
- Changing Quick View behavior.
- Removing `TextFileReader` from editor or Quick View.
- A richer binary editor.

## Plan

1. Replace `FileViewer.Show` with existence/error handling plus delegation to
   `LargeFileViewer`.
2. Add `LargeFileViewMode` and store it in `LargeFileViewerState`.
3. Change `LargeFileViewer` rendering and navigation to use current mode instead of
   `LineScanner.IsBinary` directly.
4. Add a toggle key for text/hex mode and show the current mode in the header/footer.
5. Add tests for small files using the unified path, binary default hex mode, text/hex
   toggling, and sanitized control characters.
6. Update README and run repository checks.

## Pause points

- Stop before removing `TextFileReader` from editor or Quick View.
- Stop before adding persisted viewer mode.
- Stop before adding search.

## Acceptance criteria

- F3 never calls the old in-memory `FileViewer` loop.
- Small and large files both use the streaming viewer path.
- Text-looking files open in text mode by default.
- Binary-looking files open in hex mode by default.
- The user can toggle the same file between text and hex mode.
- Control characters from file content are rendered as inert visible/safe characters or
  spaces, not sent to the console as controls.
- Existing large-file navigation and follow behavior still work.
- The mouse wheel scrolls the visible file content vertically.
- Quick View remains a bounded preview.

## Outcome

Implementation summary:
- `FileViewer` now only checks that the path exists and delegates F3 to `LargeFileViewer`.
  The old in-memory viewer loop was removed from `FileViewer`.
- Added `LargeFileViewMode` and stored the current mode in `LargeFileViewerState`.
- `LargeFileViewer` now renders based on explicit mode instead of using binary detection
  directly. Binary-looking files start in hex mode; text-looking files start in text mode.
- Added `H` to toggle the current file between text and 16-byte-per-row hex dump mode.
- Text, hex, page/line navigation, End/Home, `G` jumps, horizontal scrolling, and follow
  mode now use the same streaming state.
- Added `SanitizeTextForConsole` so text rendering replaces control characters before
  writing file content to the screen.
- Updated README with the unified viewer behavior and text/hex toggle.
- Added focused tests for small files using the streaming path, binary default hex mode,
  text-to-hex toggle, binary-to-text toggle, and control-character sanitizing.
- Added mouse-wheel vertical scrolling for the full-screen viewer.

Deviations:
- `TextFileReader` remains in the codebase because editor and Quick View still use it.
- Toggling between text and hex resets horizontal scroll to zero; byte/line position is
  preserved as closely as the target mode permits.

Follow-up:
- Manual QA for real terminal rendering of files containing escape bytes is still useful.
- Persistent per-file display mode remains out of scope.
