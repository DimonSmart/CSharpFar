# 008. Верхняя строка меню и общая тень popup/dialog окон

## Цель

Добавить в CSharpFar верхнюю строку меню в стиле Far Manager и сделать общий механизм popup-окна с тенью.

Меню скрыто в обычном состоянии и появляется только при явной активации:

```text
F9
Left mouse down/click на верхней строке экрана, Y = 0
```

На первом этапе реализуются только три верхних пункта, строго в таком порядке:

```text
Left    Right    Options
```

`Files` и `Commands` в этот этап не добавлять.

Меню работает как overlay поверх текущего экрана: панели под ним не перестраиваются, layout не меняется, после закрытия меню интерфейс возвращается в обычное состояние.

Поведение Far подтверждено документацией: меню активируется через `F9` или левую кнопку мыши по верхней строке; при `F9` выбирается меню активной панели; при активном меню `Tab` переключает меню левой/правой панели, а из `Files`/`Commands`/`Options` переключает на меню пассивной панели.

Источник: [Far Manager Documentation: Меню](https://documentation.help/Far-Manager-ru/Menus.html)

## Что исправлено относительно черновика

1. Порядок верхних пунктов должен быть `Left`, `Right`, `Options`, а не `Left`, `Options`, `Right`.
2. Изменение поведения `F9` намеренное: `F9` больше не открывает `SettingsDialog` напрямую, а открывает верхнее меню. `Ctrl+S` остается прямым shortcut для настроек.
3. Использовать существующую модель ввода `ConsoleInputEvent`, `KeyConsoleInputEvent`, `MouseConsoleInputEvent`. Новую параллельную модель `InputEvent` не создавать.
4. Использовать существующий рендеринг через `ScreenRenderer`, `Rect`, `CellStyle`. `IConsoleCanvas` не вводить.
5. Использовать существующий `PanelSide`, а не новый `PanelId`.
6. Использовать существующие enum значения `PanelViewMode.Full`, `PanelViewMode.BriefTwoColumns`, `SortMode.Name`, `SortMode.Extension`, `SortMode.Size`, `SortMode.LastWriteTime`.
7. UI label для сортировки по времени должен быть согласован с кодом: `Sort by last write time`.
8. `Wide mode` не добавлять, потому что такого режима сейчас нет.
9. Не создавать новые абстракции без необходимости: не добавлять `IOverlayRenderer`, `IPaletteService`, `ICommandDispatcher`, `ICommandAvailabilityService` в этой задаче.
10. Тень должна быть общей возможностью popup/dialog окон, а не частной реализацией dropdown-меню.

## Текущее состояние кода

В проекте уже есть:

1. Основной event loop в `Application.Run`, который читает `ConsoleInputEvent`.
2. `MouseConsoleInputEvent` с координатами, кнопкой, видом события и модификаторами.
3. `ScreenRenderer`, `Rect`, `CellStyle`, `DrawBox`, `DrawDoubleBox`, `FillRegion`.
4. `ConsolePalette` и `PaletteRegistry`.
5. `PanelSide.Left` / `PanelSide.Right`.
6. `PanelViewMode.Full` / `PanelViewMode.BriefTwoColumns`.
7. `SortMode.Name` / `Extension` / `Size` / `LastWriteTime`.
8. Настройки в `AppSettings.Panels.Options` и `AppSettings.Panels.FileHighlighting`.
9. Мышиная обработка панелей и hit testing панельных items.
10. Много modal dialogs, которые сейчас сами делают capture/restore и напрямую вызывают `DrawBox` / `DrawDoubleBox`.

Реализовать:

1. Data-driven модель верхнего меню.
2. Состояние меню.
3. Расчет layout верхних пунктов и dropdown.
4. Рендеринг верхней строки меню.
5. Рендеринг dropdown через общий popup renderer.
6. Общую тень для popup/dialog окон.

9. Hit testing верхнего меню и dropdown.
10. Выполнение команд меню через один menu command execution path в `Application`.
11. Меню `Left` и `Right` для операций над соответствующей панелью.
12. Меню `Options` для уже существующих настроек приложения.
13. Unit-тесты состояния, layout, hit testing, команд и popup shadow clipping.

Не реализовывать:

1. `Files`.
2. `Commands`.
3. Пользовательскую настройку состава меню.
4. Поиск по меню.
5. Подменю глубже одного уровня.
6. Новые настройки, которых еще нет в `AppSettings`.
7. Новый глобальный command dispatcher.
8. Новый palette service.
9. Новый canvas/rendering abstraction.
10. Mouse drag dropdown или scrollbar dropdown.

## Принцип реализации

Меню должно быть data-driven.

Нельзя рисовать меню кодом вида:

```csharp
screen.Write(1, 0, "Left", style);
screen.Write(8, 0, "Right", style);
screen.Write(16, 0, "Options", style);
```

Нужна модель данных:

```csharp
}
```

```csharp
public sealed record TopMenuItemDefinition

```


Renderer получает готовую модель, состояние и layout. Renderer не должен знать, что такое "левая панель", "сортировка", "скрытые файлы", "сохранение настроек".

## Состояние меню

Использовать отдельное состояние меню. Состояние должно быть предсказуемым: не хранить два поля, которые могут противоречить друг другу.

```csharp
public sealed class MenuState
{
    public MenuOpenState OpenState { get; set; } = MenuOpenState.Closed;
    public int ActiveTopMenuIndex { get; set; }
    public int ActiveDropdownItemIndex { get; set; }
}
```

```csharp
public enum MenuOpenState
{
    Closed,
    MenuBarOpen,
    DropdownOpen,
}
```

Видимость:

| Состояние | Что видно |
| --- | --- |
| `Closed` | Меню не видно |
| `MenuBarOpen` | Видна только верхняя строка меню |
| `DropdownOpen` | Видна верхняя строка и раскрытый dropdown |

Не добавлять отдельный `IsDropdownOpen`: `DropdownOpen` уже выражает это состояние.

## Интеграция с текущим `Application`

Меню живет в основном loop приложения.

Когда меню закрыто:

```text
F9 перехватывается menu controller.
Left mouse down/click по Y = 0 перехватывается menu controller.
Остальной input идет в текущие handlers Application.
```

Когда меню открыто:

```text
Весь key/mouse input сначала получает menu controller.
Для первой версии меню перехватывает весь input, кроме resize event.
```

Если `_panelsVisible == false` после `Ctrl+O`, меню не открывать. Это меню панелей, а не shell-underlay режим.

Если открыт modal dialog, `F9` игнорируется. В текущей архитектуре modal dialogs читают `ReadKey()` внутри своего loop, поэтому отдельный modal-stack для этого этапа не нужен.

`Ctrl+S` должен продолжить открывать текущий `SettingsDialog` напрямую.

## F9

`F9` теперь является намеренным breaking-change:

```text
Раньше: F9 открывал SettingsDialog.
Теперь: F9 открывает верхнее меню.
```

Поведение:

1. Если меню закрыто, открыть меню.
2. Если активна левая панель, выбрать `Left`.
3. Если активна правая панель, выбрать `Right`.
4. Если активной панели нет, выбрать `Options`.
5. Открывать dropdown сразу:

```text
OpenState = DropdownOpen
ActiveDropdownItemIndex = первый selectable пункт dropdown
```

Если меню уже открыто, `F9` оставляет его открытым и заново выбирает меню активной панели.

## Mouse activation

Клик мышью по верхней строке должен обрабатываться до текущей логики `HandleMouse`, иначе событие уйдет в panel hit testing.

Если меню закрыто:

```text
Left mouse down/click на Y = 0 -> открыть меню
```

Hit testing:

```text
Y = 0, X попал в Left    -> открыть Left dropdown
Y = 0, X попал в Right   -> открыть Right dropdown
Y = 0, X попал в Options -> открыть Options dropdown
Y = 0, X не попал в пункт меню -> открыть меню активной панели
```

Если меню открыто:

| Ситуация | Поведение |
| --- | --- |
| Клик по top item | Сделать top item активным и открыть его dropdown |
| Клик по dropdown item | Выполнить пункт |
| Клик по separator | Ничего не делать |
| Клик по disabled item | Ничего не делать |
| Клик вне меню | Закрыть меню |

Для первой версии использовать `MouseEventKind.Down`. Если драйвер в будущем начнет отдавать `Click`, считать `Down` и `Click` эквивалентными для меню.

## Keyboard navigation

Минимальный набор:

| Клавиша | Поведение |
| --- | --- |
| `F9` | Открыть меню активной панели; если уже открыто, перейти к меню активной панели |
| `Esc` | Закрыть меню |
| `Left` | Перейти к предыдущему верхнему пункту |
| `Right` | Перейти к следующему верхнему пункту |
| `Down` | Открыть dropdown текущего пункта или перейти к следующему selectable пункту dropdown |
| `Up` | Перейти к предыдущему selectable пункту dropdown |
| `Enter` | Выполнить текущий пункт dropdown |
| `Tab` | Переключить `Left` <-> `Right`; из `Options` перейти на меню пассивной панели |
| `Home` | Первый selectable пункт dropdown |
| `End` | Последний selectable пункт dropdown |
| `буква hotkey` | Активировать matching hotkey |

Правила:

1. Верхние пункты идут по порядку `Left`, `Right`, `Options`.
2. `Separator` пропускается при навигации.
3. Disabled-пункты пропускаются при `Up`/`Down`/`Home`/`End`, но отображаются.
4. `Enter` на disabled-пункте ничего не делает.
5. После выполнения команды меню закрывается.
6. Для checkbox/radio в первой версии тоже закрывать меню после выполнения.
7. Hotkey matching case-insensitive.
8. Если внутри одного dropdown есть конфликт hotkey, первый enabled/selectable пункт выигрывает. Конфликты не создавать.

## Layout

Layout считать отдельным сервисом/классом в `CSharpFar.App`, потому что он использует `Rect` из `CSharpFar.Console.Models`.

```csharp
public sealed record MenuLayout
{
    public required IReadOnlyList<Rect> TopItemBounds { get; init; }
    public Rect? DropdownBounds { get; init; }
}
```

```csharp
public sealed class MenuLayoutService
{
    public MenuLayout CalculateLayout(
        Rect screenBounds,
        MenuBarDefinition definition,
        MenuState state);
}
```

Правила top-level:

1. Top menu bar находится на `Y = 0`.
2. Пункты рисуются в порядке `Left`, `Right`, `Options`.
3. Между пунктами оставлять минимум три пробела визуального разделения.
4. `TopItemBounds` должны включать padding вокруг текста, чтобы mouse hit testing совпадал с тем, что видит пользователь.

Правила dropdown:

1. Dropdown открывается под активным top item.
2. Верхняя координата dropdown: `Y = 1`.
3. Ширина считается по максимальной длине отображаемого текста + padding + место под checkbox/radio marker.
4. Если dropdown не помещается справа, он сдвигается влево.
5. Если dropdown не помещается снизу, для первой версии допустимо clipped-rendering по границам экрана.
6. Layout не меняет размеры панелей.

## Общий popup renderer и тень

Тень должна быть общей для dropdown, modal dialogs и будущих popup/list окон.

Не делать shadow renderer внутри dropdown menu renderer.

Добавить concrete renderer в `CSharpFar.App.Rendering`:

```csharp
public sealed class PopupRenderer
{
    public void RenderPopup(
        ScreenRenderer screen,
        Rect bounds,
        PopupRenderOptions options,
        Action<ScreenRenderer, Rect> renderContent);
}
```

```csharp
public sealed record PopupRenderOptions
{
    public bool DrawBorder { get; init; } = true;
    public bool DrawShadow { get; init; } = true;
    public bool DrawDoubleBorder { get; init; }

    public required CellStyle BorderStyle { get; init; }
    public required CellStyle BackgroundStyle { get; init; }
    public required CellStyle ShadowStyle { get; init; }

    public int ShadowOffsetX { get; init; } = 1;
    public int ShadowOffsetY { get; init; } = 1;
}
```

Требования:

1. `PopupRenderer` сначала рисует shadow, потом background/border/content.
2. Shadow clipped по текущему screen bounds.
3. Shadow не меняет layout.
4. Shadow рисуется поверх текущего содержимого.
5. Если popup у правого или нижнего края, shadow частично или полностью обрезается.
6. В первой версии shadow можно рисовать пробелами с отдельным `ShadowStyle`.
7. Dropdown-menu обязан использовать `PopupRenderer`.
8. Все bounded modal dialogs, которые сейчас рисуют `DrawBox` или `DrawDoubleBox`, должны быть переведены на общий popup/dialog frame renderer на базе `PopupRenderer`.
9. Full-screen viewer/editor не являются popup/dialog окнами и shadow для них не требуется.

Рекомендуемый helper для диалогов:

```csharp
public sealed class DialogFrameRenderer
{
    public void RenderFrame(
        ScreenRenderer screen,
        Rect bounds,
        string title,
        bool doubleBorder,
        PopupRenderOptions options,
        Action<ScreenRenderer, Rect> renderContent);
}
```

Это concrete helper, не новая абстракция. Его задача - убрать дублирование рамки, title и shadow между текущими dialogs.

## Команды меню

Command ids:

```text
panel.setViewMode
panel.setSortMode
panel.toggleReverseSort
panel.refresh

settings.openPanelSettings
settings.toggleShowHiddenAndSystemFiles
settings.toggleHighlightFiles
settings.toggleSelectFolders
settings.toggleRightClickSelectsFiles
settings.toggleSortFoldersByExtension
settings.toggleShowStatusLine
settings.toggleShowFilesTotalInformation
settings.toggleShowFreeSize
settings.toggleShowScrollbar
settings.toggleShowSortModeLetter
settings.toggleShowParentDirectoryInRootFolders
settings.save
```

Args:

```csharp
public sealed record PanelCommandArgs
{
    public required PanelSide PanelSide { get; init; }
}
```

```csharp
public sealed record SetPanelViewModeArgs
{
    public required PanelSide PanelSide { get; init; }
    public required PanelViewMode ViewMode { get; init; }
}
```

```csharp
public sealed record SetPanelSortModeArgs
{
    public required PanelSide PanelSide { get; init; }
    public required SortMode SortMode { get; init; }
}
```

Command execution rules:

1. `panel.setViewMode` updates left/right view mode using existing `PanelViewMode` values and persists settings the same way current view-mode shortcuts do.
2. `panel.setSortMode` uses existing `PanelController.SetSortMode`. Existing behavior of toggling descending when the same sort mode is selected stays unchanged.
3. `panel.toggleReverseSort` toggles `FilePanelState.SortDescending` for the target panel and rebuilds/refreshed the panel view.
4. `panel.refresh` calls existing refresh path for the target panel.
5. `settings.openPanelSettings` opens current `SettingsDialog`.
6. `settings.toggle*` mutates existing `AppSettings` values only; no new settings fields are added.
7. Settings toggles refresh panels when they affect panel view/rendering.
8. Persist settings consistently with existing app behavior. Since current settings changes save immediately, menu settings changes may call `_saveSettings` immediately.
9. `settings.save` explicitly calls existing `_saveSettings` if available. If save callback is not available, do not add the item.

## Состав меню

### Left

Команды применяются к `PanelSide.Left`.

```text
Left
  Full mode
  Brief mode
  Separator
  Sort by name
  Sort by extension
  Sort by size
  Sort by last write time
  Separator
  Reverse sort
  Separator
  Refresh
```

Правила:

1. `Full mode` использует `PanelViewMode.Full`.
2. `Brief mode` использует `PanelViewMode.BriefTwoColumns`.
3. `Wide mode` не добавлять.
4. Current view mode можно отображать как radio checked.
5. Current sort mode можно отображать как radio checked.
6. `Refresh` disabled, если current directory target-панели не существует.
