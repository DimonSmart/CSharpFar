# 0053 SPEC Delete progress dialog

Type: spec

Replaces:
- none

Related:
- 0010
- 0019

## Goal

Show file deletion with a delete-specific progress dialog instead of reusing the copy
progress layout.

## Context

The current file operation progress dialog renders every non-scanning operation as a
copy operation. That makes delete progress show copy wording, a destination line, a
current-file byte progress bar, and resume details that only make sense for copying.

Deletion still needs aggregate progress: deleted item count and deleted byte volume
should remain visible.

## Scope

Implement a delete-specific progress rendering path:

1. Use a delete title for deletion progress.
2. Show deletion status and the current source path.
3. Keep total deleted item count and byte volume visible.
4. Keep elapsed time visible when available.
5. Remove copy-only details from delete progress:
   - destination path;
   - "to" line;
   - current-file byte progress bar;
   - resume offset and rollback details.

## Non-goals

Do not implement in this spec:

1. a new file deletion algorithm;
2. recycle-bin behavior changes;
3. provider delete behavior changes beyond progress reporting if needed;
4. pause/resume redesign;
5. progress UI changes for copy, move, or create-directory operations except where
   needed to keep the shared dialog code clear.

## Plan

1. Inspect file operation progress generation for local and provider delete.
2. Add delete-specific progress phase/reporting if the existing phase names cannot
   describe deletion without surprising callers.
3. Split `ProgressDialog` rendering so copy and delete use separate layouts.
4. Add or update focused tests for delete progress rendering and progress reports.
5. Run repository checks.

## Pause points

- Stop before changing public operation result semantics.
- Stop before changing copy progress layout beyond mechanical extraction.

## Acceptance criteria

- Delete progress dialog title is `Delete`.
- Delete progress does not show copy destination wording.
- Delete progress does not show current-file byte progress or resume details.
- Delete progress still shows deleted file/item count.
- Delete progress still shows deleted byte volume.
- Existing copy progress keeps its current copy-specific information.
