# 0020 SPEC Unified window scrollbars

Type: spec

Replaces:
- none

Related:
- 0007
- 0012
- 0014
- 0018

## Goal

Use one visual scrollbar implementation for every scrollable window-like UI surface.

Scrollbars should appear on the right window border, replacing that border segment, so
lists and form bodies do not lose an inner content column.

## Context

`ScrollBarRenderer` already exists, but file panels use it as a panel-only inner-column
feature controlled by `ShowScrollbar`. Other scrollable UI pieces keep local
`scrollTop` or `scrollOffset` state without a visual scrollbar.

The inconsistent behavior is visible in file panels, search results, command completion,
search progress results, history-style dialogs, and dialogs that do not fit vertically
in a short console window.

## Scope

- Keep `ScrollState` as the shared scroll data contract.
- Keep `ScrollBarRenderer` as the only visual scrollbar renderer.
- Draw vertical scrollbars on the right border of popup/dialog/panel windows.
- Make file panels and search result panels show scrollbars automatically when content is scrollable.
- Add shared scroll math for selected/focused list movement and form-body scrolling.
- Add scrollbar support to command completion, search progress results, history dialogs,
  drive selection, user menu, and top menu dropdowns.
- Make Search, File Operation, and Settings dialogs scroll their body when the console
  is too short for all controls.
- Remove the panel-specific `ShowScrollbar` setting and menu toggle.

## Non-goals

- Scrollbar dragging.
- Quick View scrolling; active spec `0014` keeps it out of scope.
- Full-screen viewer/editor scrollbars.
- A new UI framework.
- Compatibility support for the removed panel-only scrollbar setting.

## Plan

1. Extend `ScrollBarRenderer` with border-placement support.
2. Add optional scrollbar state to popup/frame rendering.
3. Replace panel inner-column scrollbar rendering with automatic right-border rendering.
4. Add shared scroll helpers and use them in list-like dialogs and search progress.
5. Add form-body scrolling to short-console Search, File Operation, and Settings dialogs.
6. Remove `ShowScrollbar` from settings/menu behavior.
7. Update and add focused tests.

## Pause points

- Stop before changing Quick View, viewer, or editor scroll behavior.
- Stop before adding mouse drag handling for scrollbar thumbs.

## Acceptance criteria

- There is one visual scrollbar renderer used by all scrollbar-enabled components.
- File panels and search results draw scrollbars on the panel frame border only when scrollable.
- Command completion and search progress result lists draw the same border scrollbar when scrollable.
- History, directory history, file history, drive selection, user menu, and oversized dropdowns draw the same border scrollbar when scrollable.
- Search, File Operation, and Settings dialogs remain usable in short console windows by scrolling their body.
- Panel content width is not reduced to reserve scrollbar space.
- `ShowScrollbar` is no longer a user-facing panel option.

## Outcome

Implementation summary:
- Added `ScrollStateCalculator` for shared selection/offset clamping.
- Added optional `VerticalScrollState` to popup/frame rendering and draw it through the existing `ScrollBarRenderer` on the right border.
- Changed Full and Brief file panels, including search result panels, to render automatic border scrollbars only when item count exceeds visible rows.
- Added border scrollbars to command completion, search progress results, history/file-history/directory-history dialogs, drive selection, user menu, and overflow dropdown menus.
- Added body scrolling to Search, File Operation, and Settings dialogs for short console heights.
- Removed the panel-only `ShowScrollbar` setting, menu item, and command id.
- Added focused tests for popup/frame border scrollbars, panel border placement, command completion, search progress, list dialogs, dropdowns, and short form dialogs.

Files changed:
- Core: `ScrollStateCalculator`, `AppSettings`, `MenuCommandIds`.
- Rendering/menu: popup/frame options, panel renderers, command completion, dropdown layout/rendering, panel/menu hit testing.
- Dialogs: Search, File Operation, Settings, Search Progress, History, Directory History, File History, Drive, User Menu.
- Tests: `Spec007ScrollbarTests`, `Spec007SettingsTests`, `UnifiedWindowScrollbarTests`.

Deviations:
- Scrollbars still do not render when a scrollable region has fewer than three rows, because the shared renderer needs room for up/down arrows and at least one track row.
- Short form dialogs use keyboard focus movement to keep controls visible; scrollbar dragging remains out of scope.

Follow-up:
- Manual console QA for very small terminal heights and real resize behavior is still useful because the unit tests use the fake console driver.
