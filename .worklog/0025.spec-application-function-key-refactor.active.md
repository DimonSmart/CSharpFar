# 0025 SPEC Application function key refactor

Type: spec

Replaces:
- none

Related:
- 0023
- 0013

## Goal

Reduce `Application` responsibility by moving function-key binding metadata into
a dedicated provider without changing behavior.

## Context

Architecture review 0023 identified `Application` as the largest hotspot. The
function-key binding table is a low-risk first extraction because the behavior is
already modeled by `FunctionKeyBinding`, `FunctionKeyLayer`, command ids, and
spec 0013.

## Scope

- Add a default function-key binding provider under `CSharpFar.App.FunctionKeys`.
- Make `FunctionKeyBinding` contain binding metadata only.
- Dispatch function-key execution in `Application` by command id.
- Preserve current function-key availability and unavailable-command behavior.
- Add focused provider tests.

## Non-goals

- Do not change `Program.cs`.
- Do not refactor command completion, hidden console underlay, file operations,
  search panel model, or dialog helpers.
- Do not introduce partial `Application` files.
- Do not add configurable or user-editable key bindings.

## Plan

1. Create `DefaultFunctionKeyBindingProvider` with the existing binding table.
2. Replace delegate-based `FunctionKeyBinding` with metadata and
   `RunsWhenUnavailable`.
3. Update `Application` to use command-id availability and dispatch methods.
4. Add focused tests for provider metadata.
5. Run repository checks.

## Pause points

- Stop if function-key dispatch cannot preserve the existing unavailable-command
  behavior without widening the refactor.

## Acceptance criteria

- `Application` no longer owns the function-key binding table.
- Function-key bar filtering still uses the same availability rules as dispatch.
- Existing function-key behavior remains covered by spec 0013 tests.
- Provider metadata is covered by focused tests.
- Repository checks pass.

## Outcome

Implementation summary:
- Added `DefaultFunctionKeyBindingProvider` with the existing function-key
  metadata table.
- Changed `FunctionKeyBinding` to metadata only: command id, layer, key, label,
  and `RunsWhenUnavailable`.
- Removed the large `BuildFunctionKeyBindings()` delegate table from
  `Application`.
- Routed function-key availability and execution through
  `CanExecuteFunctionKeyCommand` and `ExecuteFunctionKeyCommand`.
- Added focused tests for provider bindings, unavailable-command metadata, and
  duplicate chords.
- Reduced `Application.cs` from 2730 to 2592 lines.
