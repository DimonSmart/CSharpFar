## Goal

Add Far-like directory shortcuts so the user can configure up to ten numbered
directories and quickly navigate the active visible panel to one of them.

## Context

CSharpFar already has persisted application settings, application command
handlers, menu commands, panel navigation, modal dialogs, and rendering helpers.
Directory shortcuts are a built-in file-manager feature and should use those
existing mechanisms directly.

Each shortcut slot stores one directory path and an optional short visible name.
Slots are shown and edited in the Far-like order `1, 2, 3, 4, 5, 6, 7, 8, 9, 0`.

## Scope

- Add persisted directory-shortcut settings with ten numbered slots `0..9`.
- Add one normalization helper for malformed settings, slot ordering, empty
  paths, duplicate numbers, and names fitted to at most eight visible
  characters.
- Render configured shortcut labels over the lower double panel-border row
  without changing the function-key bar or command-line layout.
- Do not render the shortcut bar when both panels are hidden.
- Handle `RightCtrl+0..9` through the existing input infrastructure. If the
  logical input model cannot preserve right-control information, use
  `Ctrl+0..9` and document the input-layer limitation in code.
- Navigate the active visible panel to a configured existing directory, restart
  its watcher, and reset the same transient UI state as comparable navigation
  commands.
- Show a `Directory Shortcut` message and keep settings unchanged when the
  configured directory does not exist or cannot be loaded.
- Add `Options -> Directory shortcuts...`.
- Add a directory-shortcut list dialog with all ten slots and a per-slot edit
  dialog with `Name` and `Path` fields.
- Prefill an empty slot from the active panel path and derive a non-empty short
  default name when possible.
- Treat an empty edited path as clearing the slot. Do not require the edited
  path to exist.
- Persist dialog changes through the existing application settings save flow.
- Reuse shared popup/frame, text-input, button-bar, and scrollbar behavior where
  responsibilities match.
- Support basic mouse interaction for list selection, text-field focus, and
  button activation in the new reusable-control-based dialogs.
- Add focused tests for settings persistence, normalization, ordering,
  rendering, command behavior, and practical dialog behavior.

## Non-goals

- Do not support multiple paths in one slot.
- Do not add quick-save shortcuts such as `Ctrl+Shift+0..9`.
- Do not add shortcut usage statistics.
- Do not import Far Manager settings.
- Do not remove shortcuts automatically when a path is unavailable.
- Do not change `FunctionKeyBarRenderer`.
- Do not add plugin packaging or implement this through a plugin system.
- Do not add configurable key bindings.

## Plan

2. Add the settings model and normalization helper with focused tests.
3. Add shortcut-bar rendering on the panel lower-border row with focused tests.
4. Add the list and edit dialogs using existing UI controls.
5. Register menu and navigation commands and wire numeric control-key input.
6. Add integration-focused tests and run repository checks.

## Pause points

- Stop before changing the low-level console input contract if preserving
  right-control identity requires a broad input refactor.
- Stop before introducing a new general-purpose list or form abstraction if the
  existing dialogs cannot be reused without unrelated refactoring.

## Acceptance criteria

- `Options -> Directory shortcuts...` opens a dialog with ten slots ordered
  `1..9, 0`.
- The user can edit a slot name and path, clear a slot with an empty path, and
  persist configured shortcuts in the existing settings file.
- An empty slot editor is prefilled from the active panel directory.
- Configured shortcut labels appear over the lower double panel-border row and
  do not affect the function-key bar.
- No shortcut labels are drawn when no slot is configured or both panels are
  hidden.
- The supported control-number chord navigates the active visible panel to the
  configured existing directory without changing the passive panel or shortcut
  settings.
- A missing or inaccessible configured path shows an error and remains saved.
- Malformed settings are normalized predictably: invalid numbers are ignored,
  duplicate numbers use the last valid item, empty paths clear slots, and names
  are fitted to eight visible characters.
- New dialogs support their basic keyboard and mouse interactions through
  existing UI-control patterns.
- Existing tests pass.

## Outcome

Implementation summary:
- Added persisted `DirectoryShortcuts` settings with numbered shortcut items.
- Added one normalizer for malformed settings, display ordering `1..9, 0`,
  duplicate handling, empty-path clearing, eight-character names, and default
  name generation from a path.
- Added a dedicated shortcut-bar renderer and wired it over the lower panel
  border row without changing the function-key bar or command-line layout.
- Added `Options -> Directory shortcuts...`, the ten-row shortcut list dialog,
  and a two-field item editor. The list supports keyboard selection, mouse row
  selection, double-click editing, and shared dialog buttons. The editor uses
  shared text-input and button-bar controls with mouse field focus and button
  activation.
- Added application commands for opening the editor and navigating to a slot.
  Successful navigation updates only the active panel and restarts its watcher.
  Missing paths show `Directory Shortcut` feedback and remain configured.
- Wired numeric control-key handling through `Application.HandleKey`.
- Added focused tests for settings persistence, normalization, ordering,
  default names, rendering, command behavior, application key routing,
  application rendering, registry coverage, and item-editor clearing.
