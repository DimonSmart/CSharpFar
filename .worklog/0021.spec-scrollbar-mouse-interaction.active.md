# 0021 SPEC Scrollbar mouse interaction

Type: spec

Replaces:
- none

Related:
- 0020

## Goal

Make the unified vertical scrollbar respond to standard mouse actions.

## Context

Spec `0020` unified visual scrollbar rendering across panels, popups, dialogs, and
menus. The scrollbar currently looks consistent but does not provide mouse interaction.

## Scope

- Add shared scrollbar hit testing and position mapping.
- Support left-click on the up/down arrows.
- Support left-click on the track above/below the thumb.
- Support dragging the thumb to change the first visible item.
- Use the shared interaction logic in file panels, command completion, search progress,
  list dialogs, scrollable form dialogs, and dropdown menus where mouse input is already
  handled.

## Non-goals

- Horizontal scrollbar interaction.
- Quick View, full-screen viewer, or editor scrollbar behavior.
- Mouse wheel behavior changes.
- Visual pressed/hover states.

## Plan

1. Add shared scrollbar hit-test/result helpers near the existing scroll models.
2. Track scrollbar drag state in interactive components.
3. Route mouse down/move/up through the shared helper before normal item hit testing.
4. Add focused tests for arrow clicks, track clicks, thumb dragging, and panel integration.

## Pause points

- Stop before changing low-level console mouse parsing if existing events are insufficient.

## Acceptance criteria

- Clicking the top arrow scrolls up by one item.
- Clicking the bottom arrow scrolls down by one item.
- Clicking the track above the thumb scrolls up by one page.
- Clicking the track below the thumb scrolls down by one page.
- Dragging the thumb updates the first visible item proportionally.
- Normal item clicks still work outside scrollbar bounds.

## Verification

Commands:

```bash

```

## Outcome

Implementation summary:
- Added shared scrollbar hit testing, click handling, and thumb drag mapping in `ScrollBarInteraction`.
- Updated `ScrollBarRenderer` to use the same thumb geometry as mouse interaction.
- Added `ScrollBarMouseHandler` and `ScrollableListMouseHandler` for shared event handling.
- Wired scrollbar mouse support into file panels, command completion, search progress results, history-style dialogs, drive selection, user menu, scrollable form dialogs, and dropdown menus.
- Added tests for arrow clicks, track clicks, thumb dragging, and panel scrollbar click integration.

Deviations:
- Dropdown scrollbar interaction moves the active menu item to a selectable item within the newly visible range, because the menu model has an active item but no independent dropdown scroll offset.
- Components without mouse input loops remain out of scope, matching the original non-goals for Quick View, full-screen viewer, and editor.

Follow-up:
- Manual console QA for terminal-specific mouse reporting during drag is still useful.
