# 0058 SPEC Test and fallback cleanup

Type: spec

Replaces:
- none

Related:
- 0023
- 0024

## Goal

Remove low-value tests and fallback behavior that hides invalid local state.

## Context

The test suite has grown around old smoke checks, permanent skipped placeholders,
and repeated no-op fakes. Some configuration stores also silently replace corrupt
JSON with defaults or empty data, which hides broken user state instead of making
the failure visible.

## Scope

- Remove skipped or placeholder tests that do not execute useful assertions.
- Remove smoke tests that only check simple model construction or BCL behavior.
- Remove tests that preserve silent corrupt-JSON fallback behavior.
- Make settings and user-menu JSON load failures explicit.
- Move duplicated no-op shell and file-operation test doubles into shared fakes.

## Non-goals

- Do not remove FarNet compatibility inventory tests.
- Do not collapse the application command registry in this pass.
- Do not change valid settings or user-menu file formats.
- Do not refactor production command handlers beyond what is needed for tests.

## Plan

1. Delete placeholder and low-value tests.
2. Change corrupt JSON loading to fail explicitly.
3. Update JSON store tests to assert the new behavior.
4. Add shared no-op fakes and replace duplicated local definitions.
5. Run repository checks.

## Pause points

- Stop if corrupt config fallback is required by another active work document.
- Stop if shared fakes make application test setup less explicit.

## Acceptance criteria

- No permanently skipped placeholder test remains.
- No test asserts corrupt settings or user-menu JSON silently falls back.
- Repeated no-op shell and file-operation test doubles are centralized.
- `dotnet build CSharpFar.slnx` and `dotnet test CSharpFar.slnx` pass.

## Verification

Commands:

```bash
dotnet build CSharpFar.slnx
dotnet test CSharpFar.slnx
```

## Outcome

Implementation summary:
- Removed placeholder/low-value tests:
  `CoreModelSmokeTests`, `FarDefaultFixtureVerificationTests`, and
  `FileSystemServiceTests`.
- Removed two object-graph-only settings tests from `Spec007SettingsTests`.
- Changed corrupt `settings.json`, `history.json`, and `user-menu.json` loading
  to throw `InvalidDataException` instead of silently replacing the data with
  defaults, empty history, or an empty list.
- Narrowed `FileSystemLocationService` fallback to expected file-system
  metadata exceptions and documented the returned unknown-location contract.
- Updated JSON store tests to assert explicit corrupt-file failures.
- Added shared `NoOpShellService` and `NoOpFileOperationService` test fakes and
  removed duplicated local no-op implementations.

Files changed:
- Production JSON stores:
  `src/CSharpFar.App/Settings/JsonSettingsStore.cs`,
  `src/CSharpFar.App/History/JsonHistoryStore.cs`,
  `src/CSharpFar.App/UserMenu/UserMenuStore.cs`.
- File-system location fallback:
  `src/CSharpFar.FileSystem/FileSystemLocationService.cs`.
- Test cleanup and expectations:
  removed `tests/CSharpFar.Tests/CoreModelSmokeTests.cs`,
  `tests/CSharpFar.Tests/FarDefaultFixtureVerificationTests.cs`, and
  `tests/CSharpFar.Tests/FileSystemServiceTests.cs`; updated JSON store tests
  and settings tests.
- Shared fakes:
  `tests/CSharpFar.Tests/Fakes/NoOpShellService.cs` and
  `tests/CSharpFar.Tests/Fakes/NoOpFileOperationService.cs`; application-level
  tests now use these instead of local no-op classes.

Deviations:
- The first `dotnet build CSharpFar.slnx` run failed because a running
  `CSharpFar` process locked application output assemblies. The process ended
  before a forced stop could inspect it; rerunning the build passed.

Follow-up:
- Consider a separate command-layer cleanup to replace trivial one-class-per-id
  command handlers with a more compact registration model.

Verification:
- `dotnet build CSharpFar.slnx` passed with 0 warnings and 0 errors after the
  locked process was gone; the final rerun also passed with 0 warnings and 0
  errors.
- `dotnet test CSharpFar.slnx --no-build` passed: 908 passed, 0 skipped.
