# 007. Опции отображения и поведения файловых панелей

## Цель

Добавить набор Far-like опций для файловых панелей CSharpFar, но вписать их в уже существующую архитектуру проекта:

1. настройки хранятся в `AppSettings.Panels`, а не в новой корневой секции `panel`;
2. раскраска файлов уже реализована через `IFileHighlightService` и `panels.fileHighlighting`;
3. состояние панели уже представлено `FilePanelState`;
4. текущий enum сортировки называется `SortMode` и содержит `Name`, `Extension`, `Size`, `LastWriteTime`;
5. низкоуровневый console layer уже находится в `CSharpFar.Console`, Windows-specific файловые сервисы - в `CSharpFar.FileSystem`.

Опции `Show column titles` и `Show background screens number` в этот этап не входят. Уже существующий заголовок `Name` в `BriefTwoColumns` не удалять; отдельную настройку для него не добавлять.

## Что исправлено относительно черновика

1. JSON-секция должна быть `panels`, потому что текущая модель уже сериализуется как `AppSettings.Panels`. Секция `panel` не добавляется.
2. `Highlight files` не должен дублировать `highlightFiles`; он управляет существующим `panels.fileHighlighting.enabled`.
3. `Show hidden and system files` сейчас представлен двумя полями `ui.showHiddenFiles` и `ui.showSystemFiles`. В рамках этой задачи их надо перенести в настройки панели как одну Far-like опцию. Поддержку старых полей ради совместимости не добавлять, если отдельно не попросят.
4. `Select folders` по текущему поведению CSharpFar уже включен: `Insert`, `Ctrl+A` и инверсия выбирают директории. Поэтому default для новой опции - `true`.
5. `Sort folders by extension` по текущей реализации тоже фактически включен: `PanelController` применяет `SortMode.Extension` отдельно и к директориям, и к файлам. Поэтому default - `true`.
6. В текущем коде нет `PanelSortMode.DateTime` и `PanelSortMode.Unsorted`; использовать существующий `SortMode.LastWriteTime`. `Unsorted` не добавлять в этом этапе.
7. Текущий индикатор сортировки уже есть как `SortModeIndicator`: `n/e/s/w`, uppercase означает descending. Не заменять это на стрелки, чтобы не ломать одноколоночный layout.
8. Статусная область уже есть: `PanelStatusRenderer` рисует separator, строку текущего элемента и строку статистики. Спецификация должна параметризовать это поведение, а не описывать другой status line с нуля.
9. В проекте нет `IConsoleCanvas`; рендеринг идет через `ScreenRenderer`, `Rect` и `CellStyle`.
10. В проекте нет отдельного `CSharpFar.Platform.Windows`. Windows-specific код остается в существующих проектах: console input в `CSharpFar.Console`, файловые/volume сервисы в `CSharpFar.FileSystem`.
11. Для идентификации панели использовать существующий `PanelSide`, а не новый `FilePanelId`, пока панелей ровно две.
12. Auto refresh не может полагаться только на `FileSystemWatcher` event: текущий главный цикл блокируется на `ReadKey`. Нужно добавить event/input loop, который умеет просыпаться от файлового события или читать input с timeout/cancellation.

5. `disableIfObjectCountExceeds < 0` считать ошибочным значением settings и нормализовать к `0` при чтении или использовании.

## Panel view pipeline

Сейчас `PanelController.LoadDirectory` читает каталог через `IFileSystemService`, затем сортирует `state.Items`; `FileSystemService` одновременно добавляет `..`, фильтрует hidden/system и предварительно сортирует директории/файлы. Это надо разнести.

Добавить builder, который отвечает за построение view:

```csharp
public interface IPanelViewBuilder
{
    PanelView Build(PanelViewRequest request);
}

public sealed class PanelViewRequest
{
    public required string DirectoryPath { get; init; }
    public required AppSettings.PanelOptionsSettings Options { get; init; }
    public required SortMode SortMode { get; init; }
    public required bool SortDescending { get; init; }
    public required IReadOnlySet<string> SelectedPaths { get; init; }
}

public sealed class PanelView
{
    public required IReadOnlyList<FilePanelItem> Items { get; init; }
    public required PanelSummary Summary { get; init; }
    public required PanelAutoRefreshState AutoRefreshState { get; init; }
    public required bool IsRootDirectory { get; init; }
}
```

Pipeline:

```text
Read directory entries
-> Map filesystem metadata
-> Detect mount points, if enabled
-> Apply visibility filters
-> Add or remove ..
-> Sort items
-> Calculate panel summary from current view
-> Return PanelView
-> Render panel
```

Требования:

1. `IFileSystemService.ReadDirectory(path)` возвращает реальные элементы каталога без `..`, без visibility filtering и без финальной сортировки.
2. `PanelViewBuilder` добавляет `..` сам.
3. `PanelViewBuilder` скрывает Hidden/System элементы, если `ShowHiddenAndSystemFiles = false`.
4. Прямой переход в скрытый каталог не блокировать. Фильтрация применяется только к списку элементов, а не к `LoadDirectory(path)`.
5. `PanelController` не должен содержать код сортировки и filtering. Он управляет курсором, selection и навигацией.
6. Renderer не должен считать summary, определять mount points или фильтровать элементы.
7. `FilePanelState` можно расширить свойствами `PanelSummary Summary` и `PanelAutoRefreshState AutoRefreshState`, чтобы текущие renderer-ы продолжили получать один объект состояния.

## Модель элемента панели

Текущий `FilePanelItem` - sealed class, а не record. Расширять его в том же стиле:

```csharp
public sealed class FilePanelItem
{
    public required string Name { get; init; }
    public required string FullPath { get; init; }
    public required bool IsDirectory { get; init; }
    public long? Size { get; init; }
    public DateTime LastWriteTime { get; init; }
    public FileAttributes Attributes { get; init; }
    public bool IsParentDirectory { get; init; }

    public bool IsVolumeMountPoint { get; init; }
    public string? MountedVolumeName { get; init; }
    public string? MountedVolumePath { get; init; }

```

Инварианты:

1. Для обычных директорий `IsDirectory = true` и `Attributes` содержит `FileAttributes.Directory`.
2. Для `..` `IsDirectory = true`, `IsParentDirectory = true`, `Attributes = FileAttributes.Directory`.
3. `..` никогда не выбирается.
4. Если `DetectVolumeMountPoints = false`, `IsVolumeMountPoint = false`, `MountedVolumeName = null`, `MountedVolumePath = null`.

## Summary

Добавить модель:

```csharp
public sealed class PanelSummary
{
    public int VisibleItemCount { get; init; }
    public int FileCount { get; init; }
    public int DirectoryCount { get; init; }
    public long TotalFileSize { get; init; }

    public int SelectedCount { get; init; }
    public long SelectedFileSize { get; init; }

    public VolumeSpaceInfo? VolumeSpace { get; init; }
    public bool VolumeSpaceUnavailable { get; init; }
}
```

Summary считается по текущему view после фильтрации. Элемент `..` не входит в `FileCount`, `DirectoryCount`, `VisibleItemCount` для пользовательской статистики. Mount point считается директорией.

`TotalFileSize` считать только по файлам, которые прямо отображаются в текущей панели. Рекурсивный обход папок не делать.

`SelectedFileSize` считать только по выбранным файлам. Выбранные директории входят в `SelectedCount`, но не добавляют размер.

## Visibility: Show hidden and system files

Если `ShowHiddenAndSystemFiles = true`, элементы с `FileAttributes.Hidden` или `FileAttributes.System` отображаются.

Если `ShowHiddenAndSystemFiles = false`, элементы скрываются, когда выполняется хотя бы одно условие:

```csharp
(item.Attributes & FileAttributes.Hidden) != 0
|| (item.Attributes & FileAttributes.System) != 0
```

Требования:

1. Фильтровать файлы и директории.
2. Фильтрация выполняется после чтения каталога и заполнения `Attributes`, но до сортировки, summary и render.
3. Скрытые/system элементы, если они показаны, сохраняют свои attributes для highlight service.
4. `..` не скрывать этой настройкой.

## Select folders

Текущий CSharpFar уже выбирает директории. Новая настройка должна сделать это поведение управляемым.

Добавить единый predicate:

```csharp
public static bool CanSelect(FilePanelItem item, AppSettings.PanelOptionsSettings options)
{
    if (item.IsParentDirectory)
        return false;

    if (item.IsDirectory && !options.SelectFolders)
        return false;

    return true;
}
```

Применить к:

1. `PanelController.ToggleSelection`;
2. `PanelController.ToggleSelectAll`;
3. `PanelController.InvertSelection`;
4. будущему выбору по маске;
5. right click selection.

В текущем keymap выбор через `Space` не реализован: пробел является печатным символом и идет в command line. В этом этапе не менять это поведение, если отдельно не решено вводить Far-like Space selection. Требования этой опции покрывают существующие операции выбора и новый right click.

## Sorting: Sort folders by extension

Сортировку вынести из `PanelController` в сервис:

```csharp
public interface IPanelSortService
{
    IReadOnlyList<FilePanelItem> Sort(
        IReadOnlyList<FilePanelItem> items,
        SortMode sortMode,
        bool descending,
        PanelSortOptions options);

    SortDebugInfo ExplainSortKey(
        FilePanelItem item,
        SortMode sortMode,
        PanelSortOptions options);
}

public sealed class PanelSortOptions
{
    public bool SortFoldersByExtension { get; init; } = true;
    public bool KeepParentDirectoryFirst { get; init; } = true;
    public bool DirectoriesFirst { get; init; } = true;
    public StringComparer NameComparer { get; init; } = StringComparer.OrdinalIgnoreCase;
}

public sealed class SortDebugInfo
{
    public required string PrimaryKey { get; init; }
    public required string SecondaryKey { get; init; }
    public required bool IsDirectory { get; init; }
    public required bool IsParentDirectory { get; init; }
}
```

Не добавлять новые enum значения `DateTime` или `Unsorted`; использовать существующий `SortMode.LastWriteTime`.

Правила:

1. `..` всегда остается первым, если он есть.
2. При `DirectoriesFirst = true` директории остаются перед файлами при любой сортировке.
3. При `SortMode.Extension` и `SortFoldersByExtension = true` директории сортируются по `Path.GetExtension(item.Name)`, затем по имени.
4. При `SortMode.Extension` и `SortFoldersByExtension = false` директории сортируются по имени, а файлы - по extension, затем по имени.
5. При равных primary key использовать имя как secondary key.
6. Для descending менять направление primary key. Secondary key по имени оставить ascending, чтобы равные группы не "прыгали" неожиданно при переключении направления.

Пример при `SortMode.Extension`, `SortFoldersByExtension = true`, `DirectoriesFirst = false`:

```text
backup.zip
src.git
docs
images.raw
```

Пример при `DirectoriesFirst = true`:

```text
docs
src.git
backup.zip
images.raw
```

## Status line, total information, free size

Текущий `PanelStatusRenderer` всегда резервирует три строки:

```text
separator
current item row
directory stats row
```

Нужно сделать количество строк зависимым от настроек:

```csharp
public static int StatusRowCount(AppSettings.PanelOptionsSettings options)
{
    if (!options.ShowStatusLine)
        return 0;

    int rows = 2; // separator + current item row

    if (options.ShowFilesTotalInformation || options.ShowFreeSize)
        rows++;

    return rows;
}
```

Требования:

1. `PanelRenderer.VisibleRows` и `BriefTwoColumnsPanelRenderer.RowsPerColumn` используют dynamic row count, а не константу `PanelStatusRenderer.StatusRowCount`.
2. Если `ShowStatusLine = false`, separator и status rows не рисуются, список получает эти строки.
3. Если `ShowStatusLine = true`, строка текущего элемента остается.
4. Если `ShowFilesTotalInformation = true`, summary row содержит текущую статистику по view: `Bytes: <size>, files: <files>, folders: <folders>`.
5. Если есть выбранные элементы, summary row дополнительно показывает `Selected: <count>, <size>`.
6. Если `ShowFreeSize = true`, summary row дополнительно показывает `Free: <size>`.
7. Если `ShowFreeSize = true`, но свободное место получить нельзя, показывать `Free: n/a`.
8. Если `ShowFilesTotalInformation = false`, `PanelStatusRenderer` не считает files/folders/bytes сам. Он использует `PanelSummary`, если summary row нужен из-за `ShowFreeSize`.

Добавить сервис:

```csharp
public interface IVolumeInfoService
{
    VolumeSpaceInfo GetSpaceInfo(string path);
}

public sealed class VolumeSpaceInfo
{
    public required string Path { get; init; }
    public required long FreeBytesAvailable { get; init; }
    public required long TotalBytes { get; init; }
    public string? VolumeLabel { get; init; }
}
```

Windows-реализацию держать в `CSharpFar.FileSystem`. Использовать Win32 `GetDiskFreeSpaceEx` предпочтительно перед `DriveInfo`, потому что он корректнее работает для mount points, UNC и путей внутри смонтированных томов.

Ошибки доступа, недоступный сетевой путь или неготовый носитель не должны ломать render. В этом случае `PanelSummary.VolumeSpaceUnavailable = true`.

5. Если content не scrollable, scrollbar не рисовать при `DrawWhenNotScrollable = false`.
6. Mouse wheel над панелью должен менять `ScrollOffset` с учетом visible rows.
7. Drag scrollbar thumb в этот этап не входит, но hit testing должен отличать scrollbar от file item.

## Parent directory in root

`ShowParentDirectoryInRootFolders = false` соответствует текущему поведению: `..` в root не показывается.

Root folders:

1. Windows drive root: `C:\`, `D:\`.
2. UNC share root: `\\server\share\`.
3. Mounted volume root, если `DetectVolumeMountPoints = true` и текущий путь является корнем mounted volume.

Правила:

1. Если текущий каталог не root, `..` показывается всегда.
2. Если текущий каталог root и `ShowParentDirectoryInRootFolders = false`, `..` не показывается.
3. Если текущий каталог root и `ShowParentDirectoryInRootFolders = true`, `..` показывается как обычный parent item, но Enter на нем не пытается выйти выше root.
4. Для root `..` можно выставить `FullPath = CurrentDirectory`; навигационный код обязан проверить root и сделать no-op.
5. Сообщение `Already at root` допустимо, но не обязательно.

## Mouse input

Текущий `IConsoleDriver` умеет только:

```csharp
ConsoleKeyInfo ReadKey(bool intercept);
```

Добавить unified input event в `CSharpFar.Console/Input`:

## Auto refresh

Добавить watcher в `CSharpFar.Core/Abstractions` и реализацию в `CSharpFar.FileSystem`:

```csharp
public interface IFileSystemChangeWatcher : IDisposable
{
    event EventHandler<FileSystemPanelChanged>? Changed;

    PanelAutoRefreshState StartWatching(PanelWatchRequest request);
    void StopWatching(PanelSide panelSide);
}

public sealed record PanelWatchRequest
{
    public required PanelSide PanelSide { get; init; }
    public required string DirectoryPath { get; init; }
    public required int ObjectCount { get; init; }
    public required bool IsNetworkDrive { get; init; }
    public required AppSettings.PanelAutoRefreshSettings Options { get; init; }
}

public sealed record FileSystemPanelChanged(
    PanelSide PanelSide,
    string DirectoryPath,
    FileSystemChangeKind Kind);

public enum FileSystemChangeKind
{
    Created,
    Deleted,
    Changed,
    Renamed,
    Overflow,
    Unknown
}

public sealed class PanelAutoRefreshState
{
    public bool IsWatching { get; init; }
    public bool DisabledByObjectCount { get; init; }
    public bool DisabledForNetworkDrive { get; init; }
    public string? LastError { get; init; }
}
```

Правила threshold:

```text
DisableIfObjectCountExceeds > 0
and ObjectCount > DisableIfObjectCountExceeds
=> watcher не запускается
```

Network:

```text
IsNetworkDrive
and NetworkDrivesAutoRefresh = false
=> watcher не запускается
```

Windows implementation:

1. Использовать `FileSystemWatcher`.
2. Следить только за текущим каталогом панели: `IncludeSubdirectories = false`.
3. На смене каталога останавливать старый watcher панели.
4. Debounce 250-500 ms.
5. На overflow/error отправлять `FileSystemChangeKind.Overflow` и делать полный refresh.
6. Watcher event не должен напрямую менять `FilePanelState`; он только ставит событие в application queue.

Integration с текущим blocking input loop:

1. Main loop приложения должен уметь просыпаться от watcher event, даже если пользователь не нажимает клавиши.
2. Допустимые варианты: `ReadInput` с timeout/cancellation, thread-safe queue + wait handle, либо общий `ApplicationEvent` loop.
3. После события refresh делать только для панели, чей watcher сработал, и только если `CurrentDirectory` панели совпадает с `DirectoryPath` события.
4. Ручной refresh после операций copy/move/delete/create остается рабочим независимо от `PanelAutoRefreshState`.

## File system location

Добавить сервис:

```csharp
public interface IFileSystemLocationService
{
    FileSystemLocationInfo GetLocationInfo(string path);
}

public sealed class FileSystemLocationInfo
{
    public required string Path { get; init; }
    public required bool IsNetworkDrive { get; init; }
    public required bool IsRemovableDrive { get; init; }
    public required bool IsFixedDrive { get; init; }
    public required string? RootPath { get; init; }
}
```

Windows:

1. UNC paths `\\server\share\...` считать network.
2. Mapped network drives определять через `DriveInfo.DriveType == DriveType.Network`, но не читать expensive properties.
3. Fixed/removable определять через `DriveInfo.DriveType`.
4. Ошибки при определении location не должны ломать панель; возвращать `Unknown`-like info с false flags и `RootPath = Path.GetPathRoot(path)`.

Этот сервис не заменяет `IVolumeService` из спецификации `005`: `IVolumeService` перечисляет точки входа для диалога выбора диска, а `IFileSystemLocationService` классифицирует текущий путь панели.

## Volume mount points

Добавить сервис:

```csharp
public interface IVolumeMountPointService
{
    VolumeMountPointInfo GetMountPointInfo(string directoryPath);
}

public sealed class VolumeMountPointInfo
{
    public required bool IsVolumeMountPoint { get; init; }
    public string? VolumeName { get; init; }
    public string? VolumePath { get; init; }
}
```

Windows:

1. Проверить, что объект является директорией.
2. Проверить `FileAttributes.ReparsePoint`.
3. Для candidates вызвать `GetVolumeNameForVolumeMountPoint`.
4. При необходимости вызвать `GetVolumePathName`.
5. Symlink и junction не считать mount point автоматически.
6. Ошибки Win32 API трактовать как `IsVolumeMountPoint = false`, если нет более точного результата.

Когда `DetectVolumeMountPoints = true`, `PanelViewBuilder` вызывает сервис для директорий-кандидатов и заполняет:

```csharp
FilePanelItem.IsVolumeMountPoint
FilePanelItem.MountedVolumeName
FilePanelItem.MountedVolumePath
```

Когда `DetectVolumeMountPoints = false`, сервис не вызывается.

Влияние:

1. Mount point считается директорией в summary.
2. Free size для текущего пути должен считаться через `GetDiskFreeSpaceEx(path)`, чтобы путь внутри mounted volume возвращал свободное место этого тома.
3. Будущие recursive copy/delete/search/size операции не должны пересекать mount points молча. В этом этапе достаточно иметь metadata.
4. Highlight service сможет использовать `FileAttributes.ReparsePoint`, если такое правило появится позже.

## Acceptance criteria

1. Все выбранные опции представлены в `AppSettings.Panels`, кроме `Highlight files`, который остается в `Panels.FileHighlighting.Enabled`.
2. Runtime больше не использует `Ui.ShowHiddenFiles` / `Ui.ShowSystemFiles` для панели.
3. `FileSystemService` читает каталог без panel-specific filtering и без добавления `..`.
4. Построение view идет через `PanelViewBuilder`.
5. Hidden/system фильтрация работает для файлов и папок.
6. Раскраска включается и выключается через существующий highlight service.
7. Выбор папок управляется `SelectFolders`, а `..` никогда не выбирается.
8. Сортировка вынесена в `IPanelSortService`.
9. `SortFoldersByExtension` покрыт тестами и default сохраняет текущее поведение.
10. Status line можно включать и выключать, а renderer не считает summary сам.
11. Total information считается по текущему view после фильтрации.
12. Free size показывается через отдельный path-aware `IVolumeInfoService`.
13. Scrollbar реализован общим renderer-компонентом и не зашит только в файловую панель.
14. Sort mode letter управляется настройкой и использует текущий `SortModeIndicator`.
15. `..` в root управляется настройкой и Enter на нем не выходит за root.
16. Windows mouse input работает через console input API.
17. Right click selection и mouse wheel работают через hit testing.
18. Auto refresh работает через `FileSystemWatcher`, debounce и event/input loop.
19. Auto refresh отключается по object count threshold и по network drive policy.
20. Volume mount points определяются отдельным сервисом и попадают в `FilePanelItem`.
21. Renderer панели не содержит бизнес-логики фильтрации, сортировки, summary, mount point detection или auto refresh decisions.
