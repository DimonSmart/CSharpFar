# 0059 - Panel cursor normalization

Related:
- 0012
- 0030
- 0034

## Goal

Keep file panel cursor and scroll state valid after panel item lists are
refreshed, rebuilt, or sorted.

## Context

After the current file or directory disappears, `FilePanelState.CursorIndex` can
continue pointing past the end of `Items`. The renderer then has no row to
highlight, so the cursor appears to vanish. Deletion is only one source of this
state: refresh, watcher updates, external programs, virtual/search panel rebuilds
and sorting can all replace the item list.

The invariant belongs in panel control logic because cursor visibility depends
on the visible row count. `FilePanelState` should remain a state holder and the
renderer should only render the state it receives.

## Scope

1. Add a single `PanelController.NormalizeCursor` method that clamps
   `CursorIndex`, clamps or resets `ScrollOffset`, and keeps the cursor visible.
2. Use a shared restore helper after directory refresh and panel sorting.
3. Prefer restoring by item `Location`, then `FullPath`, then `Name`; if the old
   item disappeared, keep the previous numeric index and clamp it.
4. Normalize virtual and search result panels after sorting or refresh.
5. Add focused unit tests for refresh, sort, and direct normalization behavior.

## Non-goals

1. Do not change `DeleteCommand` or route this through
   `RefreshPanelsAfterFileOperation`.
2. Do not change selection pruning semantics.
3. Do not change `LoadDirectory` or `LoadLocation` cursor reset behavior when
   entering a new location.
4. Do not change panel load error handling, provider capabilities, or search
   results as a panel type.
5. Do not add correction logic to the renderer or to `FilePanelState` setters.

## Acceptance criteria

- Removing the last visible item leaves the cursor visible on the previous item.
- Removing an item in the middle leaves the cursor on the next item.
- If only `..` remains, cursor index is `0`.
- If the panel becomes empty, `CursorIndex` and `ScrollOffset` are both `0`.
- Refresh and sort do not leave `CursorIndex` outside `Items`.
- Sorting preserves the current item when it still exists.
- Search result and virtual panel sorting/refresh normalize cursor and scroll.
- Existing selection pruning behavior remains unchanged.
