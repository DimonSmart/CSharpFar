# 0004 - Column navigation, Ctrl+O command line, and resize behavior

## Context

Multi-column panels, hidden-command-line mode, shell command output, and console resize
must behave predictably on the main screen.

## Scope / Behavior

Parent navigation:

- `Backspace` and `Enter` on `..` both navigate to the parent directory.
- Before leaving a directory, the panel remembers the child directory name.
- After the parent directory loads, the cursor is placed on that child
  directory when it exists.
- The scroll offset is adjusted so the restored cursor item is visible.
- If the child item is not found, the panel remains in a normal loaded state.

Column navigation:

- In `BriefTwoColumns`, `Left` moves to the item in the visual column to the
  left while preserving the visual row when that item exists.
- If `Left` is pressed in the leftmost column, the cursor moves to the first
  item.
- In `BriefTwoColumns`, `Right` moves to the item in the visual column to the
  right while preserving the visual row when that item exists.
- If there is no item to the right on the same row, or the cursor is already in
  the last column, `Right` moves to the last item.
- After left/right movement, the panel scrolls to keep the cursor visible.
- In one-column modes, `Left` moves to the first item and `Right` moves to the
  last item.

Hidden command line:

- `Ctrl+O` switches from `Panels` to `HiddenCommandLine` and leaves the command
  line visible in its usual row.
- In `HiddenCommandLine`, the user can edit command-line text.
- `Left`, `Right`, `Home`, `End`, `Backspace`, `Delete`, `Esc`, and `Enter`
  operate on the command line.
- `Enter` runs the entered command.
- `Ctrl+O` restores panels without losing command-line text.
- Long command text scrolls horizontally so the command cursor remains visible.

Command-line cursor:

- After every command-line edit, the visible terminal cursor matches the
  logical command cursor position in both `Panels` and `HiddenCommandLine`.
- On Unix-like ANSI terminals, deleting characters with `Backspace` moves the
  visible cursor left after every deletion, returning it to the initial input
  position when all entered text has been removed.

Shell prompt after command execution:

- Before launching a command, the prompt row is cleared and displays
  `currentDirectory>command`.
- The shell process inherits the console and writes stdout/stderr to the normal
  console buffer.
- After the command exits, the command input row and row below it are cleared.
- The prompt row displays `currentDirectory>` with the cursor after `>`.
- The `Ctrl+O` underlay snapshot is taken after the new prompt is displayed.
- Visible panels render their command line over the same prompt location.
- `HiddenCommandLine` shows the prompt in that same location.
- In VT-supported Unix-like terminals, visible CSharpFar UI uses the alternate
  screen and shell output uses the main screen. `Ctrl+O` switches between these
  screens: hiding all panels leaves the alternate screen, shows the shell main
  screen, resets attributes, and shows the cursor; restoring panels enters the
  alternate screen and repaints the full UI from the application-owned buffer.
- Shell output is not part of the UI buffer and is not reconstructed by reading
  terminal screen contents.

Resize:

- A renderer frame is bound to the size captured at `BeginFrame()`.
- Writes inside a frame use the captured size rather than rereading console size
  for each write.
- Renderer buffers are recreated when size changes, and the next frame is full.
- The application coalesces consecutive resize events before redrawing.
- In VT-supported Unix-like terminals, resize is surfaced as input or an
  equivalent backend event. The application resizes the logical UI surface and
  performs a full repaint in the alternate screen.
- In hidden-command-line mode, resize preserves previously captured shell output in
  the area shared by the old underlay snapshot and the new viewport, then
  redraws the command line.
- If a hidden-command-line command-line frame is interrupted by another viewport
  change, each retry restores the shell underlay before drawing at the command
  line row for that attempt.
- After hidden-command-line resize stabilizes, exactly one command line is visible at
  the row for the final viewport, and its cursor is positioned on that row.
- If the resized viewport is smaller than the captured underlay, shell output
  may be clipped horizontally or vertically.
- If the resized viewport is larger than the captured underlay, cells outside
  the captured underlay may be blank, but the preserved portion of shell output
  must not be discarded merely because the viewport size changed.

Terminal diagnostics:

- The Options menu exposes a terminal diagnostics command.
- The command writes a readable diagnostics block to normal main-screen console
  output rather than displaying only a modal UI message.
- Diagnostics include local timestamp, runtime and process information, current
  directory, viewport and console dimensions, cursor position, panel state and
  layout rows, and terminal screen mode support and active-state information.
- Printing diagnostics preserves a consistent terminal state in both
  terminal-screen and legacy modes and leaves its output available as shell
  underlay in `HiddenCommandLine`.

## Non-goals

- This spec does not define additional panel columns beyond `BriefTwoColumns`.
- This spec does not require shell output virtualization.

## Acceptance criteria

- `Enter` on `..` and `Backspace` return to the parent with the cursor on the
  child directory when possible.
- `Left` and `Right` navigate columns and edge cases predictably.
- In `Ctrl+O`, the command line is visible, editable, and executable.
- Command-line editing keeps the visible terminal cursor synchronized with the
  logical cursor after every edit, including repeated `Backspace` on Unix-like
  ANSI terminals.
- After running a command such as `dir`, the prompt is visible in the stable
  bottom command row and is visible again through `Ctrl+O`.
- Resize redraws panels as a coherent full frame and leaves no artifacts from
  intermediate sizes.
- Repeated vertical resize in `HiddenCommandLine` leaves one command line at
  the final row, with no stale command-line copies above or below it, and
  command-line editing continues normally afterward.
- After running a command and resizing the console, `Ctrl+O` still reveals the
  command output portion that intersects the resized viewport; clipping is
  acceptable, clearing that output is not.
- On Unix-like VT terminals, repeated `Ctrl+O` switches preserve shell output in
  the main screen and repaint UI from the logical buffer when returning to the
  alternate screen.
- `Options -> Terminal diagnostics` invokes a registered command that prints
  runtime, console, panel/layout, and terminal screen mode information to the
  main console without crashing in terminal-screen or legacy mode.

## Verification

Manual checks:

1. Hide both panels, resize the console vertically up and down, and confirm only
   one command line remains at the final row and can still be edited.
2. Show panels and confirm normal resize and rendering behavior.
3. Run `Options -> Terminal diagnostics` in terminal-screen and legacy modes and
   confirm the block remains available in main-screen console output.
4. On a Unix-like terminal, type `abc` and press `Backspace` three times in
   `Panels`, then in `HiddenCommandLine`, and repeat after one resize. Confirm
   the cursor moves left each time and returns to the initial input position.
