# 0028 - Panel and hidden command-line workspace modes

Related:

- 0014
- 0022

## Goal

CSharpFar has two mutually exclusive application workspace modes:

- `Panels`, the ordinary dual-panel file-manager workspace.
- `HiddenCommandLine`, the shell-output workspace with the command line.

`Ctrl+O` is the single panel-workspace visibility shortcut. It switches between
the ordinary dual-panel workspace and the hidden command-line workspace.

## Context

The ordinary file-manager workspace is a two-panel surface. Each panel owns its
own complete frame, and the two panel sides are present together. A state where
only the left side or only the right side is hidden is not a supported
workspace.

The hidden command-line workspace is not a partial panel layout. It removes the
panel workspace entirely so shell output and the command line can use the main
screen behavior defined by `0022`.

## Scope

- Keep one application workspace state with exactly `Panels` and
  `HiddenCommandLine` values.
- In `Panels`, render both left and right panel sides with complete independent
  frames.
- In `HiddenCommandLine`, render no panel side; the command line is the
  application target over shell output.
- Keep `Ctrl+O` as the only shortcut that changes application workspace mode.
- `Ctrl+O` dispatches through the registered `TogglePanels` command so the top
  menu item and keyboard shortcut use the same command path.
- `Ctrl+F1` and `Ctrl+F2` are not panel-visibility shortcuts and are not
  reserved for independent panel visibility.
- Do not support independent hiding or showing of the left or right panel side.
- Quick View in `Panels` replaces the passive half of the workspace with the
  preview while the active half remains the active panel.
- Keep the `Swap panels` command. It exchanges the left and right panel contents
  and swaps their view modes. The active physical side remains unchanged, so if
  the left side was active before the swap, the left side remains active after
  the swap while showing the former right panel contents.
- Keep hidden command-line console scrollback behavior from spec `0022`.

## Non-goals

- Do not add a full-width single-panel mode.
- Do not move one panel to the full workspace width.
- Do not add a destination-panel UX for a single-panel workspace.
- Do not add configurable key bindings.
- Do not change file operation semantics between active and passive panels.
- Do not add compatibility wrappers for older panel-visibility state.

## Acceptance criteria

- The initial application workspace mode is `Panels`.
- `Ctrl+O` from `Panels` enters `HiddenCommandLine`.
- `Ctrl+O` from `HiddenCommandLine` returns to `Panels`.
- The active physical side is preserved across a hide/show cycle.
- `Panels` always contains both left and right panel sides.
- `Tab` in `Panels` switches the active physical side between left and right.
- Quick View in `Panels` renders `active panel | preview`.
- `HiddenCommandLine` contains no panel sides, panel hit targets, directory
  shortcut bar, or function-key bar.
- `Ctrl+F1` and `Ctrl+F2` do not change application workspace mode.
- `Swap panels` exchanges the left and right panel states without changing the
  active physical side.
- `Swap panels` exchanges `LeftViewMode` and `RightViewMode`.

## Verification

Manual and automated verification cover:

- Starting CSharpFar and observing the initial dual-panel workspace.
- Switching `Panels -> HiddenCommandLine -> Panels` with `Ctrl+O`.
- Inspecting shell scrollback and editing the command line in
  `HiddenCommandLine`.
- Returning to `Panels` with the same active side.
- Rendering both panel sides in `Panels`.
- Switching the active side with `Tab`.
- Rendering Quick View as the active panel and preview.
- Confirming that `Ctrl+F1` and `Ctrl+F2` do not toggle panel visibility.
- Confirming help text documents `Ctrl+O` as a switch between the dual-panel
  workspace and shell output with command line.
