# 0034 - Panel quick search

## Goal

Add Far-like quick search for the active file panel.

When the main file manager UI is active, pressing `Alt` plus the first filename
character starts a small `Search` input overlay on the active panel and moves the
panel cursor to the first visible item whose name starts with the typed prefix.
Further filename characters refine the prefix without requiring `Alt`.

## Context

Far Manager has a fast panel navigation mode that is different from full file
search. It does not enumerate the filesystem, search file contents, or open a
results panel. It only moves the cursor within the active panel's already loaded
items.

CSharpFar already has:

1. full search on `Alt+F7`;
2. command-line text entry and command completion;
3. Far-like command-line shortcuts;
4. multiple panel sources, including local filesystem, search results, and SFTP;
5. sorted `FilePanelState.Items` and cursor movement through `PanelController`.

Panel quick search must fit into the main key routing without breaking existing
plain typing into the command line, `Alt+F7`, command history, or panel source
behavior.

## Scope

Implement quick search as a transient active-panel navigation state.

Activation:

1. Start quick search when the main file manager UI is active and the key is
   `Alt` plus a printable character that can reasonably be part of a panel item
   name.
2. Existing reserved shortcuts keep priority. Do not intercept `Alt+F7`,
   `Alt+F8`, `Alt+1`, `Alt+2`, menu shortcuts, or any already implemented
   command chord.
3. The first version must at least support `Alt+A` through `Alt+Z`.
4. Starting quick search must not add text to the command line.
5. Do not start quick search while a modal dialog, viewer, editor, menu, or
   `HiddenCommandLine` owns input.

Input while active:

1. Printable filename characters append to the search text regardless of
   whether `Alt` remains held or is released after activation. Pressing or
   releasing `Alt` while quick search is active must not reset, replace, or
   otherwise change the accumulated search text.
2. `Backspace` removes one character. If the search text becomes empty, close
   quick search and leave the cursor where it is.
3. `Esc` closes quick search and does not run another action.
4. Any other control, navigation, function, or non-filename key closes quick
   search first. The key then continues through normal main UI handling unless
   implementation proves this is unsafe for a specific key.
5. Close quick search when the active panel changes, the application enters
   `HiddenCommandLine`, the panel is reloaded/refreshed/sorted, or another UI
   mode takes input ownership.

Matching:

1. Match against `FilePanelItem.Name`.
2. Use prefix matching: `Name.StartsWith(searchText, comparison)`.
3. Use case-insensitive matching consistent with existing panel name behavior.
4. Search the active panel's current `Items` list in its current displayed order.
5. Move to the first matching item in that order, not the next item after the
   current cursor.
6. Match files and directories.
7. Do not match the synthetic parent directory item `..`.
8. Do not search full paths, file contents, masks, wildcards, or hidden
   filesystem entries that are not already present in `Items`.
9. When the current prefix has no match, keep the search text visible and keep
   the cursor on the last matched item. Do not show a modal error.
10. Cursor movement must update scroll offset so the matched item is visible.
11. Quick search must work for any loaded panel source by using `FilePanelState`
    items only. Do not require local filesystem access.

Overlay:

1. Render a small framed input overlay titled `Search`.
2. Anchor it near the bottom of the active panel, above the command line and
   function-key bar area.
3. The overlay contains only the typed text. No buttons, result list, help text,
   or full search fields.
4. Use existing frame, palette, and rendering conventions.
5. Keep the full search text in state even if the visible input area needs to
   horizontally scroll or show only the tail of long text.
6. Redraw the active panel cursor and overlay immediately after every accepted
   quick-search character.

Preferred implementation shape:

1. Keep quick search state in the main application input layer, not in a blocking
   modal dialog loop.
2. Use explicit names such as `PanelQuickSearchState` and
   `TryFindFirstQuickSearchMatch`.
3. Put pure prefix matching in a small testable helper close to panel navigation.
4. Keep rendering separate from matching and input state.
5. Avoid adding compatibility wrappers or a new search service. This feature is
   not related to the `Alt+F7` search engine.

## Non-goals

Do not implement in this spec:

1. changes to `Alt+F7` full search;
2. filesystem enumeration or recursive search;
3. content search;
4. wildcard, mask, fuzzy, substring, or regular-expression matching;
5. filtering the panel item list;
6. a results list or virtual search panel;
7. cycling through all matches with repeated shortcuts;
8. search history;
9. configurable quick-search shortcuts;
10. source-specific remote queries for SFTP or other providers;
11. persistent settings;
12. localization of the `Search` overlay title.

## Acceptance criteria

- Pressing `Alt+G` on a panel containing `GEMINI.md` starts quick search, shows a
  small `Search` overlay with `g`, and moves the cursor to the first item whose
  name starts with `g`.
- Typing `e`, `m`, `i`, `n` after releasing `Alt` updates the overlay text and
  moves the cursor to the first item matching the longer prefix.
- Typing additional filename letters while continuing to hold `Alt` appends
  them to the existing prefix instead of restarting quick search from each
  letter; releasing or pressing `Alt` between letters has the same result.
- Matching is case-insensitive.
- Matches are chosen from the current panel order.
- Files and directories can both be matched.
- The synthetic `..` item is not matched.
- If the typed prefix has no match, the overlay remains open and the cursor stays
  on the last matched item.
- `Backspace` shortens the prefix and updates the cursor to the first match for
  the shorter prefix.
- `Esc` closes the overlay without changing command-line text or running another
  action.
- `Enter`, arrow keys, page keys, and function keys close the overlay before
  normal main UI handling continues.
- Existing `Alt+F7` search, `Alt+F8` history, `Alt+1`/`Alt+2` view mode changes,
  command-line completion, and ordinary command-line typing keep working.
- Quick search works on loaded local filesystem, search results, and SFTP panels
  because it uses the active panel's existing items.
- Quick search state closes when the active panel is switched, refreshed, sorted,
  reloaded, or hidden.

## Verification

Manual checks:

1. Open a directory with several names sharing the same first letter, press
   `Alt` plus that letter, and verify the first matching item in panel order is
   selected immediately.
2. Keep `Alt` held, type more letters, release `Alt`, type another letter, and
   verify every letter is appended to one prefix and the cursor moves as the
   prefix narrows.
3. Type a non-matching character and verify the overlay stays open without a
   modal error.
4. Press `Backspace` and verify matching recovers for the shorter prefix.
5. Press `Esc` and verify only the overlay closes.
6. Press `Enter`, an arrow key, `F3`, and `F10` from quick search and verify the
   overlay closes before normal key behavior.
7. Verify plain typing still goes to the command line when quick search is not
   active.
8. Verify `Alt+F7`, `Alt+F8`, `Alt+1`, and `Alt+2` still run their existing
   commands.
9. Repeat quick search on a search results panel and an SFTP panel if those
   panels are available.
