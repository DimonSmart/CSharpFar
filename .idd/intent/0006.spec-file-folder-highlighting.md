# 0006 - File and folder highlighting

## Context

File panels need configurable visual highlighting so users can distinguish
directories, file types, masks, attributes, and special entries while keeping
renderer behavior predictable.

## Scope / Behavior

- Highlighting is rule-based and evaluates panel items against masks,
  attributes, and item kind.
- Rules can assign foreground and background colors or named palette roles.
- Directory and file matching are explicit; a rule that is intended only for
  files does not unexpectedly match directories.
- The parent directory item `..` has predictable handling and is not treated as
  an ordinary directory when a rule says otherwise.
- File masks support common wildcard patterns and grouped masks.
- Rule ordering is deterministic. When several rules match, the selected rule is
  chosen by documented priority rather than dictionary or filesystem order.
- Highlighting integrates with panel rendering without changing selection,
  cursor, or status calculations.
- Users can enable or disable file highlighting through panel options.
- A Far-like default rule set exists, but renderer logic does not depend on a
  specific preset.
- Settings persist user-visible highlighting configuration.

## Non-goals

- This spec does not require a full Far Manager color-rule editor.
- This spec does not require importing Far Manager configuration files.
- This spec does not require syntax highlighting for file contents.

## Acceptance criteria

- Directories, normal files, and mask-matched files can be colored differently.
- Rule priority is deterministic and covered by tests.
- Toggling highlighting changes panel colors without changing panel item order
  or selection.
- The default highlighting preset is loaded through the same rule model as
  custom rules.
- Invalid or unsupported masks are handled explicitly and do not crash panel
  rendering.
