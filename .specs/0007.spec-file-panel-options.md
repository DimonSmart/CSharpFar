# 0007 - File panel options

Type: Spec
Status: Current

## Context

File panels expose Far-like display and behavior options. These options affect
which items are visible, how they are sorted, what summary information appears,
and how mouse and refresh behavior work.

## Scope / Behavior

- The panel can show or hide hidden and system files.
- File highlighting can be enabled or disabled.
- Folder selection behavior is configurable so selection commands either include
  or exclude folders according to the option.
- Folder sorting can either group folders in the default way or sort folders by
  extension when that option is enabled.
- Panels can show a status line, total information, and free-space information.
- Panels can show a scrollbar when item count exceeds visible rows.
- The current sort mode is represented by a visible sort-mode letter where the
  display mode supports it.
- A parent directory entry can be shown at filesystem root when the option
  requires it, but it must not navigate above the real root.
- Mouse input supports hit testing for panel rows, scrollbar areas, and item
  cells.
- Right-click selection behavior is configurable.
- Auto refresh updates panel contents after filesystem changes without losing
  cursor intent where possible.
- File-system location services expose current root and volume metadata used by
  panels and drive selection.
- Volume mount points are represented as navigable locations when supported.
- Settings persist these options across restarts.

## Non-goals

- This spec does not require a complete clone of every Far Manager panel option.
- This spec does not require platform-neutral volume metadata beyond what the
  active platform backend can provide.
- This spec does not require mouse operation to replace keyboard operation.

## Acceptance criteria

- Changing panel options affects both rendering and panel behavior
  consistently.
- Hidden/system visibility, highlighting, folder selection, folder extension
  sorting, status rows, scrollbars, sort letters, root parent item behavior,
  mouse hit testing, right-click selection, and auto refresh have focused
  coverage.
- Settings persistence restores panel options after restart.
- Panel options do not corrupt cursor, selection, or scroll state when item
  visibility changes.
