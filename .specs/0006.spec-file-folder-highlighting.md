# 006. Раскраска файлов и папок

Source: `.worklog/0006.spec-file-folder-highlighting.md`.

## Цель

Добавить в CSharpFar Far-like раскраску файлов и папок по правилам.

Раскраска не должна быть набором проверок расширений в renderer. Нужны отдельные:

1. модель правил;
2. matcher Far-like масок;
3. сервис сопоставления файла с правилами;
4. preset `FarDefault`;
5. интеграция renderer-ов с этим сервисом;
6. настройка включения/выключения.

Первая версия не обязана быть побайтно совместима с Far Manager, но все места, где поведение взято из Far или пока только приближено к Far, должны быть явно отделены.

## Что исправлено относительно исходного черновика

1. В текущем CSharpFar уже есть `FilePanelItem.Attributes: FileAttributes`; отдельный `FileHighlightAttributes` дублировал бы `System.IO.FileAttributes` и усложнял модель без пользы.
2. В Far исходниках текущие предопределенные группы масок отличаются от коротких примеров из справки: `arc` намного шире, `temp` равен `*.bak,*.tmp`, `exec` содержит `%PATHEXT%`.
3. Текущий Far default highlighting создается как 6 правил: hidden, system, directory, exec, arc, temp. Отдельного правила для `..` и финального directory fallback в текущем source default нет.
4. В Far цвет "black on black" означает наследование цвета панели. Поэтому модель правила не должна хранить только жесткую пару foreground/background; нужен цветовой override с nullable-компонентами.
5. Поле `IgnoreMask` из старых Far export-файлов в доменной модели лучше заменить положительным `UseMask`, чтобы имя свойства соответствовало поведению.
6. Маски Far обрабатывают `*.*` как `*`, а некоторые ошибки с `|` в русской справке описаны как tolerant parsing, а не обязательное исключение.
7. `.btm` не является явной частью текущего Far `exec`; файл `.btm` подсвечивается как executable только если он попал через `%PATHEXT%` или пользовательскую группу.

## Источники поведения

Проверенные источники на 2026-05-07:

1. Справка Far: раскраска группируется по маскам и атрибутам; группы анализируются сверху вниз; после первого совпадения следующие группы не проверяются, если не включено продолжение обработки.
2. Справка Far: маски поддерживают `*`, `?`, диапазоны `[a-z]`, списки через `,` или `;`, исключения после `|`, regex `/.../flags` и группы `<name>`.
3. Справка Far: группы масок могут ссылаться друг на друга; предопределенные группы `arc`, `temp`, `exec` можно восстановить.
4. Исходники Far `far/config.cpp`: текущие default mask groups.
5. Исходники Far `far/hilight.cpp`: текущие default highlight rules и семантика `black on black` как наследования.
6. Исходники Far `far/filemasks.cpp`: фактический parser масок, расширение групп, `%PATHEXT%`, нормализация `*.*` в `*`.
7. Microsoft Learn: `System.IO.FileAttributes`.
8. Microsoft Learn / NuGet: `Microsoft.Extensions.FileSystemGlobbing` годится для экспериментов, но не покрывает Far-семантику без адаптера.

Ссылки приведены в конце документа.

## Термины

```text
File highlight rule   правило раскраски одного класса файлов
Mask group            именованная группа Far-like масок, например arc или exec
Mask expression       строка масок Far-like синтаксиса
Row state             состояние строки: normal, selected, cursor, selectedCursor
Color override        частичное или полное переопределение цвета базовой строки
Base row color        цвет, который renderer уже выбрал бы без file highlighting
```

## Модель данных

Модели добавить в `CSharpFar.Core/Highlighting`.

### FileHighlightRule

```csharp
public sealed record FileHighlightRule
{
    public required string Id { get; init; }
    public required string DisplayName { get; init; }

    public bool Enabled { get; init; } = true;
    public int Order { get; init; }

    public bool UseMask { get; init; } = true;
    public string MaskExpression { get; init; } = "*";

    public FileAttributes RequiredAttributes { get; init; }
    public FileAttributes ForbiddenAttributes { get; init; }

    public bool ContinueProcessing { get; init; }
    public string? MarkText { get; init; }

    public required FileHighlightColors Colors { get; init; }
}
```

Правило наименьшего удивления для имен:

1. `DisplayName`, а не `Name`, потому что `Id` является стабильным именем правила, а это текст для UI.
2. `UseMask`, а не `IgnoreMask`, потому что положительное условие проще читать.
3. `RequiredAttributes` и `ForbiddenAttributes`, а не Far-specific `IncludeAttributes` / `ExcludeAttributes`, потому что это прямо описывает predicate.

При импорте Far export или fixture адаптер должен понимать старые имена `IgnoreMask`, `IncludeAttributes`, `ExcludeAttributes`, `UseMask`, `AttrSet`, `AttrClear`.

### FileHighlightColor

```csharp
public sealed record FileHighlightColor(int? Foreground, int? Background);
```

`null` в компоненте означает "унаследовать компонент из базового цвета строки".

Примеры:

```csharp

new FileHighlightColor(null, 4)  // только background Red
new FileHighlightColor(10, 1)    // полная пара LightGreen on Blue
```

В Far legacy-экспортах `black on black` трактовать как `FileHighlightColor(null, null)`, а не как реальный черный текст на черном фоне.

## Использование `FilePanelItem`

Текущая модель уже подходит:

```csharp
public sealed class FilePanelItem
{
    public required string Name { get; init; }
    public required string FullPath { get; init; }
    public required bool IsDirectory { get; init; }
    public long? Size { get; init; }
    public DateTime LastWriteTime { get; init; }
    public FileAttributes Attributes { get; init; }
    public bool IsParentDirectory { get; init; }
}
```

Требования:

1. `FileSystemService` должен заполнять `Attributes` для всех элементов.
2. Для директорий `Attributes` должен содержать `FileAttributes.Directory`.
3. Для элемента `..` тоже нужно выставить `Attributes = FileAttributes.Directory`.
4. Инвариант: `item.IsDirectory == item.Attributes.HasFlag(FileAttributes.Directory)` для обычных элементов.
5. Сопоставление масок идет по `item.Name`, не по `FullPath`.
6. Скрытые и системные файлы, если они показаны в панели, должны сохранять свои атрибуты для highlight service.

Отдельный enum `FileHighlightAttributes` не добавлять в первой версии. Если позже понадобятся атрибуты вне `System.IO.FileAttributes`, тогда добавить отдельную модель и adapter, но не раньше.

## Attribute matching

```csharp
static bool AttributesMatch(
    FileAttributes itemAttributes,
    FileAttributes required,
    FileAttributes forbidden)
{
    return (required == 0 || (itemAttributes & required) == required)
        && (forbidden == 0 || (itemAttributes & forbidden) == 0);
}
```

`FileAttributes.Normal` учитывать как обычный флаг .NET, но помнить, что он валиден только когда используется один. Не добавлять специальную поддержку `Normal` до появления правила, которому это нужно.

## Движок раскраски

Интерфейс:

```csharp
public interface IFileHighlightService
{
    HighlightResult GetHighlight(FilePanelItem item, FileRowState state);
}
```

Алгоритм:

```text
result = empty

for each rule ordered by Order, then Id:
    if !rule.Enabled:
        continue

    if !AttributesMatch(item.Attributes, rule.RequiredAttributes, rule.ForbiddenAttributes):
        continue

    if rule.UseMask:
        if !MaskMatcher.IsMatch(rule.MaskExpression, item.Name, groups):
            continue

    stateColor = rule.Colors[state]
    merge stateColor into result.ColorOverride
    merge rule.MarkText into result.MarkText if MarkText is not null
    append rule.Id to result.MatchedRuleIds

    if !rule.ContinueProcessing:
        break

return result
```

Для `FarDefault` все правила имеют `ContinueProcessing = false`.

Если правило совпало, но цвет для текущего состояния равен `null`, правило все равно считается совпавшим. В этом случае renderer оставит базовый цвет строки.

## Разрешение цвета в renderer

Renderer сначала выбирает базовый цвет строки, как сейчас:

```text
normal        обычный файл / папка
selected      выбранный файл
cursor        строка под курсором
selectedCursor выбранная строка под курсором
```

Затем применяет `HighlightResult.ColorOverride`:

```csharp
static ConsoleColorPair ResolveColor(
    ConsoleColorPair baseColor,
    FileHighlightColor? highlight)
{
    if (highlight is null)
        return baseColor;

    return new ConsoleColorPair(
        highlight.Foreground ?? baseColor.Foreground,
        highlight.Background ?? baseColor.Background);
}
```

Раскраску применять только к имени файла/папки. Размер, дата, рамки, заголовки, status bar, command line и keybar не перекрашивать file highlighting-ом.

В `Full` renderer придется писать имя и колонку размера разными вызовами `_screen.Write`, потому что сейчас строка имени и размера строится одной строкой.

В `BriefTwoColumnsPanelRenderer` раскрашивать только область имени в соответствующей колонке, разделитель колонок остается цветом рамки/заголовка.

## Маски файлов

Компоненты добавить в `CSharpFar.Core/FileMasks`.

```csharp
public interface IFileMaskMatcher
{
    bool IsMatch(
        string maskExpression,
        string fileName,
        IReadOnlyDictionary<string, MaskGroup> groups);
}
```

### Поддерживаемый синтаксис

| Возможность | Пример |
| --- | --- |
| `*` | `*.cs` |
| `?` | `file?.txt` |
| диапазоны | `[c-f]*.txt` |
| списки | `*.cs,*.csproj;*.sln` |
| quotes в списках | `"file,1.txt",*.cs` |
| exclude | `*.*\|*.bak,*.tmp` |
| группа | `<arc>` |
| вложенная группа | `<archives>` где внутри `<arc>,*.7zip` |
| группа + exclude | `<arc>\|*.rar` |
| regex | `/^pict\d{1,3}\.gif$/i` |
| `%PATHEXT%` | `*.exe,%PATHEXT%` |

### Семантика

1. Wildcard-маски matching-ятся case-insensitive.
2. Wildcard-маска должна покрывать все имя файла.
3. Regex-маска использует `Regex.IsMatch`, а не принудительное `^...$`; если нужно полное совпадение, пользователь пишет anchors сам.
4. Regex flag `i` включает ignore-case.
5. `*.*` нормализуется в `*`.
6. Пустой include при непустом exclude трактуется как `*`, поэтому `|*.bak` равно `*|*.bak`.
7. Пустой exclude после `|` игнорируется, поэтому `*.*|` работает как `*.*`.
8. Второй `|` не должен ронять приложение. Для первой версии достаточно обработать часть до второго `|` и записать diagnostic в тестируемую ошибку parser-а или debug log.
9. Неизвестная группа разворачивается в пустое выражение и не дает совпадения.
10. Циклические группы не должны приводить к рекурсии; повторно встреченная группа разворачивается в пустое выражение.
11. `%PATHEXT%` разворачивается из environment в список масок `*.ext`. Если переменной нет, использовать пустой список.

Matching нельзя реализовывать через `Contains`, `EndsWith` или ручную проверку расширений. Нужно парсить выражение и компилировать wildcard/regex tokens.

### Compile pipeline

```text
maskExpression
  -> expand groups and %PATHEXT%
  -> split include/exclude by first |
  -> split each side into tokens by comma/semicolon outside quotes and regex
  -> compile wildcard tokens to Regex
  -> compile /regex/flags tokens to Regex
  -> cache compiled expression by expression + group version + PATHEXT snapshot
```

Результат:

```text
match = any(include token matches fileName)
     && no(exclude token matches fileName)
```

## NuGet dependencies

Не добавлять production-зависимость на globbing-пакеты в первой версии.

Причина: `Microsoft.Extensions.FileSystemGlobbing` поддерживает include/exclude и in-memory matching, а начиная с версии 10 умеет ordered filters, но Far-specific слой все равно нужен для:

1. списков через `,` и `;`;
2. `|`;
3. `<groups>`;
4. `/regex/flags`;
5. `%PATHEXT%`;
6. нормализации `*.*`;
7. Far-compatible tolerant parsing.

Допускается отдельный экспериментальный тестовый проект:

```text
tests/CSharpFar.FileMasks.Experiments
```

Он может сравнивать custom matcher, `Microsoft.Extensions.FileSystemGlobbing` и `DotNet.Glob`, но это не часть runtime.

### FarDefault rules

Цвета хранить как foreground override, background наследовать из base row color.

| Order | Id | DisplayName | UseMask | MaskExpression | RequiredAttributes | ForbiddenAttributes | Normal | Cursor | Selected | SelectedCursor |
| ---: | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 10 | `far.hidden` | Hidden files | false | `*` | Hidden | none | fg 3 | fg 8 | default | default |
| 20 | `far.system` | System files | false | `*` | System | none | fg 3 | fg 8 | default | default |
| 30 | `far.directory` | Directories | false | `*` | Directory | none | fg 15 | fg 15 | default | default |
| 40 | `far.exec` | Executable files | true | `<exec>` | none | none | fg 10 | fg 10 | default | default |
| 50 | `far.archive` | Archive files | true | `<arc>` | none | none | fg 13 | fg 13 | default | default |
| 60 | `far.temp` | Temporary files | true | `<temp>` | none | none | fg 6 | fg 6 | default | default |

`default` означает `null`: не переопределять цвет базовой строки.

В этой таблице нет `ForbiddenAttributes = Directory` для `exec`, `archive`, `temp`, потому что FarDefault directory rule идет раньше и матчится по атрибуту с `UseMask = false`. Поэтому папка `test.zip` попадет в `far.directory`, а не в `far.archive`.

Если пользовательские правила меняют порядок или отключают directory rule, пользователь сам должен добавить `ForbiddenAttributes = Directory` в file-only правила.

```csharp
public sealed class FileHighlightingSettings

    public string Mode { get; set; } = "PresetPlusUserRules";
```

JSON:

```json
{
  "panels": {
    "leftViewMode": "Full",
    "rightViewMode": "Full",
    "fileHighlighting": {
      "enabled": true,
      "preset": "FarDefault",
      "mode": "PresetPlusUserRules",
      "rules": [],
      "maskGroups": []
    }
  }
}
```

Modes:

| Mode | Правила | Группы масок |
| --- | --- | --- |
| `PresetOnly` | только preset rules | только preset groups |
| `PresetPlusUserRules` | preset + user rules, общий sort by `Order`, then `Id` | preset groups + user groups, user group replaces same preset name |
| `UserRulesOnly` | только user rules | только user groups |

Если `Enabled = false`, `IFileHighlightService` не вызывается либо возвращает empty result; renderer полностью использует стандартные цвета панели.

В `SettingsDialog` первой версии добавить только checkbox `File highlighting`. Полноценный редактор правил не делать.

## Интеграция

### FileSystemService

1. Заполнять `Attributes` для `..`.
2. Сохранять `FileAttributes.Hidden` и `FileAttributes.System` даже если соответствующие элементы не фильтруются.
3. Не вычислять дополнительные Win32 metadata в первой версии.

### Application composition

Создать один экземпляр `FileHighlightService` на приложение после загрузки settings.

При изменении настройки `Enabled`, `Preset`, `Mode`, `Rules`, `MaskGroups` пересоздавать service или обновлять immutable snapshot.

### PanelRenderer

Поддержать раскраску в:

1. `PanelRenderer.RenderFull`;
2. `BriefTwoColumnsPanelRenderer.RenderCell`.

Renderer должен:

1. вычислить `FileRowState`;
2. вычислить base row color;
3. получить `HighlightResult`;
4. слить `ColorOverride` с base row color;
5. применить итог только к имени.

Не добавлять бизнес-логику масок в renderer.

Важно: не считать, что `Far.exe /export` всегда содержит highlighting. В Far 3 настройки раскраски могут лежать в отдельном `highlight.db`. Verification script должен:

1. скачать или принять путь к чистому Far build;
2. запустить Far с чистым profile path;
3. восстановить предопределенные groups/rules, если требуется;
4. попробовать получить export;
5. если export не содержит `<highlight>`, прочитать `highlight.db` через documented/observed schema;
6. записать fixture;
7. сравнить fixture с `FarDefaultHighlightPreset`.

Если актуальный Far отличается от `FarDefault`, тест должен падать с diff, а spec/preset нужно обновить осознанно.

## Acceptance criteria

1. `.exe`, `.cmd`, `.bat`, `.com` подсвечиваются как executable.
2. Расширения из `%PATHEXT%` подсвечиваются как executable после expansion.
3. Архивы из текущей группы `<arc>` подсвечиваются как archive.
4. `*.bak` и `*.tmp` подсвечиваются как temporary.
5. Папки подсвечиваются отдельно от файлов.
6. Скрытые и системные элементы подсвечиваются по атрибутам.
7. Папка `test.zip` подсвечивается как directory, не как archive.
8. Цвет зависит от row state.
9. Для selected states FarDefault не переопределяет выбранный цвет строки.
10. `panels.fileHighlighting.enabled=false` возвращает стандартные цвета панели.
11. Маска `*.*|*.bak,*.tmp` работает как include + exclude.
12. `<arc>|*.rar` работает корректно.
13. `*.*` матчится как `*`, включая имена без точки.
14. Все FarDefault rules покрыты unit tests.
15. Есть verification task или skipped test с понятным сообщением для сравнения с актуальным Far fixture.
