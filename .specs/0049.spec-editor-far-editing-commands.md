# 0049 - Editor Far editing commands

Type: Spec
Status: Current
Replaces:
- none

Related:
- 0013
- 0046

## Goal

Add the remaining Far-like editor commands from the implementation analysis as
native CSharpFar editor behavior.

The work should extend the `EditorSession` model introduced by `0046`, keep
undo/redo and dirty tracking correct, and make the supported command set easy
to verify from tests. This spec is a follow-up to `0046`; it does not replace
the editor core, the file format model, or the editor-domain event plan
described there.

## Context

`0046` delivered the built-in editor core: file format preservation, an editor
session, undo/redo, word navigation, find/replace, linear and rectangular
selection state, clipboard integration, bookmarks, status rendering, and an
editor key bar.

The comparison report shows that the current implementation still misses a set
of editing commands expected by users familiar with Far Manager. The current
code also differs from some claims in the report:

1. `Ctrl+Ins`, `Shift+Ins`, and `Shift+Del` are not handled by `FileEditor`.
   The editor currently handles `Ctrl+C`, `Ctrl+X`, and `Ctrl+V`.
2. `F8` currently calls `DeleteForward()`. It does not delete the current line
   when there is no selection.
3. There are no editor commands for deleting the current line, deleting to the
   end of the line, deleting the previous or next word, duplicating a block or
   line, changing selection case, numbered bookmarks, or inserting file names.
4. `Ctrl+Up` and `Ctrl+Down` are not handled in the editor.
5. Original Far Manager editor key mappings have priority over generic Windows
   editor conventions when a conflict appears. This means `Ctrl+Y` is delete
   line, `Ctrl+P` is block copy, and redo uses `Ctrl+Shift+Z`.

The implementation should prefer explicit editor command names over hidden
switch-case growth. Each class and method added for this work must follow the
principle of least surprise: command names should say what text or editor state
they mutate.

## Scope

### Command inventory and dispatch

Create one reviewed inventory of editor commands and key chords before changing
behavior. The inventory must distinguish:

1. commands already implemented;
2. missing commands that can be added without key conflicts;
3. missing commands with existing CSharpFar key conflicts;
4. reported Far features that are not editing commands and stay out of scope.

The implementation may keep simple handling in `FileEditor` for small changes,
but repeated command metadata must not drift from rendered labels or tests. If
the switch grows beyond a few related chords, introduce an editor-local command
dispatch helper with explicit command ids.

### Alternate clipboard chords

Add Far/Windows-style clipboard aliases:

1. `Ctrl+Ins` copies the current selection.
2. `Shift+Del` cuts the current selection.
3. `Shift+Ins` pastes clipboard text.

These aliases must use the same clipboard service and selection paths as
`Ctrl+C`, `Ctrl+X`, and `Ctrl+V`.

### Deletion commands

Add session-level text commands, with tests, for:

1. delete selected text;
2. delete the current line when no selection exists;
3. delete from the cursor to the end of the current line;
4. delete the previous word;
5. delete the next word.

`F8` should delete the current selection or current line, not a single
character. Word deletion must use `Editor.WordDiv`, matching word navigation
and whole-word search from `0046`.

Potential Far chords include `Ctrl+Y`, `Ctrl+K`, `Ctrl+Backspace`, `Ctrl+Del`,
and `Alt+D`. Exact binding must be confirmed in the command inventory before
implementation.

Every deletion command must be one undo transaction, must restore cursor and
selection correctly on undo/redo, and must keep line bookmarks predictable when
lines are removed.

### Block and line transforms

Add editor-native commands for:

1. duplicate the selected block when a non-empty selection exists;
2. duplicate the current line when no selection exists;
3. convert the selected block to uppercase;
4. convert the selected block to lowercase;
5. convert case for the current word only if no selection exists and the chosen
   command semantics make that behavior unsurprising.

Candidate chords from the original Far editor mapping include `Ctrl+D` delete
block, `Ctrl+P` copy block to cursor, `Ctrl+M` move block to cursor,
`Alt+U` shift block left, `Alt+I` shift block right, and `Ctrl+U` clear
selection.

Rectangular selections must be handled deliberately. If a command supports
rectangular selection, it should transform only selected columns. If it does
not, it must leave the document unchanged and show clear feedback instead of
falling back to linear behavior.

### Numbered bookmarks

Keep the existing session-local line bookmarks from `0046` and add numbered
bookmark slots `0` through `9`.

The first implementation may keep numbered bookmarks session-local. Persistence
is out of scope. Line movement rules must be the same as ordinary bookmarks:
insertions and deletions before a bookmark move it predictably.

The exact set/go-to chords for numbered bookmarks are TBD because the report
only names `Ctrl+0` through `Ctrl+9`. The implementation must not consume
number chords until the command inventory states whether the chord sets,
toggles, or jumps to a bookmark.

### File name insertion

Add editor commands for inserting file names into text:

1. insert the edited file name or path when the command is editor-local;
2. insert the active panel item name or path when the command needs panel
   context.

Candidate chords from the report are `Shift+Enter` and `Ctrl+Shift+Enter`.
Because `FileEditor` currently receives only the edited file path, panel-based
insertion needs an explicit context object or callback supplied by the
application. Do not read panel state through hidden global dependencies.

### Navigation adjuncts

Add `Ctrl+Up` and `Ctrl+Down` only after their intended editor behavior is
recorded in the command inventory. If they scroll the viewport without moving
the cursor, the implementation must update `EditorViewport` directly and keep
cursor visibility rules predictable.

## Non-goals

Do not implement in this spec:

1. macro recording, macro language, Lua compatibility, or Far macro playback;
2. `F1` help, `F9` menus, `Alt+F9`, or other global shell/menu behavior;
3. printing;
4. syntax highlighting or color-span providers;
5. editor plugin projection;
6. persistent bookmarks;
7. user-configurable key bindings;
8. exact Far Manager compatibility for conflicting chords without an explicit
   project decision;
9. a new plugin, adapter, or compatibility layer around the editor model.

## Acceptance criteria

- `Ctrl+Ins`, `Shift+Del`, and `Shift+Ins` behave like aliases for copy, cut,
  and paste.
- `F8` deletes the selected block, or the current line when there is no
  selection.
- Delete-line, delete-to-end-of-line, delete-previous-word, and delete-next-word
  commands are undoable and redoable.
- Word deletion uses the configured `Editor.WordDiv` separators.
- `Ctrl+D` deletes the selected block.
- `Ctrl+P` copies the selected block to the cursor without using the clipboard.
- `Ctrl+M` moves the selected block to the cursor.
- `Alt+U` and `Alt+I` shift selected lines or the current line left and right.
- `Ctrl+U` clears selection.
- Numbered bookmarks can be set and reached within one editor session once their
  chords are decided.
- Bookmark line positions remain predictable after inserting or deleting lines.
- File-name insertion uses explicit editor/application context and is covered by
  tests.
- `Ctrl+Z` remains undo and `Ctrl+Shift+Z` is redo.
- Existing save, find, replace, selection, clipboard, and ordinary bookmark
  behavior from `0046` keeps passing.

## Verification

Commands:

```bash
dotnet build
dotnet test
```

Focused automated checks:

- `FileEditorTests` for `Ctrl+Ins`, `Shift+Ins`, `Shift+Del`, `F8` selected
  delete, and `F8` current-line delete.
- `EditorSessionTests` for delete-line, delete-to-end-of-line, word deletion,
  Far block copy/move/delete, line shifting, numbered bookmarks, and bookmark
  movement.
- Regression tests for `Ctrl+Z`, `Ctrl+Shift+Z`, `Ctrl+Y`, `Ctrl+B`, `Ctrl+N`,
  `Ctrl+P`,
  `Ctrl+C`, `Ctrl+X`, and `Ctrl+V`.
- Rectangular selection undo tests before adding new rectangular transforms.
- Command inventory tests that fail on duplicate chords or unresolved conflicts.

Manual checks:

1. Open a real text file and exercise the new clipboard aliases.
2. Use `F8` with and without selection and verify undo restores the previous
   text.
3. Delete words across punctuation configured by `Editor.WordDiv`.
4. Copy, move, and delete a selected block using `Ctrl+P`, `Ctrl+M`, and
   `Ctrl+D`.
5. Shift a selected block left and right using `Alt+U` and `Alt+I`.
6. Set numbered bookmarks, edit lines above them, and jump back to the expected
   lines.
7. Insert edited-file and panel-file names after the application context path is
   implemented.
