# 005. Выбор диска / тома

Source: `.worklog/0005.spec-drive-volume-selection.md`.

## Цель

Добавить Far-like диалог выбора диска для активной или указанной панели.

Диалог должен выглядеть и вести себя похоже на окно `Change drive` из Far Manager: поверх панелей открывается компактное модальное окно со списком доступных дисков, типом носителя, общим размером и свободным местом там, где эти данные можно получить быстро. При этом внутренняя модель не должна быть привязана только к Windows drive letters, чтобы позже не мешать реализации для Unix и macOS.

Открытие диалога должно быть быстрым. Нельзя блокировать UI попытками прочитать размер или свободное место сетевых дисков, которые могут быть недоступны.

## Термины

В пользовательском интерфейсе Windows можно использовать слово `drive`, потому что оно привычно для Far-like сценария.

В коде использовать термин `volume`, а не `drive`, если речь идет об абстрактной точке входа в файловую систему:

```text
Volume        логический диск, mount point или другой корень файловой системы
RootPath      путь, в который можно перейти панели: C:\, D:\, /, /mnt/data, /Volumes/Data
DisplayName   короткое имя для списка: C:, D:, /, Data, server/share
Shortcut      быстрая клавиша выбора, если она есть и не конфликтует
```

Это сохраняет ожидаемое поведение на Windows и оставляет место для Unix/macOS, где корни не обязаны быть буквами.

## Вызов

Минимальные горячие клавиши:

```text
Alt+F1  открыть выбор диска для левой панели
Alt+F2  открыть выбор диска для правой панели
```

Если выбранная панель не активна, после успешного перехода она становится активной. Это соответствует принципу наименьшего удивления: команда `Alt+F1` явно относится к левой панели, `Alt+F2` явно относится к правой.

Дополнительно можно позже добавить вызов через верхнее меню, но первая реализация не должна зависеть от меню.

## Внешний вид диалога

Диалог рисуется модально через `ScreenRenderer.BeginFrame()` поверх текущего экрана и после закрытия восстанавливает подложку так же, как существующие диалоги.

Базовый вид для Windows:

```text
 ╔══════════════ Change drive ══════════════╗
 ║Disk              │ Free       │ Total     ║
 ║──────────────────┼────────────┼───────────║
 ║C: fixed          │      656 G │    1,86 T ║
 ║D: fixed          │      415 G │    1,86 T ║
 ║E: fixed          │     21,2 G │     466 G ║
 ║P: network        │            │           ║
 ║Y: disconnected   │            │           ║
 ║Z: network        │            │           ║
 ╚══════════════ Enter  Esc ═════════════════╝
```

The dialog frame has a one-character outer padding inside the modal background so it does not touch the edge of the popup area. The size columns are presented as a small table with short English headers: `Disk`, `Free`, `Total`. `Free` and `Total` are separated by single vertical lines.

Для CSharpFar первая версия может не реализовывать plugin/pseudo entries (`NetBox`, `Process list`, `Temporary panel`). Но layout должен сразу учитывать, что список может содержать не только обычные тома. Поэтому строки списка должны строиться из общей модели элемента выбора, а не из жестко зашитого массива букв `A..Z`.

## Данные в списке

Для каждого обычного тома показывать:

```text
DisplayName  Kind  FreeSize  TotalSize
```

Требования:

1. `DisplayName` для Windows-диска: `C:`, `D:`, `Z:`.
2. `DisplayName` для mount point без буквы: короткое имя mount point или сам путь, если короткого имени нет.
3. `Kind` должен быть понятным пользователю: `fixed`, `removable`, `network`, `cdrom`, `ram`, `unknown`.
4. Если том недоступен, показывать `disconnected` или `not ready`, не пытаться читать размер.
5. `TotalSize` и `FreeSize` показывать только если они доступны быстро и без ошибки.
6. Размеры форматировать в стиле текущей Far-like панели: `659 G`, `21,2 G`, `1,86 T`.
7. Список сортировать по стабильному пользовательскому порядку:
   `C:`, `D:`, `E:` на Windows по букве, затем остальные mount points; на Unix/macOS по `RootPath`.
8. Для сетевых томов отсутствие `TotalSize` и `FreeSize` является нормальным состоянием, а не ошибкой. В UI такая строка выглядит как `P: network` с пустыми колонками размеров.

## Производительность и сетевые тома

Диалог выбора диска является навигационным UI, а не диагностикой сети. Он должен открываться предсказуемо быстро даже при наличии недоступных mapped network drives.

Требования:

1. Первый кадр диалога строится только из дешевой локальной информации: имя, корневой путь, тип тома, shortcut.
2. Синхронное чтение `TotalBytes` и `FreeBytes` разрешено только для томов, для которых это не приводит к сетевому обращению.
3. Для `VolumeKind.Network` при открытии диалога не выполнять синхронные проверки, которые могут обращаться к сети: чтение размера, свободного места, перечисление корня, проверку доступа к UNC.
4. `VolumeKind.Network` без подтвержденной доступности отображается как `network`, без размеров, и может быть выбран пользователем.
5. Если для сетевого диска можно дешево определить локально, что подключение отсутствует, строка отображается как `disconnected`.
6. Если доступность сетевого диска неизвестна, не показывать `disconnected` только на основании отсутствующих размеров.
7. Реальная проверка доступности сетевого диска выполняется при переходе панели в `RootPath`; если переход не удался, показывается обычная ошибка навигации.
8. Любое необязательное обогащение данных тома размером или меткой должно быть отложенным, кэшированным или ограниченным коротким timeout. Оно не должно задерживать открытие диалога.

Практическое правило для первой реализации: как Far Manager на примере выше, локальные fixed-диски могут показывать размеры сразу, сетевые диски показываются без размеров.

## Много дисков

Количество томов не ограничивать шириной или высотой диалога.

Требования:

1. Диалог имеет максимальную высоту, зависящую от размера консоли.
2. Если элементов больше, чем помещается, список прокручивается.
3. `Up` / `Down` двигают курсор на одну строку.
4. `PageUp` / `PageDown` двигают курсор на высоту видимой области.
5. `Home` / `End` переходят к первому / последнему элементу.
6. Активная строка всегда видима после перемещения.
7. При малой высоте консоли диалог должен оставаться управляемым: минимум рамка, одна видимая строка списка и нижняя строка подсказок.
8. При узкой консоли длинные имена и пути обрезаются без выхода за рамку.

Не делать допущение, что на Windows максимум 26 релевантных элементов. Кроме буквенных дисков могут быть mount points, сетевые подключения, WSL/виртуальные файловые системы, subst-диски и будущие псевдопанели.

## Начальная позиция курсора

При открытии диалога курсор позиционируется на томе, чей `RootPath` является корнем текущего каталога целевой панели.

Требования:

1. Для поиска совпадения проверять, является ли `RootPath` тома префиксом `CurrentDirectory` целевой панели (без учёта регистра на Windows).
2. Если несколько томов подходят как префикс, выбирать тот, у которого `RootPath` длиннее — чтобы корректно работать с mount points, вложенными в другие тома.
3. Если совпадение не найдено, курсор стоит на первом элементе списка.
4. Если курсор оказывается не в первой видимой строке, список прокручивается так, чтобы курсор находился примерно в центре видимой области.

Пример: панель показывает `C:\Users\foo` — при открытии диалога курсор стоит на строке `C:`.

## Навигация и выбор

Клавиши внутри диалога:

| Клавиша | Действие |
| --- | --- |
| `Up` / `Down` | переместить курсор |
| `PageUp` / `PageDown` | прокрутить список на страницу |
| `Home` / `End` | первый / последний элемент |
| `Enter` | выбрать текущий том |
| `Esc` / `F10` | закрыть без изменения панели |
| буква/цифра shortcut | выбрать первый видимый или первый полный элемент с таким shortcut |

Поведение shortcut:

1. Для Windows-дисков shortcut равен букве диска: `C`, `D`, `E`.
2. Регистр не важен.
3. Если shortcut уникален, выбор выполняется сразу.
4. Если shortcut не уникален, курсор переходит к следующему элементу с этим shortcut; повторное нажатие циклически перебирает совпадения.
5. Shortcut не должен быть единственным способом выбора, потому что на Unix/macOS у томов может не быть естественной буквы.

## Переход панели

После выбора обычного доступного тома:

1. Целевая панель загружает `RootPath` через `PanelController.LoadDirectory`.
2. `CurrentDirectory` панели становится равен нормализованному `RootPath`.
3. `CursorIndex`, `ScrollOffset` и `SelectedPaths` сбрасываются так же, как при обычном переходе в новый каталог.
4. Путь добавляется в историю каталогов.
5. Экран перерисовывается полностью.

Если целевая панель находилась в quick view, выбор диска применяется к файловой панели, а quick view после этого либо выключается, либо перерисовывается согласно текущей общей логике. Предпочтительно выключить quick view при смене корня, потому что пользователь явно меняет содержимое панели.

## Ошибки

Нельзя падать из-за недоступного диска.

Требования:

1. Ошибки перечисления одного тома не должны ломать весь список.
2. Недоступный том остается в списке, если платформа сообщает о его существовании, но строка помечается как `disconnected` или `not ready`.
3. `Enter` на недоступном томе показывает `MessageDialog` с понятной ошибкой и оставляет диалог открытым.
4. Если том исчез между открытием диалога и выбором, показать ошибку и обновить список.
5. Если список томов пуст, показать модальное сообщение `No volumes found`.

## Интерфейсы

Добавить контракты в `CSharpFar.Core`, а платформенную реализацию держать вне UI-кода.

Рекомендуемая модель:

```csharp
public interface IVolumeService
{
    IReadOnlyList<FileSystemVolume> GetVolumes();
}

public sealed class FileSystemVolume
{
    public required string Id { get; init; }
    public required string DisplayName { get; init; }
    public required string RootPath { get; init; }
    public VolumeKind Kind { get; init; }
    public VolumeStatus Status { get; init; }
    public long? TotalBytes { get; init; }
    public long? FreeBytes { get; init; }
    public string? Shortcut { get; init; }
}

public enum VolumeKind
{
    Unknown,
    Fixed,
    Removable,
    Network,
    CdRom,
    Ram,
    MountPoint,
    Pseudo
}

public enum VolumeStatus
{
    Ready,
    Unchecked,
    NotReady,
    Disconnected,
    Error
}
```

Правила именования:

1. `IVolumeService.GetVolumes()` должен только быстро перечислять точки входа в файловую систему.
2. `FileSystemVolume.RootPath` должен быть путем, пригодным для `IFileSystemService.ReadDirectory`.
3. `FileSystemVolume.DisplayName` должен быть короткой строкой для UI, а не путем для навигации.
4. Не добавлять в `FilePanelState` свойства вроде `DriveLetter`; панели должны продолжать работать с `CurrentDirectory`.
5. Не добавлять Windows-only типы в `CSharpFar.Core`.
6. `VolumeStatus.Unchecked` означает, что доступность специально не проверялась, чтобы не блокировать UI. Для UI это не ошибка: строка отображается по `VolumeKind`, а выбор разрешен.
7. `VolumeStatus.Ready` означает, что доступность проверена дешево или уже известна.

Для будущих псевдопанелей не расширять `FileSystemVolume` случайными nullable-полями. Лучше добавить отдельный тип строки выбора:

```csharp
public sealed class VolumeSelectionItem
{
    public required string Label { get; init; }
    public string? Shortcut { get; init; }
    public FileSystemVolume? Volume { get; init; }
    public VolumeSelectionAction Action { get; init; }
}
```

В первой реализации можно использовать только элементы с `Action = OpenVolume`.

## Windows-реализация

Windows-реализация может использовать `System.IO.DriveInfo.GetDrives()`.

Маппинг:

| `DriveType` | `VolumeKind` |
| --- | --- |
| `Fixed` | `Fixed` |
| `Removable` | `Removable` |
| `Network` | `Network` |
| `CDRom` | `CdRom` |
| `Ram` | `Ram` |
| другое | `Unknown` |

Правила:

1. `RootPath` брать из `DriveInfo.RootDirectory.FullName`.
2. `DisplayName` для `C:\` делать `C:`.
3. `Shortcut` для `C:\` делать `C`.
4. Для `DriveType.Network` не вызывать синхронно `DriveInfo.IsReady`, `DriveInfo.TotalSize` и `DriveInfo.AvailableFreeSpace` при открытии диалога.
5. Для `DriveType.Network` по умолчанию ставить `Status = Unchecked`, `TotalBytes = null`, `FreeBytes = null`.
6. Если локальный Windows API может без сетевого обращения определить, что mapped drive отключен, ставить `Status = Disconnected`.
7. Для `DriveType.Fixed` можно читать `IsReady`, `TotalSize` и `AvailableFreeSpace` синхронно.
8. Для `DriveType.Removable` и `DriveType.CDRom` сначала проверять готовность; размеры читать только если носитель готов. Если проверка может зависнуть или платформа ведет себя нестабильно, лучше оставить размеры пустыми.
9. Любые исключения при чтении размера локального тома переводить в `Status = Error`, но сам том оставлять в списке.
10. Не считать `null` в `TotalBytes` / `FreeBytes` ошибкой для `Network`, `Unchecked`, `NotReady` и `Disconnected`.

Не использовать `DriveInfo` как единственный источник истины, если конкретное свойство может вызвать медленную сетевую проверку. Для быстрого начального списка на Windows допустимо использовать более низкоуровневые локальные API, например перечисление логических дисков и определение типа диска, а проверку доступности сетевого пути оставить на момент выбора.

## Unix/macOS на будущее

В этой задаче Unix/macOS не реализуются, но архитектура не должна закрывать этот путь.

Ожидаемое будущее поведение:

1. Linux показывает `/` и обнаруженные mount points из системного источника, например `/proc/mounts`.
2. macOS показывает `/` и пользовательские тома из `/Volumes`.
3. У томов может не быть shortcut.
4. `RootPath` может быть `/`, `/home`, `/mnt/data`, `/Volumes/Data`.
5. UI не должен требовать двоеточие после имени и не должен считать, что имя занимает ровно два символа.
6. Форматирование списка должно выравнивать колонки по доступной ширине, а не по позиции Windows-буквы.

## Тесты

Минимальные регрессии:

1. `Alt+F1` открывает выбор тома для левой панели и после выбора меняет только левую панель.
2. `Alt+F2` открывает выбор тома для правой панели и после выбора меняет только правую панель.
3. Диалог прокручивает список, если томов больше видимой области.
4. `Enter` на доступном томе вызывает `PanelController.LoadDirectory` с `RootPath`.
5. `Esc` закрывает диалог без изменения панелей.
6. Shortcut `C` выбирает том с shortcut `C`, если он уникален.
7. Недоступный том показывает ошибку и не меняет `CurrentDirectory`.
8. Ошибка чтения размера одного тома не скрывает остальные тома.
9. Unix-like `RootPath = "/"` и `RootPath = "/Volumes/Data"` корректно отображаются без предположения о drive letter.
10. Сетевой том с `Status = Unchecked` отображается как `network`, не показывает размеры и не считается ошибкой.
11. Открытие диалога не вызывает чтение размеров для сетевых томов.
12. Несколько недоступных сетевых дисков не задерживают появление первого кадра диалога.
13. При открытии диалога для панели с текущим каталогом `D:\foo`, курсор установлен на строке с томом `D:`.
14. Если текущий каталог панели не совпадает ни с одним томом, курсор стоит на первом элементе списка.

## Acceptance criteria

1. Есть отдельный `IVolumeService` и модель `FileSystemVolume` без Windows-only типов.
2. В приложении есть модальный Far-like диалог выбора диска / тома.
3. `Alt+F1` работает для левой панели, `Alt+F2` работает для правой.
4. Список поддерживает больше элементов, чем помещается на экран.
5. Доступные Windows-диски показывают букву, тип, общий размер и свободное место.
6. Сетевые Windows-диски отображаются без синхронного чтения размера и свободного места.
7. Недоступные и отключенные диски отображаются без падения приложения.
8. Выбор тома меняет каталог целевой панели на `RootPath`.
9. Диалог не содержит логики `A..Z` как основы модели.
10. Core-слой остается пригодным для будущего Linux/macOS provider.
11. Наличие нескольких недоступных network drives не делает открытие диалога заметно долгим.
12. При открытии диалога курсор стоит на томе текущего каталога целевой панели.
