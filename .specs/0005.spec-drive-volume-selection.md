# 0005 - Drive and volume selection

Type: Spec
Status: Current

## Context

Users need a Far-like way to switch a panel to another drive, volume, or file
system root without typing paths manually.

## Scope / Behavior

- Each panel can open a drive or volume selection dialog.
- The dialog is invoked for the active panel and changes only that panel after a
  successful selection.
- The list includes available local drives and mounted volumes discoverable on
  the current platform.
- The current panel location determines the initial selected list item when a
  matching root exists.
- The dialog shows enough information to distinguish roots, including display
  name, root path, drive type, and available size when that data can be obtained
  without blocking the UI.
- Slow or unavailable network volumes must not freeze the application while the
  list is being built.
- Keyboard navigation supports moving through the list, accepting the selected
  root, and canceling.
- Cancel leaves the panel unchanged.
- If changing to the selected root fails, the panel remains at its previous
  location and the user sees an error.
- Windows implementation uses Windows drive and volume metadata.
- Unix/Linux implementation exposes at least the root file system and any local
  drives discoverable without blocking through the platform volume service. It
  must not construct Windows-only volume services.

## Non-goals

- This spec does not require editing mount points.
- This spec does not require showing every possible remote provider.
- This spec does not require complete Unix mount metadata parity.
- This spec does not require macOS support.

## Acceptance criteria

- The active panel can open a drive/volume list.
- Accepting an accessible root changes the active panel to that root.
- Canceling or failing to open a root leaves the panel unchanged.
- The dialog remains responsive when slow or unavailable volumes exist.
- Initial selection follows the current panel root when possible.
- On Unix/Linux, the volume list includes `/` at minimum and accepting it opens
  the root directory when accessible.
