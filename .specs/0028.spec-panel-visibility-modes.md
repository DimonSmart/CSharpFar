# 0028 - Panel visibility modes

Type: Spec
Status: Current
Replaces:
- none

Related:
- 0013
- 0014
- 0022

## Goal

Let the left and right file panels be hidden independently, while keeping `Ctrl+O` as the
shortcut for hiding or showing both panels at once.

## Context

The regular file panels currently share one center border line. Quick View already renders
as a separate full-frame panel, so the main two-panel layout should follow the same
principle: each panel owns its full frame.

The existing hidden-panel state is a single boolean. That makes it impossible to hide only
one side and makes `Ctrl+O` the only panel visibility mode.

## Scope

- Render the left and right panels as adjacent independent panels with full double-line
  frames.
- Replace the boolean panel visibility state with a flags enum that can represent hidden
  left, hidden right, or hidden both.
- Add `Ctrl+F1` to toggle the left panel and `Ctrl+F2` to toggle the right panel.
- Keep `Ctrl+O` as the both-panels shortcut: if both panels are hidden it shows both;
  otherwise it hides both.
- `Ctrl+O` dispatches through the registered both-panels visibility command so
  the top menu item and keyboard shortcut use the same command path.
- Add a `Swap panels` command that exchanges the left and right panel contents
  and swaps their view modes. The active physical side remains unchanged, so if
  the left side was active before the swap, the left side remains active after
  the swap while showing the former right panel contents.
- When hiding the active panel while the other panel remains visible, move active focus to
  the visible panel.
- Keep hidden-both console scrollback behavior from spec 0022.

## Non-goals

- Do not add configurable key bindings.
- Do not change Quick View rendering beyond respecting hidden panel sides.
- Do not change file operation semantics between active and passive panels.
- Do not add compatibility wrappers for the old boolean visibility field.

## Acceptance criteria

- Left and right main panels each have a complete frame.
- `Ctrl+F1` hides and shows the left panel independently.
- `Ctrl+F2` hides and shows the right panel independently.
- Hiding both panels one at a time reaches the same hidden-both behavior as `Ctrl+O`.
- `Ctrl+O` from a partial hidden state hides both panels.
- `Ctrl+F1` and `Ctrl+F2` still work while both panels are hidden.
- `Tab` does not move focus to a hidden panel.
- `Swap panels` exchanges the left and right panel states without changing the
  active physical side.
- `Swap panels` exchanges `LeftViewMode` and `RightViewMode`.

## Verification

Commands:

```bash
dotnet build
dotnet test
```

Manual check:

- Start CSharpFar in a Windows console and try `Ctrl+F1`, `Ctrl+F2`, and `Ctrl+O` in
  visible, partial hidden, and both-hidden states.
