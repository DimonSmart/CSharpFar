# 0051 - Editor Unicode scalar cursor behavior

Related:
- 0046
- 0049
- 0050

## Context

The built-in editor stores text as .NET strings, so internal positions are
UTF-16 indexes. Unicode scalar values outside the Basic Multilingual Plane use
surrogate pairs. Editor cursor movement, deletion, status display, and rendering
must not split those pairs when users edit UTF-8 files containing four-byte
characters.

## Scope / Behavior

- Editor left/right movement treats a Unicode scalar value represented by a
  surrogate pair as one logical character.
- Backspace and Delete remove a whole scalar value rather than one surrogate
  half.
- Direct editor operations normalize cursor positions so they do not remain
  inside surrogate pairs.
- Status display reports the full scalar code point for the current character,
  for example `U+1F642/128578`.
- Logical editor columns count a non-BMP scalar as one symbol while preserving
  virtual columns after line end.
- Screen cursor placement accounts for terminal display cell width. Wide
  scalars occupy two screen cells where the editor width rules classify them as
  wide.
- When the cursor is on a wide surrogate-pair glyph, the editor highlights the
  whole glyph range and hides the native one-cell console cursor.
- Wide-glyph cursor blinking is handled by the editor while waiting for input.
- Splitting a line at the end of text containing non-BMP characters preserves
  the intended new cursor location.

## Non-goals

- This spec does not require changing the editor storage model away from UTF-16
  strings.
- This spec does not require full grapheme-cluster cursor semantics.
- Combining marks, zero-width joiner emoji sequences, East Asian width rules,
  and terminal-specific display-width differences require a separate decision
  or spike if they become product requirements.

## Acceptance criteria

- Cursor movement cannot land inside a surrogate pair.
- Selection, Backspace, Delete, and line splitting do not split four-byte UTF-8
  characters.
- Status reporting shows the complete scalar code point.
- Renderer cursor positioning accounts for known wide scalar display width.
- Tests cover movement, deletion, selection, status reporting, save behavior,
  wide-glyph cursor styling, and line splitting around non-BMP characters.
