# 0010 - File operations

Related:
- 0007
- 0008
- 0060

## Context

CSharpFar file operations must be predictable for everyday Far-like workflows:
copy, move, delete, and create directory. Operations need explicit dialogs,
progress, cancellation, conflict handling, metadata options, mask filtering, and
structured results. Single-item rename through `Shift+F6` is owned by `0060`.

Copy operations separate two concerns:

- conflict decision: what to do when the destination item already exists;
- copy mode: how to behave after read or write failures during content copy.

Append is not a supported operation or conflict decision.

## Scope / Behavior

Operation model:

- File operations use a request/result model rather than string overloads.
- Supported operation kinds are copy, move, delete, and create directory.
- Requests include sources, optional destination, and operation options.
- Options include copy mode, default conflict decision, timestamp preservation,
  attribute preservation, security mode, symlink mode, recycle-bin delete, and
  optional file mask.
- Execution is asynchronous, observes cancellation, and reports structured
  progress.
- Cancellation may leave already completed items in place but must not report
  success.
- Results report copied, moved, deleted, skipped, failed, canceled counts, byte
  totals, and item errors.
- Old string-based operation APIs are not kept as compatibility shims.

Pre-operation dialogs:

- `F5` opens a copy dialog.
- `F6` opens a move dialog unless the single-source edited destination is a
  rename case owned by `0060`.
- Copy and move destinations are prefilled from the opposite panel directory.
- Copy dialog fields include destination path, copy mode, access-rights mode,
  existing-file behavior, preserve timestamps, preserve attributes,
  symlink-content option, use-filter option, and filter mask.
- Move dialog fields include destination path, mask for multi-source moves, and
  conflict/metadata options needed when move falls back to copy-delete. Move and
  rename do not expose copy mode.
- Delete dialog shows item count, total size when reasonably available, and
  permanent-delete versus recycle-bin behavior.
- Dialogs use the shared popup/dialog renderer, active palette, and existing
  input patterns.
- Last selected operation options are stored in settings and reused as defaults.
- The built-in default access-rights mode for new settings is `Default`.

Copy dialog UI:

- The copy dialog title is `Copy`.
- It contains the line `Copy <source> to:`, destination input, copy-mode radio
  row, access-rights radio row, existing-file behavior row, timestamp checkbox,
  attributes checkbox, symlink checkbox, filter checkbox, filter mask input, and
  `{ Copy } [ Cancel ]` buttons.
- Local copy modes are `Normal`, `Reliable`, and `Fast salvage`.
- Provider copy exposes only `Normal` until provider copy supports seekable
  resume validation and per-file salvage continuation.
- Access rights use exactly radio-style notation such as
  `(x) Default ( ) Copy ( ) Inherit`.
- The existing-file choices are one logical focusable control even when they
  occupy multiple visual lines below `Already existing files:`. Left and Right
  move through all values, Tab leaves the whole group, and the console cursor
  follows the selected radio-style marker across lines.
- Copy conflict values are `Ask`, `Overwrite`, `Skip`, `Rename`, and `Only
  newer`.
- Move and rename conflict values are `Ask`, `Overwrite`, `Skip`, and `Rename`.
- The default filter mask text is `*`.
- When `Use filter` is off, the mask is not applied.
- When `Use filter` is on, only matching files are copied; recursive directory
  copy still traverses directories whose names do not match.
- Multiple simple masks may be separated with `;`; whitespace is trimmed.
- The dialog is compact, centered, visually modal, and uses shared dialog colors
  and shadow behavior.

Progress:

- Copy progress has a preliminary scan phase and an operation phase.
- The visible console cursor is hidden while the modal copy operation runs; the
  next active surface controls cursor visibility after the operation closes.
- During scan, the dialog shows current scanned folder, discovered file count,
  folder count, and accumulated byte count. It does not show a progress bar.
- During operation, the dialog shows current file path, destination path when
  applicable, current file progress, total byte progress, processed item counts,
  elapsed time, estimated remaining time, and speed.
- Total progress is byte-based after scan denominators are known.
- Long paths may be shortened in the middle.
- Progress bars use block-style characters, not `#`.
- `Esc` during scan cancels immediately.
- `Esc` during copying opens a warning confirmation dialog with `{ Yes }` and
  `[ No ]`; copying is paused while confirmation is open.
- Modal file-operation dialogs use shared dialog/window helpers.

Conflict handling:

- Supported copy conflict decisions are `Ask`, `Overwrite`, `OverwriteAll`,
  `Skip`, `SkipAll`, `Rename`, `RenameAll`, `OnlyNewer`, and `Cancel`.
- Supported move and rename conflict decisions are `Ask`, `Overwrite`, `Skip`,
  `Rename`, and `Cancel`.
- `OverwriteAll`, `SkipAll`, and `RenameAll` apply only to the current
  operation.
- `OnlyNewer` overwrites only when source last-write time is newer than
  destination last-write time. It is copy-only.
- `Rename` asks for a destination name.
- `RenameAll` generates names in the existing destination directory by appending
  ` (2)`, ` (3)`, and so on before the extension.
- Directory/file type conflicts are never overwritten silently.
- Conflict dialogs show source and destination metadata when available.
- Copy conflict dialogs may show `{ Overwrite } [ Skip ] [ Rename ] [ Cancel ]`.
- Move and rename conflict dialogs do not show append or only-newer.
- Remembered choices become operation-wide choices for the current operation.

Copy modes:

- `Normal` performs ordinary copying with no automatic retry. Destination
  conflicts use the configured conflict decision.
- `Reliable` aims to produce a complete correct destination file. It uses the
  tail-validation resume behavior from `0019`, retries source read and
  destination write failures, closes streams after failures, rereads the real
  destination length from disk, validates the source snapshot, and resumes only
  from the analyzer's safe offset.
- `Fast salvage` quickly copies readable files from a failing source. Source
  read failure marks that file failed, records an item error, cleans up the
  partial destination by default, and continues with later files. If
  `KeepPartialFilesOnError` is enabled, the partial destination is renamed with
  a safe partial-file suffix instead of being deleted. Destination write
  failures get at most one short retry and must not silently skip large batches
  of files.

Mask filtering:

- File operations use the existing Far-like mask support from
  `CSharpFar.Core.FileMasks`.
- Empty mask means all selected sources.
- Recursive masked copy applies masks to files and still traverses directories
  that may contain matching files.
- Empty directories are copied only when the mask is empty.
- File operations do not introduce a second mask syntax.

Metadata:

- When enabled, copied files preserve creation time, last-write time, and
  last-access time after content copy.
- Directory timestamps are restored after child processing.
- When enabled, ordinary file attributes are preserved after content copy.
- `ReadOnly` is preserved by default.
- Default security mode is `Default`.
- `CopyAccessControl` attempts to copy Windows ACLs and reports ACL failures as
  item errors.
- No cross-platform compatibility layer is added for ACL behavior.

Symlinks and reparse points:

- Supported policies are `CopyLink` and `CopyTargetContents`.
- Links and junction-like reparse points are detected before recursive
  traversal.
- Default behavior is `CopyLink`.
- If link recreation is not permitted, the operation reports a clear item error
  instead of silently copying target contents.
- `CopyTargetContents` follows the target and copies contents.

Delete:

- Delete supports permanent delete and recycle-bin delete.
- `AppSettings.FileOperations.UseRecycleBinForDelete` requests recycle-bin
  deletion only when the active source is local and the current platform
  reports recycle-bin support.
- Recycle-bin behavior is Windows-specific and belongs in platform-specific
  filesystem code.
- When the platform does not support recycle-bin deletion, the normal delete
  command confirms permanent deletion and deletes permanently regardless of
  the recycle-bin setting.
- Shared file-operation workflow delegates platform-specific filesystem details
  to platform-owned operations. Windows owns recycle-bin delete, reparse-point
  link recreation, attributes, and ACL copying. Unix/Linux owns symlink
  recreation and Unix file mode preservation and must not silently treat an
  explicit recycle-bin delete request as permanent delete.
- Delete progress shows item count and current path.
- Permanent recursive delete reports per-item failures.

Create directory:

- `F7` can create nested directories such as `a\b\c`.
- Invalid names fail before partial creation where possible.
- After success, the panel refreshes and places the cursor on the created
  top-level directory when possible.

## Non-goals

- Alternative NTFS data streams.
- Copying to multiple destination directories.
- Copying to `NUL`.
- `.diz` description preservation.
- Append.
- Rescue copy, sparse reconstruction of unreadable ranges, zero-filled damaged
  ranges, rsync-like block matching, whole-file hash verification, and parallel
  copy of a single file.
- Creating hard links or symbolic links instead of copying ordinary files.
- Hard-link identity preservation.
- Plugin or virtual-filesystem copy/move APIs.
- Archive, FTP, SFTP, or cloud file operations.
- Background minimized progress that allows panel browsing during an operation.
- Compatibility wrappers for old file-operation APIs.
- CD/DVD-specific read-only attribute clearing.

## Acceptance criteria

- `F5` opens a copy dialog with destination and operation options.
- A copy dialog opened with built-in default settings selects access-rights
  mode `Default`.
- Copy dialog keyboard navigation can move focus to the fixed `{ Copy }` and
  `[ Cancel ]` buttons, and Enter or Space activates the focused button.
- Moving focus from the destination input to access-rights, existing-file, or
  checkbox options moves the visible console cursor into the active option
  marker; it never remains visible in the previous input.
- `F6` opens a `Move` dialog prefilled with the opposite panel directory.
- Confirming an unchanged `F6` dialog moves current or selected items into the
  opposite panel directory.
- `F8` can delete permanently or send to recycle bin.
- On a supported platform, local `F8` delete uses the recycle bin when
  `UseRecycleBinForDelete` is enabled and confirms permanent deletion when it
  is disabled.
- On Unix/Linux, local `F8` delete always confirms permanent deletion and
  deletes permanently, including when `UseRecycleBinForDelete` is enabled.
- `F7` can create nested directory paths.
- Copy and move progress show current file progress and total progress.
- The console cursor does not blink over the copy progress dialog while copying
  is running.
- Progress shows speed, ETA, and processed item count.
- User cancellation stops the operation and reports cancellation.
- UI does not synchronously block on async file operations.
- Conflict handling supports overwrite, overwrite all, skip, skip all, rename,
  rename all, only-newer for copy, and cancel.
- Copy mode is selected independently from conflict handling and supports
  `Normal`, `Reliable`, and `Fast salvage` for local copy.
- `Append`, `AppendAll`, and resume-with-tail-validation are not conflict
  decisions.
- Old persisted `Append` and `AppendAll` conflict settings fall back to `Ask`.
- Old persisted `ResumeWithTailValidation` conflict settings become
  `CopyMode = Reliable` with a non-overwriting conflict decision.
- Rename-generated names use `name (2).ext`, `name (3).ext`, and so on.
- File masks use the existing Far mask parser.
- Masked recursive copy does not copy non-matching files.
- Timestamps, attributes, ACL mode, and symlink mode follow selected options.
- Link creation failure is reported clearly and does not silently copy target
  contents.
- On Unix/Linux, explicit recycle-bin delete fails clearly until Freedesktop
  Trash support exists.
- Directory/file type conflicts are not overwritten silently.
- Operation defaults persist under `AppSettings.FileOperations`.
- Panel refresh and selection clearing remain predictable after operations.
