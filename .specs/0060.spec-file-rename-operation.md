# 0060 - File rename operation

Type: Spec
Status: Current
Related:
- `0010.spec-file-operations.md`
- `0013.spec-dynamic-function-key-bar.md`

## Goal

Make renaming the current panel item a distinct user command. Plain `F6`
remains primarily move; `Shift+F6` is the direct rename shortcut.

## Scope

### Trigger

`Shift+F6` starts rename for the current item under the cursor.

Rules:

1. `Shift+F6` operates only on the current item under the cursor.
2. `Shift+F6` ignores the current multi-selection for source selection.
3. The command applies to files and directories.
4. The rename dialog title is `Rename`.
5. The destination input is prefilled with the current item name only, without
   the parent directory.
6. For a file, the prefilled value includes the file extension.
7. For a directory, the prefilled value is the directory name.
8. Confirming the unchanged dialog is a no-op.
9. Editing the prefilled name and confirming renames the current item in its
   current parent directory.

`F6` can also reach rename for a single source when the user edits the `Move`
dialog destination to a plain name. The default `F6` destination is still the
opposite panel directory and is owned by `0010.spec-file-operations.md`.

If the entered destination contains a path, the operation follows the normal
move or move-and-rename rules from `0010.spec-file-operations.md`.

### Dialog

Rename uses the same Far-like modal dialog family as file operations in
`0010.spec-file-operations.md`.

Rename dialog fields:

1. new file name;
2. conflict mode.

Rules:

1. a plain entered value is interpreted as an item name in the current parent
   directory;
2. an entered path follows the move or move-and-rename rules from
   `0010.spec-file-operations.md`;
3. invalid names fail before changing the filesystem;
4. `Append` is not shown because appending is only meaningful for copy
   conflicts;
5. conflict handling follows the shared conflict model from
   `0010.spec-file-operations.md`.

### Command binding

`Shift+F6` must have its own stable command id and function-key binding instead
of being inferred from plain `F6`.

The function-key bar plain `F6` label may remain `RenMov` for Far-like
compatibility. The `Shift+F6` label is `Rename`.

### Result

After a successful rename:

1. refresh the affected panel;
2. keep panel refresh and selection clearing predictable according to
   `0010.spec-file-operations.md`;
3. leave the file in its original directory under the new name.

## Acceptance criteria

1. `Shift+F6` on a file opens a `Rename` dialog whose destination input is the
   file name including extension.
2. Changing that value to another plain name renames the file in the current
   directory.
3. `Shift+F6` on a directory opens the same rename flow with the directory name
   prefilled.
4. If multiple items are selected, `Shift+F6` still renames only the current
   item under the cursor.
5. Confirming an unchanged `Shift+F6` dialog is a no-op.
6. Entering a path instead of a plain name follows the normal move or
   move-and-rename destination rules.
7. Rename conflicts support shared overwrite, skip, rename, only newer and
   cancel behavior where those decisions apply.
8. Rename conflicts never show `Append`.
9. `Shift+F6` has its own function-key binding and command id.
