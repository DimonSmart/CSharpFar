# 060. File rename operation

Type: spec

Related:
- `0010.spec-file-operations.md`

## Goal

Make renaming one file from the active panel a distinct file operation instead
of treating it as an incidental branch of move.

## Scope

### Trigger

`F6` can start file rename when all of these are true:

1. exactly one file is selected, or no files are selected and the current panel
   item is a file;
2. the user enters a new name without a path;
3. the operation stays in the source file's current directory.

If the destination contains a path, the operation is move and is owned by
`0010.spec-file-operations.md`.

If multiple sources are selected, `F6` is move to the specified directory and is
owned by `0010.spec-file-operations.md`.

### Dialog

Rename uses the same Far-like modal dialog family as file operations in
`0010.spec-file-operations.md`.

Rename dialog fields:

1. new file name;
2. conflict mode.

Rules:

1. the entered value is interpreted as a file name, not as a destination path;
2. invalid names fail before changing the filesystem;
3. `Append` is not shown because appending is only meaningful for copy
   conflicts;
4. conflict handling follows the shared conflict model from
   `0010.spec-file-operations.md`.

### Result

After a successful rename:

1. refresh the affected panel;
2. keep panel refresh and selection clearing predictable according to
   `0010.spec-file-operations.md`;
3. leave the file in its original directory under the new name.

## Acceptance criteria

1. `F6` with one file and a new name without a path renames the file.
2. `F6` with a destination path moves instead of renaming.
3. `F6` with multiple sources moves to the specified directory.
4. Rename conflicts support shared overwrite, skip, rename, only newer and
   cancel behavior where those decisions apply.
5. Rename conflicts never show `Append`.
