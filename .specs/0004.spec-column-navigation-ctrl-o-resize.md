# 004. Навигация по колонкам, командная строка при Ctrl+O и устойчивый resize

Source: `.worklog/0004.spec-column-navigation-ctrl-o-resize.md`.

## Цель

Уточнить поведение основного экрана CSharpFar после добавления многоколоночного режима панели:

1. выход из каталога наверх должен возвращать курсор на каталог, из которого вышли;
2. клавиши `Left` / `Right` должны перемещать курсор между визуальными колонками панели;
3. командная строка должна оставаться видимой и редактируемой, когда панели скрыты через `Ctrl+O`;
4. после выполнения shell-команды приглашение ввода должно появляться в стабильной нижней строке;
5. изменение размера окна не должно оставлять частично прорисованные панели.

## Навигация вверх из каталога

Выход наверх должен работать одинаково для `Backspace` и для `Enter` на элементе `..`.

Требования:

1. Перед переходом запоминается имя текущего каталога.
2. После загрузки родительского каталога курсор ставится на этот дочерний каталог.
3. Если каталогов и файлов много, `ScrollOffset` должен быть установлен так, чтобы этот элемент был виден.
4. Если элемент не найден, панель остаётся в обычном состоянии после загрузки родителя.

## Left / Right в многоколоночной панели

В режиме `BriefTwoColumns`:

1. `Left` перемещает курсор в колонку слева, сохраняя визуальную строку, если там есть элемент.
2. Если курсор уже в левой колонке, `Left` ставит курсор на первый элемент списка.
3. `Right` перемещает курсор в колонку справа, сохраняя визуальную строку, если там есть элемент.
4. Если справа нет элемента в той же строке, курсор ставится на последний элемент списка.
5. Если курсор уже в последней колонке, `Right` ставит курсор на последний элемент списка.
6. После перемещения панель должна прокручиваться так, чтобы новый элемент был виден.

Для одноколоночного режима допустимое поведение:

```text
Left  -> первый элемент
Right -> последний элемент
```

Это сохраняет предсказуемую семантику "левее / правее" без отдельной скрытой логики для командной строки.

## Командная строка при скрытых панелях

После `Ctrl+O` панели скрываются, но командная строка остаётся видимой в той же строке, где она была при показанных панелях.

Требования:

1. В скрытом режиме пользователь может вводить команду.
2. `Left` / `Right` в скрытом режиме двигают курсор внутри командной строки.
3. `Home` / `End`, `Backspace`, `Delete`, `Esc` работают с командной строкой.
4. `Enter` выполняет введённую команду.
5. `Ctrl+O` возвращает панели без потери текста командной строки.
6. Если команда длиннее ширины экрана, отображаемый фрагмент должен прокручиваться так, чтобы курсор оставался видимым.

## Приглашение после shell-команды

После выполнения shell-команды нужно снова вывести приглашение командной строки в стабильной нижней строке экрана.

Требования:

1. Перед запуском команды строка приглашения очищается и в неё выводится `currentDirectory>command`.
2. Shell-команда наследует консоль и выводит stdout/stderr в обычный console buffer.
3. После завершения команды очищаются строка командного ввода и нижняя строка под ней.
4. В строку командного ввода выводится `currentDirectory>` и курсор ставится после `>`.
5. Снимок underlay для `Ctrl+O` делается после вывода нового приглашения.
6. При показанных панелях renderer рисует командную строку в той же позиции поверх underlay.
7. При скрытых панелях пользователь видит то же приглашение в той же позиции.

## Resize

Resize может приходить несколькими событиями подряд. Отрисовка не должна смешивать старый и новый размеры в одном кадре.

Требования:

1. `ScreenRenderer.BeginFrame()` фиксирует размер кадра.
2. Все записи внутри кадра используют зафиксированный размер, а не перечитывают размер консоли на каждом `Write`.
3. При изменении размера буферы renderer пересоздаются, следующий кадр считается полным.
4. Основной цикл приложения должен коалесцировать resize-события короткой паузой до стабильного размера.
5. В скрытом режиме resize восстанавливает подходящий underlay, а при несовпадении размера очищает экран и заново рисует командную строку.

## Acceptance criteria

1. `Enter` на `..` и `Backspace` возвращают в родителя с курсором на каталоге, из которого вышли.
2. В `BriefTwoColumns` `Left` / `Right` перемещают курсор между колонками и корректно обрабатывают края.
3. В `Ctrl+O` командная строка видна, принимает ввод и поддерживает перемещение курсора.
4. После `dir` и последующего `Ctrl+O` приглашение `currentDirectory>` видно в нижней строке, а курсор не остаётся в последней строке вывода `dir`.
5. При resize панели перерисовываются полным кадром и не оставляют артефакты от промежуточных размеров.
