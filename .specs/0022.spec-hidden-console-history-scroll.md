# 0022 SPEC Hidden console history scroll

Source: `.worklog/0022.spec-hidden-console-history-scroll.active.md`.

Type: spec

Replaces:
- none

Related:
- 0004
- 0016

## Goal

When panels are hidden through `Ctrl+O`, the user can scroll the console viewport up to
inspect older command output. CSharpFar must not immediately pull the viewport back down
just because only the viewport origin changed.

When the user resumes interaction with CSharpFar, the application returns the visible
viewport to the bottom of the console buffer and keeps the existing command-line and panel
rendering behavior.

## Context

Viewport-bound rendering from `0016` correctly treats `Left`, `Top`, `Width`, and `Height`
as part of the physical rendering target. That is required while panels are visible.

In hidden-panel mode, however, a user scrolls console history by changing the visible
viewport origin. Treating that origin-only change as a redraw trigger immediately restores
the hidden screen and redraws the command line, making old shell output hard to inspect.

## Scope

- Distinguish hidden-panel origin-only viewport moves from real width/height changes.
- Leave origin-only viewport moves alone while panels are hidden.
- Scroll the viewport back to the bottom before ordinary hidden-mode input edits the
  command line.
- Scroll the viewport back to the bottom before returning from hidden mode to panel mode.
- Add a console-driver API for explicit bottom scrolling.
- Cover the behavior with fake-driver unit tests.

## Non-goals

- Do not pin the CSharpFar command line while the user is viewing old output.
- Do not change visible-panel resize behavior.
- Do not add a new scrollback UI or a custom command-output viewer.
- Do not emulate Windows console history scrolling on non-Windows platforms.

## Acceptance criteria

- Hidden panels plus origin-only viewport movement does not redraw the command line or
  restore stale underlay.
- Hidden panels plus actual size change still redraws the hidden command line.
- After hidden scroll up, ordinary command input returns the viewport to bottom and edits
  the command line.
- After hidden scroll up, `Ctrl+O` returns the viewport to bottom and redraws panels.
- Visible panels still redraw on viewport-origin changes.

## Verification

Commands:

```bash
dotnet build
dotnet test
```

Manual check:

- Run app in a real Windows console.
- Execute enough output to create scrollback.
- Press `Ctrl+O`, scroll up, and verify the viewport stays up.
- Press a character and verify the viewport returns down and edits the command line.
- Scroll up again, press `Ctrl+O`, and verify panels return at the bottom.
