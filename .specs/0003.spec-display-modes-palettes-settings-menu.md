# 0003 - Display modes, palettes, and settings menu

## Context

CSharpFar needs configurable panel display modes, configurable console palettes,
and an in-application settings dialog so users can change the main visual
presentation without editing configuration files manually.

## Scope / Behavior

Panel display modes:

- `Full` shows the detailed file list with name, size, and directory marker.
- `BriefTwoColumns` shows two Far-like name columns without file sizes or
  `<DIR>` markers in the list.
- `BriefTwoColumns` fills the first column top-to-bottom, then the second
  column.
- Cursor highlighting applies only to the current item cell in the relevant
  column.
- Up/down navigation moves through the logical item list, not only through the
  visual column.
- Long names are clipped to the column width.
- The user can switch modes through documented hotkeys and through settings.
- The selected mode is persisted. Per-panel modes are preferred; a shared mode
  is acceptable only when the code remains open to later per-panel storage.

Palettes:

- A `Default` palette preserves the existing CSharpFar look.
- A `FarClassic` palette approximates Far Manager colors.
- Renderers obtain colors from the active palette instead of hard-coded color
  literals.
- The active palette name is persisted in settings.
- An unknown palette name in settings falls back to `Default` without crashing.
- `FarClassic` panel titles use text color rather than a wide title background.
- `FarClassic` uses a green cursor background with black foreground.

Settings menu:

- The application provides a modal settings window over the current UI.
- The settings window allows changing panel view mode and palette.
- `Esc` closes without saving changes.
- `F10` saves changes and closes.
- The active row is highlighted using selection colors from the active palette.
- Closing the settings dialog redraws the panels.

Main-screen Far-like polish:

- `BriefTwoColumns` displays a column header equivalent to `n Name | Name`,
  where `n` indicates name sorting.
- Each panel has status rows for the current item and directory totals.
- Directory totals count bytes for files only, count files, and count folders
  excluding the parent `..` item.
- Human-readable sizes in status rows use a comma decimal separator, such as
  `18,9 K`.
- The function-key bar labels are:
  `1Help 2UserMn 3View 4Edit 5Copy 6RenMov 7MkFold 8Delete 9ConfMn 10Quit`.
- The current time is drawn in the top-right area as `H:mm` without breaking
  the panel border or exceeding screen width.

## Non-goals

- This spec does not require removing existing display modes.
- This spec does not require a full Far Manager settings system.
- This spec does not require every future setting to be exposed immediately.

## Acceptance criteria

- `Full` and `BriefTwoColumns` are separate modes.
- The user can switch display mode and palette.
- Selected display mode and palette are restored after restart.
- Renderers use the active palette.
- The settings dialog supports save and cancel behavior.
- Far-like panel status, function-key labels, and clock rendering are visible
  without overlapping other UI.
