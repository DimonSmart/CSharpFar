# Дополнение к ТЗ: режимы отображения, палитры и меню настроек

Source: `.worklog/0003.spec-display-modes-palettes-settings-menu.md`.

## 17. Поддержка нескольких режимов отображения панели

### 17.1. Цель

Не заменять текущий режим отображения файлов, а добавить инфраструктуру для нескольких режимов просмотра панели.

Текущий одноколоночный режим с именем, размером и признаком директории должен остаться доступным. Новый Far-like двухколоночный режим должен быть добавлен как отдельный режим.

### 17.2. Требования

Добавить модель режимов отображения панели.

Пример:

```csharp
public enum PanelViewMode
{
    Full,
    BriefTwoColumns
}
```

Названия можно скорректировать под текущую архитектуру проекта, но смысл должен сохраниться:

```text
Full             текущий подробный режим
BriefTwoColumns  новый Far-like двухколоночный режим
```

### 17.3. Поведение режимов

#### Full

Текущий режим отображения CSharpFar.

Пример:

```text
README.md                       9K
src                            <DIR>
tests                          <DIR>
```

Требования:

1. Сохранить текущую функциональность.
2. Показывать имя, размер, признак директории.
3. Не ломать существующую навигацию, выделение и скроллинг.

#### BriefTwoColumns

Новый режим, визуально похожий на Far Manager.

Пример:

```text
n        Name        │        Name
..                   │ CSharpFar.slnx
.claude              │ README.md
.codemie             │
.git                 │
artifacts            │
src                  │
tests                │
```

Требования:

1. Внутри панели две колонки имён.
2. Размеры файлов в списке не показывать.
3. Директории не помечать `<DIR>` внутри списка.
4. Добавить строку заголовков `n Name | Name`.
5. Список заполнять сверху вниз по первой колонке, затем по второй.
6. Активный элемент выделять только в своей колонке.
7. Навигация вверх/вниз должна работать внутри общего списка элементов, а не только внутри визуальной колонки.
8. Скроллинг должен корректно работать, если элементов больше, чем помещается в две колонки.
9. Обрезка длинных имён должна учитывать ширину колонки.

### 17.4. Переключение режимов

Добавить возможность переключать режимы отображения.

Минимальный вариант для первого этапа:

```text
Alt+1 или Ctrl+1 → Full
Alt+2 или Ctrl+2 → BriefTwoColumns
```

Точную комбинацию можно выбрать по текущей архитектуре ввода, но она должна быть явно описана в README или help screen.

Дополнительно режим должен переключаться через меню настроек.

### 17.5. Хранение выбранного режима

Выбранный режим должен храниться в настройках приложения.

Минимальная структура:

```json
{
  "panels": {
    "leftViewMode": "BriefTwoColumns",
    "rightViewMode": "BriefTwoColumns"
  }
}
```

Допускается общий режим для обеих панелей, если так проще для первого этапа:

```json
{
  "panelViewMode": "BriefTwoColumns"
}
```

Но архитектурно лучше предусмотреть разные режимы для левой и правой панели.

### Acceptance criteria

1. Старый режим отображения не удалён.
2. Новый двухколоночный режим добавлен отдельно.
3. Пользователь может переключить режим.
4. После перезапуска выбранный режим восстанавливается.
5. Far-like внешний вид включается через `BriefTwoColumns`, а не через переписывание единственного существующего renderer.

## 18. Поддержка набора палитр

### 18.1. Цель

Не захардкодить одну цветовую схему, а добавить поддержку набора палитр.

На первом этапе достаточно одной-двух палитр, но код должен быть устроен так, чтобы новые палитры добавлялись без переписывания renderer.

### 18.2. Требования

Добавить модель палитры.

Пример:

```csharp
public sealed class ConsolePalette
{
    public string Name { get; init; }

    public ConsoleColor Background { get; init; }
    public ConsoleColor PanelBackground { get; init; }
    public ConsoleColor PanelBorder { get; init; }
    public ConsoleColor PanelTitle { get; init; }
    public ConsoleColor ColumnHeader { get; init; }

    public ConsoleColor NormalFile { get; init; }
    public ConsoleColor Directory { get; init; }

    public ConsoleColor SelectedForeground { get; init; }
    public ConsoleColor SelectedBackground { get; init; }

    public ConsoleColor StatusForeground { get; init; }
    public ConsoleColor CommandLineForeground { get; init; }
    public ConsoleColor FunctionKeyNumber { get; init; }
    public ConsoleColor FunctionKeyText { get; init; }
    public ConsoleColor FunctionBarBackground { get; init; }
}
```

Имена свойств можно адаптировать, но палитра должна покрывать все основные элементы экрана:

```text
общий фон
фон панели
рамки панели
заголовок панели
заголовки колонок
обычные файлы
директории
активный элемент
статусные строки панели
командная строка
нижняя function-key bar
```

### 18.3. Начальный набор палитр

Добавить минимум две палитры.

#### Default

Текущая или близкая к текущей цветовая схема CSharpFar.

Назначение: не ломать привычный вид текущей реализации.

#### FarClassic

Цветовая схема, похожая на Far Manager.

Примерные настройки:

```text
Background            = DarkBlue
PanelBackground       = DarkBlue
PanelBorder           = Cyan / BrightCyan
PanelTitle            = Cyan / BrightCyan
ColumnHeader          = Yellow

NormalFile            = Cyan / BrightCyan
Directory             = Cyan / BrightCyan

SelectedBackground    = Green
SelectedForeground    = Black

StatusForeground      = Cyan / BrightCyan
CommandLineForeground = White

FunctionBarBackground = Gray
FunctionKeyNumber     = Blue / DarkCyan
FunctionKeyText       = Black
```

Точные значения зависят от доступной `ConsoleColor`-палитры и renderer, но визуально результат должен быть ближе к Far Manager.

### 18.4. Хранение выбранной палитры

Выбранная палитра должна храниться в настройках приложения.

Пример:

```json
{
  "palette": "FarClassic"
}
```

При старте приложения:

1. Загрузить настройки.
2. Найти палитру по имени.
3. Если палитра не найдена, использовать `Default`.
4. Не падать из-за некорректного имени палитры в конфиге.

### 18.5. Использование палитры

Renderer не должен напрямую использовать цвета из разных мест кода.

Плохо:

```csharp
Console.ForegroundColor = ConsoleColor.White;
Console.BackgroundColor = ConsoleColor.Blue;
```

Хорошо:

```csharp
_console.SetForeground(palette.PanelBorder);
_console.SetBackground(palette.PanelBackground);
```

### Acceptance criteria

1. В коде есть отдельная модель палитры.
2. Есть минимум две палитры: `Default` и `FarClassic`.
3. Палитра выбирается через настройки.
4. Renderer использует цвета из активной палитры.
5. Добавление новой палитры не требует переписывать логику отображения файлов.

## 19. Окно меню настроек

### 19.1. Цель

Добавить отдельное окно настроек, через которое пользователь сможет менять внешний вид CSharpFar без ручного редактирования конфигурационного файла.

На первом этапе меню настроек должно покрывать:

```text
режим отображения панелей
палитру
```

Дальше его можно расширить под другие параметры.

### 19.2. Вызов меню настроек

Добавить команду открытия окна настроек.

Варианты:

```text
F9 → меню
Options → Settings
```

или временно:

```text
Ctrl+S → Settings
```

Если полноценное верхнее меню пока не реализовано, допускается открыть окно настроек по отдельной горячей клавише.

Комбинация должна быть указана в README.

### 19.3. Внешний вид окна

Окно настроек должно быть модальным псевдографическим окном поверх панелей.

Пример:

```text
╔════════════ Settings ════════════╗
║                                  ║
║  View mode:  Brief two columns   ║
║  Palette:    FarClassic          ║
║                                  ║
║  Enter - change                  ║
║  Esc   - close                   ║
║  F10   - save and close          ║
║                                  ║
╚══════════════════════════════════╝
```

Требования:

1. Окно рисуется поверх текущего экрана.
2. После закрытия экран панелей должен быть перерисован.
3. Окно использует активную палитру.
4. Активная строка внутри окна выделяется цветом selection из палитры.
5. `Esc` закрывает окно.
6. `Enter` меняет выбранное значение или открывает список значений.
7. `F10` сохраняет настройки и закрывает окно.

### 19.4. Настройки режима отображения

В меню настроек добавить пункт:

```text
View mode
```

Значения:

```text
Full
Brief two columns
```

Минимальный вариант: настройка применяется сразу к обеим панелям.

Лучший вариант: добавить отдельные настройки:

```text
Left panel view mode
Right panel view mode
```

Для первого этапа можно сделать общий режим, но код лучше не закрывать от будущего разделения по панелям.

### 19.5. Настройки палитры

В меню настроек добавить пункт:

```text
Palette
```

Значения:

```text
Default
FarClassic
```

При смене палитры экран должен перерисовываться сразу или после сохранения. Предпочтительно сразу, чтобы пользователь видел результат.

### 19.6. Сохранение настроек

После изменения настроек и нажатия `F10` сохранить конфигурацию на диск.

Пример структуры:

```json
{
  "palette": "FarClassic",
  "panelViewMode": "BriefTwoColumns"
}
```

Если поддержаны отдельные режимы панелей:

```json
{
  "palette": "FarClassic",
  "panels": {
    "leftViewMode": "BriefTwoColumns",
    "rightViewMode": "BriefTwoColumns"
  }
}
```

### 19.7. Отмена изменений

Минимальный вариант:

```text
Esc закрывает окно без сохранения изменений.
```

Лучший вариант:

1. При открытии окна создать копию текущих настроек.
2. Все изменения применять к копии.
3. `F10` сохраняет копию.
4. `Esc` выбрасывает копию и возвращает старые настройки.
5. После `Esc` экран перерисовывается со старой палитрой и старым режимом.

### Acceptance criteria

1. В приложении есть модальное окно настроек.
2. Через него можно выбрать режим отображения.
3. Через него можно выбрать палитру.
4. Настройки сохраняются на диск.
5. После перезапуска приложения выбранные настройки применяются.
6. `Esc` закрывает окно без сохранения.
7. `F10` сохраняет настройки и закрывает окно.

## 21. Обновлённый Definition of Done

Задача считается выполненной, если:

1. В CSharpFar есть минимум два режима отображения: текущий `Full` и новый `BriefTwoColumns`.
2. Новый Far-like режим не ломает старый режим.
3. В CSharpFar есть минимум две палитры: `Default` и `FarClassic`.
4. Renderer не использует цвета напрямую, а берёт их из активной палитры.
5. Есть окно настроек.
6. Через окно настроек можно выбрать режим отображения и палитру.
7. Настройки сохраняются и восстанавливаются после перезапуска.
8. Far-like внешний вид достигается комбинацией:

   ```text
   palette = FarClassic
   panelViewMode = BriefTwoColumns
   ```
9. Архитектура позволяет позже добавить новые режимы, например:

   ```text
   Full
   BriefTwoColumns
   Detailed
   Wide
   CustomColumns
   ```
10. Архитектура позволяет позже добавить новые палитры без изменения логики panel renderer.

## 22. Пропущенная Far-like полировка из исходного визуального ТЗ

После сверки с исходным сравнением Far Manager и CSharpFar нужно также реализовать следующие детали. Они относятся не к инфраструктуре режимов и палитр, а к визуальному совпадению основного экрана.

### 22.1. FarClassic без яркой плашки заголовка

В палитре `FarClassic` путь в верхней рамке панели должен отображаться на синем фоне панели, без широкой cyan-заливки.

Требования:

1. Активный заголовок панели в `FarClassic` отличается цветом текста, а не заливкой.
2. Цвет рамки и текста заголовка близок к cyan.
3. `Default` может сохранить прежний вид, чтобы не ломать старую тему.

### 22.2. Зелёный курсор FarClassic

В `FarClassic` текущий элемент под курсором должен выделяться зелёным фоном.

Требования:

```text
CursorActiveBackground = Green
CursorActiveForeground = Black
```

Выделение должно покрывать только ячейку элемента в текущей колонке.

### 22.3. Заголовок `n Name | Name`

В `BriefTwoColumns` первая строка внутри панели должна содержать индикатор сортировки:

```text
n        Name        │        Name
```

Минимальный вариант: `n` означает сортировку по имени. Позже индикатор можно расширить под другие режимы сортировки.

### 22.4. Far-like статусные строки панели

Внизу каждой панели должны быть две строки состояния вместо одного текста `N items`.

Строка текущего элемента:

```text
..              Up    07.05.26 00:19
README.md       9K    07.05.26 00:19
src             <DIR> 07.05.26 00:19
```

Строка статистики директории:

```text
Bytes: 18,9 K, files: 4, folders: 7
```

Требования:

1. `Bytes` считает только файлы текущей директории.
2. `files` считает только файлы.
3. `folders` считает директории без родительского `..`.
4. Размеры в статусе форматируются с запятой: `18,9 K`, `19,7 M`, `1,2 G`.
5. Статусные строки используют цвета активной палитры.
6. Расчёт видимых строк панели должен учитывать эти две нижние строки.

### 22.5. Function-key bar как в Far

Подписи нижней строки должны быть:

```text
1Help 2UserMn 3View 4Edit 5Copy 6RenMov 7MkFold 8Delete 9ConfMn 10Quit
```

### 22.6. Часы справа сверху

В правом верхнем углу экрана отображать текущее время в формате:

```text
H:mm
```

Требования:

1. Часы рисуются поверх правой верхней области панели, как в Far.
2. Цвет берётся из активной палитры.
3. Часы не должны ломать рамку и не должны выходить за ширину экрана.

### 22.7. Обрезка имён в brief-режиме без `~`

В `BriefTwoColumns` длинные имена должны просто обрезаться по ширине колонки, без добавления `~`.

Требования:

1. Имя не выходит за границы колонки.
2. Имя не затирает вертикальный разделитель.
3. Не использовать DOS-like `~` truncation в Far-like brief-режиме.

### 22.8. Рисование модальных окон без мигания

Модальные окна, включая окно настроек, должны перерисовываться через frame-buffer `ScreenRenderer.BeginFrame()`.

Требования:

1. Навигация внутри окна настроек не должна визуально очищать и заново рисовать всё окно в несколько отдельных console-write операций.
2. При `Up`/`Down` достаточно атомарно обновлять изменившиеся строки.
3. После закрытия окна подложка восстанавливается, затем основной экран перерисовывается обычным циклом приложения.

### 22.9. Far-like рамки панелей

Панели в Far-like режиме должны использовать двойную псевдографику:

```text
╔════════════════════╗
║                    ║
╚════════════════════╝
```

Требования:

1. Цвет рамок берётся из активной палитры.
2. Для `FarClassic` активная и неактивная панели используют cyan-рамки.
3. Центральная граница двух панелей выглядит как часть общей Far-like псевдографики без белого зазора.

### Acceptance criteria

1. `FarClassic + BriefTwoColumns` даёт cyan-рамки, жёлтый `n Name | Name`, зелёный курсор и без cyan-плашки заголовка.
2. Внизу каждой панели есть строка текущего элемента и строка статистики директории.
3. Нижняя function-key bar использует `UserMn`, `MkFold`, `ConfMn`.
4. Справа сверху отображаются часы.
5. Длинные имена в brief-режиме обрезаются без `~`.
6. Меню настроек не мигает при перемещении фокуса.
7. Панели используют double-line Far-like рамки.
