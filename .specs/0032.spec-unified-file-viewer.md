## Goal

Use one F3 viewer implementation for every file and let the user switch between text and
hex display modes.

## Context

`0031` introduced a streaming viewer for large files, but kept the old in-memory
`FileViewer` path for small text files. The current requirement removes that split:
viewing should always use the streaming byte reader, block cache, lazy scanner, and bounded
rendering path.

The viewer also needs an explicit display mode. Binary files should open as hex by
default, but every file must be viewable as text or as a 16-byte-per-row hex dump. Text
rendering must sanitize control characters before writing to the console so file content
cannot act as terminal control input/output.

## Scope

- Route every F3 open through the streaming viewer.
- Remove the old in-memory viewer loop from `FileViewer`.
- Keep `TextFileReader` only for other existing users such as editor and Quick View.
- Add a viewer display mode state with text and hex modes.
- Default binary-looking files to hex and text-looking files to text.
- Add a keyboard command to toggle the current file between text and hex mode.
- Keep navigation, horizontal scrolling, jump, and follow behavior working in both modes.
- Support vertical text scrolling with the mouse wheel.
- Render text mode with sanitized control characters.
- Render hex mode as a 16-byte-per-row dump with printable ASCII preview only.
- Update tests and README.

## Non-goals

- Editing files in either mode.
- Search in the viewer.
- Persistent per-file mode settings.
- Changing Quick View behavior.
- Removing `TextFileReader` from editor or Quick View.
- A richer binary editor.

## Acceptance criteria

- F3 never calls the old in-memory `FileViewer` loop.
- Small and large files both use the streaming viewer path.
- Text-looking files open in text mode by default.
- Binary-looking files open in hex mode by default.
- The user can toggle the same file between text and hex mode.
- Control characters from file content are rendered as inert visible/safe characters or
  spaces, not sent to the console as controls.
- Existing large-file navigation and follow behavior still work.
- The mouse wheel scrolls the visible file content vertically.
- Quick View remains a bounded preview.
