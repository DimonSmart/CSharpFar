# 0009 - Open panel item

Type: Spec
Status: Current

Related:
- 0001
- 0004
- 0007

## Context

Users open directories and files from panel items with `Enter` and mouse
actions. Single-click selection must remain distinct from opening an item.

## Scope / Behavior

- `Enter` on a directory changes the active panel to that directory.
- `Enter` on `..` navigates to the parent directory using the parent-navigation
  behavior from `0004`.
- `Enter` on a regular file opens the file through the configured platform file
  association or executable-launch path.
- `Enter` on an executable file launches it according to the current execution
  rules.
- A single mouse click on an item makes it current without opening it.
- A double-click on a directory behaves like `Enter` on that directory.
- A double-click on a regular file behaves like `Enter` on that file.
- Two separate single clicks on different items must not be interpreted as a
  double-click open.
- Open failures are reported to the user and leave the panel in a predictable
  state.

## Non-goals

- This spec does not define custom file associations.
- This spec does not define plugin-provided open handlers.
- This spec does not require opening unsafe executables without normal platform
  checks.

## Acceptance criteria

- `Enter` and double-click open directories and files consistently.
- Single-click changes only the current item.
- Double-click detection is item-specific and time-bounded.
- Failed opens report an error without corrupting panel state.
- Parent item opening follows the cursor restoration behavior from `0004`.
