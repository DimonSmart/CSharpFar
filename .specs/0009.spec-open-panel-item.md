# 009. Открытие элемента файловой панели через мышь и Enter

Source: `.worklog/0009.spec-open-panel-item.md`.

Type: spec

Related:
- `.worklog/0004.spec-column-navigation-ctrl-o-resize.md`
- `.worklog/0007.spec-file-panel-options.md`

## Goal

Добавить единый пользовательский путь открытия текущего элемента файловой панели.

Один и тот же механизм должен использоваться для:

1. двойного щелчка мышью по элементу панели;
2. нажатия `Enter` на выбранном элементе.

Для каталогов это навигация внутри панели. Для файлов это запуск через стандартный механизм операционной системы.

## Context

CSharpFar имитирует Far Manager, но на этом этапе не реализует собственную систему file associations Far Manager.

В Far Manager пользователь может назначать отдельные действия для `Enter`, `F3`, `F4` и других команд. Эта задача не должна вводить такую таблицу соответствий, настройки ассоциаций или hardcoded-приложения для расширений вроде `.txt`, `.pdf`, `.png`.

Открытие файла должно опираться на Windows Shell:

1. документ или другой неисполняемый файл открывается зарегистрированным системным действием по умолчанию;
2. `.exe` запускается как выбранная программа;
3. `.bat`, `.cmd`, `.com` и похожие типы идут через стандартное поведение Shell без отдельной логики CSharpFar.

## Scope

Реализовать общий метод уровня файловой панели, контроллера панели или приложения. Точное имя выбрать по текущей архитектуре, например:

```text
OpenCurrentItem()
OpenPanelItem(FilePanelItem item)
```

Поведение общего метода:

1. если элемент является директорией, выполнить обычный переход панели в эту директорию;
2. если элемент является файлом, открыть его через системный механизм запуска;
3. для Windows-запуска файла использовать `ProcessStartInfo` с `UseShellExecute = true`;
4. `Enter` и двойной щелчок должны вызывать этот общий путь, а не расходиться по поведению.

Двойной щелчок мышью:

1. по папке делает эту папку текущей и обновляет активную панель;
2. по файлу вызывает общий обработчик открытия файла;
3. срабатывает только если оба клика относятся к одному и тому же элементу панели;
4. не срабатывает, если первый клик был по одной строке, а второй по другой;
5. не срабатывает, если клик был вне области файлового списка.

Одинарный щелчок мышью:

1. только выбирает элемент в панели;
2. не открывает файл;
3. не входит в папку.

`Enter`:

1. на папке использует тот же путь выполнения, что и двойной щелчок по папке;
2. на файле делает то же самое, что двойной щелчок по этому файлу;
3. на `..` сохраняет существующую навигационную семантику возврата в родительский каталог.

## Non-goals

Не реализовывать в этой задаче:

1. собственные file associations CSharpFar;
2. импорт или повторение Far Manager file associations;
3. отдельные команды открытия для `F3`, `F4` или других действий;
4. таблицу расширений файлов;
5. hardcoded-приложения для конкретных расширений;
6. специальную логику для `.bat`, `.cmd`, `.com` сверх Windows Shell;
7. поддержку совместимости со старыми путями открытия, если они будут заменены общим обработчиком;
8. кроссплатформенный запуск файлов вне текущих требований Windows.

## Acceptance criteria

1. `Enter` на директории и двойной щелчок по этой директории используют общий путь открытия.
2. `Enter` на файле и двойной щелчок по этому файлу используют общий путь открытия.
3. Одинарный щелчок по элементу только выбирает его и не открывает.
4. Двойной щелчок по папке переходит в папку и обновляет активную панель.
5. Двойной щелчок по файлу открывает файл через Windows Shell.
6. `.exe` запускается выбранным файлом через стандартный Shell path.
7. `.bat`, `.cmd`, `.com` не имеют отдельной ветки CSharpFar и обрабатываются Shell.
8. В коде нет собственной таблицы file associations.
9. В коде нет hardcoded-приложений для `.txt`, `.pdf`, `.png` и других расширений.
10. `ProcessStartInfo.UseShellExecute` явно установлен в `true` при открытии файла.
11. Двойной щелчок срабатывает только если оба клика пришли по одному и тому же item панели.
12. Два клика по разным строкам не открывают элемент.
13. Клик вне области файлового списка не открывает элемент.
14. Навигация `Enter` на `..` визуально остается прежней.

## Verification

Commands:

```bash
dotnet build
dotnet test
```

Manual checks:

1. Открыть приложение, перейти на файл с зарегистрированной системной ассоциацией и нажать `Enter`.
2. Дважды щелкнуть по тому же типу файла.
3. Нажать `Enter` на `.exe` или дважды щелкнуть по нему на тестовом безопасном исполняемом файле.
4. Один раз щелкнуть по файлу и убедиться, что он только становится текущим.
5. Щелкнуть по одному элементу, затем по другому, и убедиться, что открытия нет.
6. Дважды щелкнуть по папке и сравнить результат с `Enter` на той же папке.
