# 0014 - Quick View Panel

Type: Spec
Status: Current
Replaces:
- none

Related:
- none

## Goal

Provide an enhanced Quick View panel (Ctrl+Q) that shows rich, context-aware previews of
the currently selected item in the opposite panel — including full directory details with
asynchronous size calculation and text file metadata at the bottom.

## Context

The original Quick View showed only a file's text content or a one-line directory summary.
Users need richer information without opening a full viewer: directory stats, BOM / EOL type,
and the associated application — all without blocking the UI.

## Scope

### Panel frame and styling
- Double-line border (`╔═╗║╚╝`) matching the style of regular panels.
- Same colors as an active panel (`PanelBorderActiveFg`, `NormalFileFg`, `PanelTitleFocusedFg`).
- The active panel retains its own view mode (Full / Brief) while Quick View is active.
- Mouse clicks on the Quick View panel are ignored.

### Directory view
- Full path, creation / modification / access timestamps.
- Count of files and direct subdirectories (top level only, synchronous).
- Total size computed asynchronously in a background task:
  - Starts when the cursor moves to a directory.
  - Cancelled and restarted when the cursor moves to a different item.
  - Progress updated with 300 ms throttle; UI woken via `_refreshCts.Cancel()`.
  - Intermediate display: `<size>  (scanning…)`, final: `<size>`.
  - Access errors collected and displayed below `Errors: N` in a dark-yellow style.
- File attributes (ReadOnly, Hidden, System, Archive, Compressed, Encrypted).

### Text file view
- File content displayed in the upper portion.
- A horizontal separator line (`╟──[ AppName ]──╢`) at the bottom, with the name of the
  associated Windows application (from registry `HKCR\<ext>\shell\open\command`) centred.
- Below the separator: one info row showing `BOM: <value>  EOL: <value>`.
  - BOM values: `UTF-8`, `UTF-16 LE`, `UTF-16 BE`, `UTF-32 LE`, `UTF-32 BE`, `No`.
  - EOL values: `Windows (CRLF)`, `Unix (LF)`, `Classic Mac (CR)`, `Mixed`.
- Info section only shown when the panel has more than 3 visible rows.
- Recognised text extensions: ~70 common types including source code, markup, config,
  script, and data formats (see `TextFileInfo.TextExtensions`).

### Async correctness
- Background callbacks use `_refreshCts.Cancel()` only (no `Exchange`).
- `ResetRefreshCts()` — called in the `catch (OperationCanceledException)` branch —
  is the only place that creates a new CTS, eliminating the missed-signal race.

## Non-goals

- Scrolling inside Quick View.
- Binary file preview.
- Editable Quick View.
- Cross-platform registry fallback for associated apps.
- Recursive first-level file / subdirectory counts (only top level).

## Acceptance criteria

- Ctrl+Q shows the opposite panel as Quick View; active panel keeps its view mode.
- Selecting a directory shows path, timestamps, file/dir counts, async total size, attributes.
- While scanning, size shows current partial value with `(scanning…)` suffix.
- On completion, `(scanning…)` disappears.
- Moving to another item cancels the previous size scan immediately.
- For text files, separator line shows associated app name; info row shows BOM and EOL.
- `BOM: No` shown when there is no BOM (not "No BOM").
- Errors during size scan shown in dark-yellow below the error count.
- No UI freeze while size is being computed.

## Verification

```
dotnet build
dotnet test
```

Manual: open a large directory in Quick View and navigate away before scan completes —
the scan must stop; a new one must start for the new directory.
