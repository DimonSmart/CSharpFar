# 0027 - Far command-line shortcuts

Related:
- 0013
- 0018
## Goal

Add Far-like command-line insertion shortcuts for panel items, panel paths, and
command history navigation.

## Context

CSharpFar already has a command line, panel navigation, and command history. Far
Manager users expect several common `Ctrl` shortcuts to place panel-derived text
or recent commands into the command line without opening a dialog.

## Scope

Implement these key bindings when the main file manager UI is active:

1. `Ctrl+Enter` inserts the file or folder name under the cursor into the
   command line.
2. `Ctrl+F` inserts the full file path for the item under the cursor in the
   active panel.
3. `Ctrl+[` inserts the current full path of the right panel.
4. `Ctrl+]` inserts the current full path of the left panel.
5. `Ctrl+E` replaces the command line with the previous command from history.
   Repeated presses continue moving older through history.
6. `Ctrl+X` replaces the command line with the most recent command from history
   on first press. Repeated presses continue moving newer through history once
   navigation has moved to older commands.

Inserted names and paths must be quoted when the inserted text contains spaces.
History navigation through `Ctrl+E` and `Ctrl+X` must replace any currently typed
command-line text.

## Non-goals

Do not implement in this spec:

1. configurable keyboard shortcuts;
2. shell-specific escaping beyond quoting inserted text that contains spaces;
3. file-name completion or path completion;
4. persistent history changes beyond using the existing command history;
5. changes to function-key bar labels.

## Acceptance criteria

- `Ctrl+Enter` appends the current item name to the command line.
- `Ctrl+F` appends the current active-panel item full path to the command line.
- `Ctrl+[` appends the right panel current path.
- `Ctrl+]` appends the left panel current path.
- Inserted text containing spaces is wrapped in quotes.
- Inserted text without spaces is not wrapped in quotes.
- `Ctrl+E` replaces existing command-line text with the most recent history
  command on first press and then older commands on repeated presses.
- `Ctrl+X` replaces existing command-line text with the most recent history
  command on first press and with newer history commands when navigation is
  already active.
- Existing command execution, `Alt+F8`, visible history completion, hidden-panel
  Up/Down browsing, and panel navigation keep working.

## Verification

Manual checks:

1. Put the cursor on a file and press `Ctrl+Enter`; verify the file name appears
   in the command line.
2. Put the cursor on a file in a path with spaces and press `Ctrl+F`; verify the
   full path is quoted.
3. Press `Ctrl+[` and `Ctrl+]`; verify they insert the right and left panel
   paths respectively.
4. Type arbitrary command-line text, press `Ctrl+E`, then press `Ctrl+E` and
   `Ctrl+X` repeatedly; verify the line is replaced and history moves in both
   directions.
