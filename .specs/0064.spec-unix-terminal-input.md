# 0064 - Unix terminal input

Type: Spec
Status: Current
Related:
- 0004
- 0017
- 0057

## Goal

On Unix-like hosts, CSharpFar can read keyboard, mouse, and resize input from a
raw VT byte stream while preserving an explicit legacy `Console.ReadKey`
fallback. The application continues to consume only the shared
`ConsoleInputEvent` model through `IConsoleDriver`.

## Backend selection

The Unix console driver owns a replaceable internal input reader. The
`CSHARPFAR_UNIX_INPUT` environment variable selects it:

- `raw-vt` selects raw VT keyboard and SGR mouse input;
- `legacy` selects the key-only `Console.ReadKey` reader;
- an unset variable selects `legacy` for a conservative rollout;
- any other value is rejected with a clear configuration error.

Diagnostics identify the selected backend and whether mouse tracking is
enabled. Windows input behavior and the public `IConsoleDriver` contract remain
unchanged by this feature.

## Raw VT behavior

The raw VT reader:

- enables Unix raw terminal mode while it is active;
- parses ordinary and UTF-8 text, Enter, Tab, shifted Tab, Backspace,
  standalone Escape, arrows, navigation keys, F1-F12, CSI modifiers,
  Alt-prefixed text, and Alt-prefixed VT keys;
- parses SGR mouse button press, release, wheel, and button-motion packets into
  zero-based viewport-relative `MouseConsoleInputEvent` coordinates;
- polls terminal size and emits `ConsoleResizeInputEvent` when it changes;
- ignores malformed mouse packets and handles unknown keyboard sequences
  without terminating the application;
- does not require enhanced keyboard protocols or modifier-only events.

`ReadKey` skips mouse and resize events until it receives a key event so legacy
dialog callers keep their existing contract.

## Terminal mode lifetime

Only the raw VT backend enables xterm cell-motion tracking (`1002`) and SGR
mouse encoding (`1006`). It does not enable all-motion tracking (`1003`).

Leaving the application, disposing the driver, or entering child-process
console mode restores the original terminal input mode and disables mouse modes
`1000`, `1002`, `1003`, and `1006`. Returning from child-process mode restores
raw input and `1002 + 1006` mouse tracking.

The diagnostic input laboratory remains available and does not define
production input behavior.

## Acceptance Criteria

- `AnsiTerminalConsoleDriver` delegates production input to one selected
  internal input-reader abstraction.
- `legacy`, `raw-vt`, and the conservative unset default select the documented
  readers.
- Raw VT keyboard and SGR mouse input produce only standard
  `ConsoleInputEvent` values.
- Mouse coordinates are zero-based and viewport-relative on Unix, matching the
  application-facing mouse coordinate semantics.
- Raw mode and mouse tracking are disabled for child processes and restored
  after the child-process scope ends.
- Disposal restores terminal and mouse modes even after input or parsing
  failures.
- Unknown keyboard sequences and malformed mouse packets do not crash the UI or
  write diagnostic text into it.
- Parser tests cover text, UTF-8, standalone Escape, navigation, F1-F12,
  modified keys, Alt-prefixed text and navigation, mouse buttons, release,
  wheel, motion, modifiers, and coordinate conversion.
- Backend-selection tests cover `legacy`, `raw-vt`, and the unset default.

## Verification

Automated verification:

```bash
dotnet build
dotnet test
```

Manual verification in Windows Terminal plus WSL runs the Unix host with
`CSHARPFAR_UNIX_INPUT=raw-vt` and checks keyboard navigation, F-keys,
Alt/Ctrl chords, Latin and Cyrillic Alt-prefixed text, mouse buttons, wheel,
drag, child-process entry/return, and clean terminal restoration. A second run
with `CSHARPFAR_UNIX_INPUT=legacy` checks the previous keyboard behavior and
confirms that mouse reporting is not enabled.
