# 002. Стратегия буферизации отрисовки

Source: `.worklog/0002.adr-render-buffering-strategy.md`.

## Цель

Убрать мигание и лишние записи в консоль без ручного протаскивания dirty regions через UI-код.

## Решение

`ScreenRenderer` владеет двумя буферами отображения:

* `front buffer` — последний известный кадр, уже отправленный в console driver;
* `back buffer` — новый логический кадр, который строится текущей отрисовкой.

Приложение рисует полный логический кадр через `BeginFrame()`. На завершении кадра renderer сравнивает `back buffer` с `front buffer`, группирует изменившиеся ячейки в горизонтальные участки с одинаковым стилем и вызывает `IConsoleDriver.WriteAt` только для этих участков.

## Правила реализации

* `Application` не вычисляет dirty regions и не передаёт область перерисовки в renderer.
* `Application` решает только, нужно ли перерисовать UI после события.
* Если UI видим и нужна перерисовка, рисуется полный логический экран.
* `Rect` остаётся типом геометрии компонентов, областей очистки и `Capture`/`Restore`; он не используется как dirty-region API.
* `Capture` и `Restore` должны синхронизировать `front buffer`, иначе следующий diff может считать реальные ячейки устаревшими.
* При изменении размера консоли буферы пересоздаются, следующий видимый кадр считается полным.
* На Windows событие `WINDOW_BUFFER_SIZE_EVENT` должно будить главный цикл и приводить к полному логическому redraw.
* Для UI-отрисовки на Windows отключается wrap-at-EOL, чтобы запись в правую нижнюю ячейку не переносила нижнюю строку и не скроллила окно. Перед shell output исходный режим консоли восстанавливается.

## Нижняя строка меню

Нижняя строка меню не переносится на новую строку. Если все элементы не входят по ширине:

* содержимое обрезается в доступной ширине;
* в крайней правой позиции рисуется красное `...`;
* если ширина меньше трёх символов, троеточие обрезается до доступной ширины.

## Тесты

Минимальные регрессии:

* повторный идентичный кадр не делает физических `WriteAt`;
* перемещение курсорной подсветки меняет только реально отличающиеся строки;
* `Capture`/`Restore` синхронизируют буфер renderer;
* resize форсирует новый полный кадр;
* нижняя строка меню остаётся одной строкой на чётных и нечётных ширинах;
* режим UI-output делегируется в драйвер и может отключать wrap-at-EOL.
