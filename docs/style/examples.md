# README style examples

## Example 1: project intro

Bad:

```markdown
This powerful and intuitive library is designed to simplify text comparison and unlock new possibilities for developers.
```

Good:

```markdown
This library compares two text values and renders the difference as Markdown.
```

## Example 2: agentic tool intro

Bad:

```markdown
A revolutionary AI-powered solution that transforms the way developers interact with browser automation.
```

Good:

```markdown
BrowserCommander lets an LLM agent inspect and control an open browser tab through MCP tools.
```

## Example 3: review tool intro

Bad:

```markdown
This project provides a robust and scalable solution for automated pull request reviews.
```

Good:

```markdown
This project runs automated review rules on pull request diffs and can create line comments for changed files.
```

## Example 4: limitations

Bad:

```markdown
The project is constantly evolving and will continue to improve over time.
```

Good:

```markdown
The current version only reviews changed lines. It does not perform full-project static analysis.
```

## Example 5: usage

Bad:

```markdown
Getting started is quick and easy.
```

Good:

```markdown
Run the reviewer against a local diff:
```
