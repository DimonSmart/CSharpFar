# README review checklist

Before committing README changes, check:

## Meaning

- [ ] The text says what the project actually does.
- [ ] No new unsupported facts were added.
- [ ] No fake maturity claims were added.
- [ ] Limitations are not hidden.

## Style

- [ ] The first paragraph is concrete.
- [ ] There is no marketing tone.
- [ ] There are no generic LLM phrases.
- [ ] There is no filler after headings.
- [ ] Sentences are not all the same length.
- [ ] The text does not sound like a product landing page.
- [ ] Long dash characters are not used.

## Technical clarity

- [ ] Installation steps are executable.
- [ ] Usage examples compile or are clearly marked as pseudo-code.
- [ ] Configuration examples match actual repository files.
- [ ] Terminology is consistent.
- [ ] Links point to existing files.
